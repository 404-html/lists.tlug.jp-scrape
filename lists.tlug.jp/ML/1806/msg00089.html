 > zip and p7zip are not actually upstream of the Python standard library
 > Zip code, IIRC; it's a completely standalone, separate implementation.

That's correct.  The low-level compression code (zlib, bzip2, lzma) is
imported from the independent projects by those names, but the code to
handle the zip file format is written in Python.
