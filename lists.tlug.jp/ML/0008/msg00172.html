Hiroyuki Ohba (ohba@example.com) wrote:

Hiroyuki> <meta http-equiv="Content-Type" content="text/html; charset=x-sjis">
                                                                      ^^^^^^^^
this is evil.  charset=s-jis is correct.  Start with http://www.w3c.org/ (the
World Wide Web Consortium) for more.

Those who say it is not an Apache problem are correct.  This is
either an encoding problem, or a files-mangled-in-upload problem.
Also, the most accepted standard for web page encoding is iso-2022-jp.
Plus, of course, it's kind of embarrassing to be serving up SJIS from 
a *NIX box.
