> Japanese input with OOo works if LC_CTYPE is set to a Japanese locale.

This really tends to vary with distro (or O/S, e.g the BSDs.)
Sometimes, you need the LC_CTYPE to be set to ja_JP.UTF-8 sometimes it
will work with en_US.UTF-8 (most Linux distros do it a bit differently,
I think, the UTF-8 as opposed to utf-8 or utf8 is on FreeBSD).  

I've also found that sometimes, you can set the LC_CTYPE to English and
an applicatoin will work as long as you make sure to add the
GTK_IM_MODULE="scim" to whatever program calls OO.

On the other hand, I'm referring to using scim and I believe the OP is
using uim.  Also, at this point, we're not even sure that it's not
simply an Ubuntu problem, correct?
