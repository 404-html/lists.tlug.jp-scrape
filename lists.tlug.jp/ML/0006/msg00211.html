OK, digging a bit in the docs, I found the equivalent of the 'permit tcp
... established' thing in Cisco.. basically you have to say something
like:

ipchains -A input -p tcp -s 0/0 -d 192.168.81.0/24 !-y -j ACCEPT

which means, "allow packets that do NOT have the SYN bit set and do NOT
have the ACK and FIN bits cleared".

Found it in the manpage, actually.  Provides a good description of how it
works, IMHO.
