What's the equivalent, in ipchains, of the following Ciscoism:

access-list 101 permit tcp 192.168.81.0 0.0.0.255 any established

(basically what this does is allow 192.168.81.0/24's ESTABLISHED TCP
connections to anywhere.. ie, if the RST or ACK bits are set on an
incoming packet, it flags it as part of the already-established connection
and lets it through.  If the bits are not present, it thinks it's a new
connection and therefore denies it).

How does ipchains do that?
