Eric Standlee <fwiw3980@example.com> writes:

> Why does my netscape blow up when it encounters java and how do I fix it?
> sys RH 6.0 with Japanese enabled Netscape.

I had the   same Problem  with  RH   6.0 and Netscape  4.6   (`normal'
Netscape,  NOT Muriyari). The  following advice (which I received from
the Halloween-Linux mailing-list) fixed it:

Type 

   /usr/sbin/chkfontpath --list

You will probably get:

    1: /usr/X11R6/lib/X11/fonts/misc:unscaled
    2: /usr/X11R6/lib/X11/fonts/75dpi:unscaled
    3: /usr/X11R6/lib/X11/fonts/100dpi:unscaled
    4: /usr/X11R6/lib/X11/fonts/misc
    5: /usr/X11R6/lib/X11/fonts/Type1
    6: /usr/X11R6/lib/X11/fonts/Speedo

i.e. you have only entries "...:unscaled" for 75dpi and 100dpi fonts.
As superuser, type

    /usr/sbin/chkfontpath --add /usr/X11R6/lib/X11/fonts/75dpi
    /usr/sbin/chkfontpath --add /usr/X11R6/lib/X11/fonts/100dpi

Of course you should have the 100dpi fonts installed before you do
that. If not do `rpm -i XFree86-100dpi...' first.
After you have done that, use `chkfontpath --list' again, the result
should now look like:

    mike@example.com ~$ /usr/sbin/chkfontpath --list
    Current directories in font path:
    1: /usr/X11R6/lib/X11/fonts/misc:unscaled
    2: /usr/X11R6/lib/X11/fonts/75dpi:unscaled
    3: /usr/X11R6/lib/X11/fonts/100dpi:unscaled
    4: /usr/X11R6/lib/X11/fonts/misc
    5: /usr/X11R6/lib/X11/fonts/Type1
    6: /usr/X11R6/lib/X11/fonts/Speedo
    7: /usr/X11R6/lib/X11/fonts/100dpi
    8: /usr/X11R6/lib/X11/fonts/75dpi

I still have no idea why this font setup fixes the Netscape crashes on
Java pages, but it really seems  to do it. I did  this 2 months ago on
my RH 6.0 and  didn't have a single Netscape  crash since then. Before
it was crashing regularly.
