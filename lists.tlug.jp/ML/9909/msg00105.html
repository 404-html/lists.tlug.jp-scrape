"Stephen J. Turnbull" <turnbull@example.com> writes:

> Another problem is that Red Hat still does not support XEmacs; it's
> contrib AFAIK so Red Hat probably doesn't care enough to test it.
> If it's not even distributed by RH (Jens's URL was somewhere else,
> then all bets are off).

XEmacs is actually in Redhat's Powertools (XEmacs-21.1.2 for the 6.0
Powertools) to be precise: it worked as well for me as the newer
21.1.4 JRPMs.  Apart from version differences the main difference 
is that there are JRPMS for various combinations of input methods
(Canna, Wnn, XIM) with or without X11 support.  (With both of them the
documentation for dumped symbols is offset, but for people learning
XEmacs this shouldn't matter.)
