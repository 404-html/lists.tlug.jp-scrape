><HTML>
>Hi all,
>
><P>I've been using Netscape Communicator for Linux for a couple of weeks
>and I've noticed the following errors:

One "feature" you didn't notice: it's configured to send mails
as "rich text" by default. Yechh!
>
><P>1)&nbsp; Doesn't check for mail automatically when you open the mail
>window.

This isn't that bad for all people who use dial-up networking as
it allows you to write a mail message offline.

><BR>2)&nbsp; Doesn't change to arrow to allow scrolling through the page
>until it has finished loading.

This depends on the page you're loading. Long plain-text pages
can be scrolled while loading without a hitch - it really depends
on how far Netscape can predict the layout in advance. This is
obviously not possible for pages with long/complex tables and
lots of images without size information. 
>
><P>Have any of you had the same problems, and if so did you find a fix
>for these problems?

You missed one - you can't switch the font size for japanese
text without restarting the beast. 

There's really only one big problem with Netscape 4 - it dies
even more often than Netscape 3. Oh well, you get what you
pay for.
