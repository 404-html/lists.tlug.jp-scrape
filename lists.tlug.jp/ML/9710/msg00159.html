on Sun Oct 26  Magnus Therning <d94magth@example.com> wrote,

< Warning: can't open rule.func

The file name rule.func is set in the /usr/lib/X11/ccdef/ccdef.kinput2.
but there is not a file named rule.func, so warning occurs.
you comment out rule.func and set to include rule.eggfunc in the ccdef.kinput2 file. 
be sure there is a rule.eggfunc in the /usr/lib/X11/ccdef  directory.

	    #include 'rule.func'
	    include 'rule.eggfunc'

< Warning: ERROR: setdic: can't create hindo (usr/root/kihon.h)

usr/root/kihon.h is a /usr/local/lib/wnn/ja_JP/doc/usr/root/kihon.h.
the meaning of the warning is wnn can't create a file in the directory.
make wnn's account in the /etc/passwd. make owner of the /usr/local/lib/wnn/ja_JP/doc/usr/root directory wnn.
 
	wnn:*:15:15:Wnn:/bin:  		only an example

	chown wnn /usr/local/lib/wnn/ja_JP/doc/usr/root

[Ctrl Kanji] starts conversion, ends by [Shift Space].
