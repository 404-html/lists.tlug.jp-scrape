  Hi,

  I mentioned that xcdroast's latest incarnation now makes use of 
cdrecord (rather than cdwrite) to do its dirty work.  It seems to
work pretty nicely, and the X interface sure is nicer than the command
line.

  I'm having a few problems with CD Mastering ( which uses the  mkisofs
1.11 to do the nitty-gritty) that I haven't figured out yet.  A single
file in 100s of megabytes will contain an error that makes it
unreadable (this is not a CD-writing error, but an error in the image
before it's written out to the CD).  I need to play around with this
some more to really be sure what's happening.  But it is an annoying
glitch.  

  What I really wanted to ask about, was if anyone knows how I can
access the tracks of a multi-session CD beyond the first track.  You
can actually write multi-session CDs using 'cdrecord' directly (xcdroast
doesn't yet support multi-session), but I don't see how I can read 
these.  Xcdroast can see the tracks, but lists them as unreadable,
just like the data track on some audio CDs:
	DATA: CD-Extra (unreadable)
And, of course, when you 'mount' a CD with multiple tracks, you only get
the first track.

  If I could access these multisession tracks somehow, it would make
the CDs much more usable for regular backups, since I wouldn't have
to squander a whole CD blank, just to write out 30 or 40 megabytes
(although with CD blank prices now down to Y300 or lower, I'm tempted
to use them anyway for my regular weekly data backups).

  Any thoughts out there on this?
