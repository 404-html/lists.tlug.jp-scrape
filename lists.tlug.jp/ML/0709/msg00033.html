On Tue, 4 Sep 2007, emiddleton@example.com wrote:

If it starts using swap as cache then you are going to see serious
churn.


If it's the buffer cache you're talking about, this is impossible. Both
disk partitions and swap are the backing store for pages in memory; it
would be quite illogical to take a page for which you already have a
location on disk that holds the data and write the data again to the
disk in another place.

When running low on memory, dirty pages in the buffer cache are written
and discarded, and clean pages are simply discarded.
