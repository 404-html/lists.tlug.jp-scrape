>> If you would prefer to use an
>> English locale, some hacking is required in /etc/fonts.
>
> I definitely want the English locale. Do you have some pointers for the
> /etc/fonts hacking, as I've never done that before.

/etc/fonts holds the configuration for fontconfig, the font subsystem
currently used for both GTK (Gnome) and QT (KDE), replacing the
antiquated "core X font subsystem."  You can see the man page using
`man fonts.conf`, or here is an online version:
http://fontconfig.org/fontconfig-user.html

I currently cannot afford to get (too) distracted, but at some point I
will prepare a clean VM, test some settings, and report back.
