Christian,

Thanks so much for all the help.

Armed with your advice, I was up until 4 last night trying to get the
damned thing to work.

In the end? Turns out that the Remote Desktop server was not being
enabled on startup. I'm surprised that makes a difference, since I would
have thought it would start at the moment I manually enable it in the GUI.

Nonetheless, checking to enable it in the start up applications
interface, and then rebooting, has done the trick. I'm both happy that
I've got it working, and frustrated that I went all around the world
before finding the right box to check.

In any case, thanks so much for all the diagnostic assistance. In the
end, the only way I was able to find the right answer was by eliminating
the other possibilities.

Thanks as always to all of TLUG for everyone's great advice.
