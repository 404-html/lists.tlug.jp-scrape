>> On Tue, Apr 10, 2001 at 07:02:54PM +0200, Mike Fabian wrote:
>> > I think several applications look for LC_CTYPE to decide whether they
>> > should use XIM or not.
>> 
>> Okay, I'll clarify: XIM seems to care about LANG, and only LANG.  I regard
>> applications that depend on other environment variables to determine whether
>> to register with XIM as broken -- they add unnecessary complexity to an
>> already overcomplex system.

Amen to that. I think it's perfectly right to expect the supported
languages to be stated in LANG, but to expect a user to convert the
whole locale to that language before being permitted to enter text in it
is plain dumb.

However, the man page for XOpenIM says:

	"The XOpenIM function opens an input method, matching the
	current locale and modifiers specification."

This seems to contradict what Chris said, unless "locale" is loose and
includes the values in LANG.

Another question. Having multiple languages in LANG is presumably OK?
I currently have it set to "en_US ja_JP", and RH seems to behave with
it.
