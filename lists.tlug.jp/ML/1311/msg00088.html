Thanks. And now I will read up on how to use them properly. I seem to know a bunch of improper ways ... :-[
VDIs ar not stand-alone, there is a XML description to them in another file and sometimes paths are mixed-up. That is why you need to at least import VDI on the new machine, it will map the UUID of the VDI properly (unless you copied the original in the first place and now have VDI with the same UUID but different content. Also with VDIs, make sure you get all the snapshots as well, they may be in another directory.
This is what I've done a couple of times, actually. I've always had fewer problems changing HDDs if I just mirror the .vdi files and let the new version of VirtualBox create new subdirectories, complete with new supporting files. This is the first time I haven't wanted everything else on the disk because I am changing OSes completely.
