I concuded that Ahmed is using rp-pppoe as he has said that he is using 
"Roswell", "NTT FLETS", "OCN".

U may or may not know that rp-pppoe's adsl-setup is a script which 
updates following major files
    "/etc/ppp/pap-secrets"
    "/etc/ppp/chap-secrets"
    "/etc/ppp/pppoe.conf"

Hence, I had suggested him to update the files accordingly. (and u again 
misunderstood my comments for not using adsl-setup).

In first two files, he is needed to write as follows
    "login-name * password"
and in the last file, updating "USER=....." line is enough.

Added to these, adsl-setup updates /etc/resolv.conf and 
/ets/sysconfig/network files

Why I asked him to delete all the above files and reinstall, and update 
manually, because, sometimes, the firewall conf script of rp-pppoe makes 
conflicts with RH firewall scripts. Added to this, rp-pppoe in Roswell 
has one problem. It don't provide "/etc/ppp/pppoe.conf" by default (wrt 
RH7.0). Now, it is decided that it is not a bug, but the setup policy is 
changed. These are too confusing information to the original poster 
hence I have not mentioned all of these things

There are also more things involved, but I was waiting for Ahmed's 
feedback before putting my suggestions.
