I found it quite good actually.   Like qemu, by default it does nat type
networking, where the virtual machines gets an address of 10.x.x.x
(Actually, I think the first and possibly the second x there are
predetermined) and can't ping and so forth.  One had to use bridging to
give it an address on the local LAN, and so it was a bit more of a
nuisance than VMWare server, especially for only occasional use. 

It's definitely worth checking out, especially if you are only using it
for occasional testing and there's no need to give it an address on your
LAN.  (Not that it's hard, but it just involves a little more work.)
