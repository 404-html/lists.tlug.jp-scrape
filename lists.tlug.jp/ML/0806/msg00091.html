I'm not quite sure how to troubleshoot this one.  
Fedora Rawhide, which is often broken.
64 bit, with nspluginwrapper.  As per Fedora wiki instructions both 64
bit and x68 nspluginwrapper.

I boot into text mode and have a little script to let me choose between
Gnome, KDE and fluxbox.  

Sound works in general.  However, sound in flash videos only works if I
boot into Gnome.  If I boot into flux or KDE, and go to youtube, playing
a video has no sound.  (Other sounds work in both, e.g. mplayer, the KDE
startup sounds, etc.)

The only error message I found was 
which: no soundwrapper in (and then my $PATH.)

Googling found little.  Some people have had the issue in SuSE, but it
wasn't DE or WM specific.  AFAICT, there's no soundwrapper package for
Fedora--from googling, it seems as if the package is used in some other
distros (or O/S's)  :), and it seems to be a somewhat problematic
package. 

So, it seems as if Gnome does something in place of soundwrapper, but I
have no idea what.  I'm not sure what other information is useful--it's
already 2:00 AM here, and I'm done trying to solve it tonight.  

Any Gnome experts out there who can tell me what it does that the others
don't?  (Or what else to look for, what other info would be useful,
etc.)

I've posted the question on Fedora-testing and the forums--one fellow on
the forums said he'd try to duplicate it, but I was already tired when
posting and neglected to mention the essential fact that it is a 64 bit
install, so I don't know if he'll have similar results or not. 

I don't want to use gnash or that swfsomething, they seem to also be
problematic even on 32 bit systems.  (Linux himself had a bug report on
RH's bugzilla.)   Flash (usually) seems less problematic, even on 64
bit.  I don't have the problem on a 64 bit CentOS installation. 


Thanks for any insight.
