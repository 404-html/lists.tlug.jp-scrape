On 6 September 2011 17:02, Darren Cook <darren@example.com> wrote:
> Could it be the JSON is sending an HTTP header that says a different
> encoding?

One tip to check this out:

Install the Firebug extension for Firefox if you haven't already, and
look under its "Console" tab. (You may need to set the console display
to "有効" or "Active" or something.)

Use your app so it makes the ajax request you're suspicious of, then
look at the header information - it'll give you both what your page is
sending and what the server is sending back.
