On Fri, Jun 27, 1997 at 06:00:44PM +0900, Tim Meggs wrote:
> The wierd thing is that it *only* happens the first time I boot the machine
> from power on - normally withing say 5-10mins.   If after beeing dumped back 

That sounds rather like a hardware problem... with some temperature
sensitivity.

I had been having the Watchdog reset... Window underflow errors, but they
only seemed to occur when I was using Grail for web browsing *and* had
graphics enabled (and did some mouse activity?).  I recently updated my libc
and X* stuff with the RedHat 4.2 stuff and the problem has gone away.  My
machine now stays up for days/weeks without problems even with browsing.

Jim

P.S.  I've also seen repeated whining about "progressive slowdowns" on Sun4C
machines on the sparclinux list, which people blame on "memory
fragmentation" and/or some unspecified SCSI problems.  I've not noticed such
an effect over a matter of weeks... but those that do see it claim that it
will happen in hours.  I suppose it could be network related, since I'm on
a quiet home LAN.
