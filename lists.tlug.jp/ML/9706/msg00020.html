>>>>> "Craig" == Craig Oda <craig@example.com> writes:

    Craig> From: Steve Dunham <dunham@example.com>

    >> Actually, I have the "Menu" button on my MS keyboard mapped to
    >> "Kanji".
    >> 
    >> xmodmap -e "keycode 117 = Kanji"

    Craig> Steve, thanks for the tip on this.  I was too lazy to find
    Craig> the keycode for that funny square key with the cross
    Craig> through it.  Is there any way to call up a program that
    Craig> tells me what the keycode is just by pressing the key?

xev will do this for you; unfortunately you apparently can't set a
mask so that it doesn't tell you about all the motion-notify events
when you move the mouse.  (Ie, you will get gobs of irrelevant
output.)  There are also keyboard mapping programs (I'm pretty sure I
saw one in dselect for Debian but don't remember its name).

    Craig> One thing that I'm curious about though, what is the
    Craig> advantage of having the Kanji henkan key defined in
    Craig> kinput2?

This depends on the actual keyboard layout.  My rule is just pay
attention to times when "I wish this function was attached to a
convenient key" and either note them for later processing or do them
on the spot.
