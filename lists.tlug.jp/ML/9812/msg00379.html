Scott Stone <sstone@example.com> wrote:
>   On Tue, 15 Dec 1998 mike.fabian@example.com wrote:
>   > [... problem with new samba RPM from TL on RH 5.2 ...]
>   > Is this possibly a compatibility problem of Redhat 5.2 and TL3.0E?
>
>   try the TL binary RPM as well (should be lib compatible with RH 5.2).  If
>   THAT works, then something is messed up with your compiler, since
>   smbclient -L host does work on TL.  

I tried again and it works now. I only had to use 

smbclient -U fabian -L host

and then I  got the prompt for   the password and got the  connection.
The old samba version  from Redhat used  the login-name of the user as
default, the new samba version from Turbolinux  doesn't. Why not?  But
apart from that it works well. Both the binary RPM downloaded directly
from TL and the binary RPM compiled from the source RPM work. 

Thank you very much. The smbsh feature is really nice. You wrote:

>  Drawbacks are that it can take a long time for new
>  machines/shares to show up in /smb, but it does work.

Do they show up in  a "ls /smb" after some  time? If so I didn't  have
enough patience.   It appears that they  can   be accessed immediately
though,  one   only has   to    know  the  name   then  one   can  "cd
/smb/mwt414/tmp; ls".

>   What does an 'ldd /usr/bin/smbclient' look like on your system?

With the "old" samba from RH 5.2 it looks like this:

fabian@example.com ~$ ldd /usr/bin/smbclient
        libnsl.so.1 => /lib/libnsl.so.1 (0x40004000)
        libcrypt.so.1 => /lib/libcrypt.so.1 (0x4000a000)
        libdl.so.2 => /lib/libdl.so.2 (0x40037000)
        libpam.so.0 => /lib/libpam.so.0 (0x4003b000)
        libc.so.6 => /lib/libc.so.6 (0x40042000)
        /lib/ld-linux.so.2 => /lib/ld-linux.so.2 (0x00000000)

And after installing the TL binary RPM:

fabian@example.com ~$ ldd /usr/bin/smbclient 
        libdl.so.2 => /lib/libdl.so.2 (0x40004000)
        libcrypt.so.1 => /lib/libcrypt.so.1 (0x40007000)
        libpam.so.0 => /lib/libpam.so.0 (0x40034000)
        libc.so.6 => /lib/libc.so.6 (0x4003c000)
        /lib/ld-linux.so.2 => /lib/ld-linux.so.2 (0x00000000)

but I don't know whether this is OK or not.
