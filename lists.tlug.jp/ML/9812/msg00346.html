BTW, I wouldn't trust egcs as a replacement of gcc in the
first place.  I am pretty sure that it breaks sometimes, ie,
generates bad code.  This may not occur in handwritten
programs, but I am using a high-level compiler, which
generates C code and leaves the low-level code generation to
the C compiler.  The generated code admittedly uses some of
gcc's very special features to produce optimized code.

I tried to use egcs instead of gcc and got some pretty wired
error messages about registers which should have not been
spilled etc.  Translating machine generated code is always
the hardest stress test for a compiler and egcs still seems
to have a way to go.
