On Fri, 11 Dec 1998, Renaud ITIER wrote:

> Hello,
> 
> I am installing a new linux box with Rh 5.2 to act as a firewall.
> 
> My machine is a Compaq Deskpro, and I have 2 netcards. One is a Realtek
> RTL8019 Pnp ISA bus and the other one a AMD PCNET Family Ethernet Adapter
> PCI. Well none of them are not found by the install...
> 
> I finished the installation without the network setup, but I don't really
> know what to do next to fix the netcard problem.
> 
> Then I try to add the interface inside netcfg but couldn't activate the eth0
> and eth1 (Activate delayed).
> 
> I reached my knowledge limit, so please would you tell me what should I do?

OK, just to simplify:

eth0 == RTL8019 == NE2000 clone == use ne.o
eth1 == AMD Lance == AMD Lance driver == use lance.o

Set up /etc/conf.modules like this:

alias eth0 ne
options ne io=0x300
alias eth1 lance

you may need an options line for the lance, too, if it's ISA.  NE2k cards
come from the factory set to 0x300, so the above should at least get eth0
working.

A better solution is to use PCI cards - they're autodetectable and barely
cost any more (maybe 1 or 2 dollars) than their ISA counterparts.
