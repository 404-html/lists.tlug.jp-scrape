I see.  Hm.. the old 'chicken and egg' problem revisited, although
slightly different.

Oh well, sometimes I'm wrong.  At least now I know.  I really thought that
egcs/gcc didn't require themselves to build, but... live and learn, I
guess :)

> 
> Look:
> 
> bash-2.02$ ../egcs-1.1b/configure --prefix=/usr i586-linux

> ../egcs-1.1b/configure: cc: command not found
> *** The command 'cc -o conftest -g   conftest.c' failed.
> *** You must set the environment variable CC to a working compiler.

> creating cache ./config.cache
> checking host system type... i586-pc-linux-gnu
> checking target system type... i586-pc-linux-gnu
> checking build system type... i586-pc-linux-gnu

> configure: error: no acceptable cc found in $PATH
> 
> (not really 1.1b, it's patched up to 1.1.1)
> 
> It don't go.  No configure, then no Makefile.  No Makefile, no compiler.
