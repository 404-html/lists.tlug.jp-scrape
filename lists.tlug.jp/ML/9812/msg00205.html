On Wed, 9 Dec 1998, Jim Schweizer wrote:

> Hi all,
> 
> I'm just not getting this upgrade with package managers thing.
> 
> Say, for example, that I want to upgrade apache to 1.3 and I
> want to add mod_speling which is not included by default. How do
> I do it? I don't see how rmp -Uvh gives one tuning control.

upgrade, then edit /etc/httpd/conf/httpd.conf and add the line (I think):

CheckSpelling 	on

(yes, this is correct, just checked http://www.apache.org, where the full
docs reside.  Good reading)


I think it's a bit unresasonable to expect RPM or any package manager to
be able to understand and configure any/every package's configuration
files.  We're not to THAT level of automation yet - it still requires some
good old-fashioned know-how to be a system administrator, regardless of
what the MCSE people will try and tell you :)

> 
> BTW, if you're wondering what mod_speling does, it attempts to
> correct misspellings of URLs that users might have entered, by
> ignoring capitalization and by allowing up to one misspelling.

It works, too... I've tried it.  Not the most useful thing in the world,
but interesting.
