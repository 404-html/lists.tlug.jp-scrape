I've been experimenting building modularized kernels. It works
beautifully, except for sound. I'm pulling my hair out trying to work
out why.

The sound module is /lib/modules/2.0.36/misc/sound.o . In the same
directory there is also the psaux.o mouse driver module, and others. All
modules, such as the psaux, load fine automatically.

But doing a "/sbin/modprobe sound" produces the "can't locate sound
module" message.

Any ideas? I've tried passing the fully-qualified pathname to modprobe,
putting sound.o in all the nearby directories, but it still doesn't
"locate" it.

I can provide the output of "/sbin/modprobe -c" if it helps. Redhat 5.1
with (surprise) 2.0.36 kernel. My /lib/modules/2.0.36/modules.dep reads
as follows:-
