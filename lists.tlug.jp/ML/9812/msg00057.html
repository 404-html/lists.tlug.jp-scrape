On Wed, 2 Dec 1998 mike.fabian@example.com wrote:

> 今日は皆さん、
> 
> I tried to install Redhat 5.0 on a PC which has a
> 
> "Adaptec AHA-294xU2/295xU2/AIC-789x PCI Ultra2 SCSI Controller"
> 
> The boot.img disk of Redhat 5.0 could not deal with this
> controller and not access the SCSI CDROM.

5.0 doesn't have a driver for it.

> 
> Therefore I downloaded a new boot.img and supp.img from
> a mirror of the ftp.redhat.com and tried whether these work
> with the old RedHat CD.
> 
> It looks like the new  disk images have a  driver for this Controller,
> they don't complain that they can't find the SCSI  device like the old
> ones did.
> 
> But after the install procedure mounts the CDROM it fails
> with the error message:
> 
> > /usr/bin/runinstall2:   error in loading shared libraries.
> > libresolv.so.2:  cannot open shared object file: No such file or directory.
> > 
> Does this mean that the old RedHat CD is incompatible with the
> new disk images?

yes, that's what it means.

> 
> Would it work when I get a new RedHat CD?
> 

yes, with 5.2.

> Would it work with the current version of Turbolinux?
> 

It will work with 3.0.1 English or 3.0J, but (probably) not with 2.0 E or
J.  3.0J will be released in a couple of days, and 3.0.1 English shortly
thereafter.  3.0 English (the 'Comdex Special Edition') can be FTP'ed now,
and it will also work.
