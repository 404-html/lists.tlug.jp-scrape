On Wed, Dec 09, 1998 at 09:30:44AM +0900, Frank Bennett wrote:
> Note that there is one limitation to a masquerading setup; masquerading
> only works for outbound connections, so you won't be able to station a
> public server behind the firewall -- public servers would have to run on
> the firewall machine itself.

Although you can redirect incoming connections to specific ports on the
masquerading hosts to ports on hosts hiding behind it.  (Recent versions
of the xinetd replacement for inetd include this capability for example.
There is also a small standalone 'redir' program.)
