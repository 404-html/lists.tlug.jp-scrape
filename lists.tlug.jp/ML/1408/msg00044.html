On 08/15/2014 12:17 PM, Stephen J. Turnbull wrote:

Ignore the people who say "ALSA seems to be used as well".  What's
going on is that pulseaudio implements the ALSA driver API (as well as
the OSS API) so that it can talk to the hardware using existing ALSA
drivers, and then there's a (presumably hacked-up) ALSA user-space
library that can talk to ALSA applications.  But pulseaudio infests
everything if you're running it, it appears.
Yes, but where? The device that I'm not able to use has a C-Media Electronics CM108 Audio Controller 0d8c:013c. Google that and you get carefully worded instructions on configuring sound output on your Raspberry Pi, which, apparently, has one soldered on the main motherboard. When not there, the chip seems to be mostly used for USB 5.1 and 7.1 audio soundcards. And, everything I find there, says to use ALSA at which point the Ourobourous known as Skype bites its tail.
