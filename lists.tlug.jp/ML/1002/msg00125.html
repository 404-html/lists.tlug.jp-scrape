Hi,

> One question.  If a person hits reply to an email, and then changes
> the subject completely and changes the body of the message
> completely (empties it and starts over) is there any reason that an
> email client should put that message into anything other than a new
> thread?

Yes, the mail user agent (Gmail web interface, for you) can add some
headers (mail metadata) indicating the this message is a response to
another message. These headers can be 

* an explicit "In-Reply-To:" header, with the message-id of the
  previous message
* a "References:" header, with a list of previous message -ids of the
  thread
