On Thu, Feb 04, 2010 at 01:50:00PM +0900, Dave M G wrote:
> The only thing that's a hint of a problem is that if I type in
> http://mysite2.com, it changes to http://www.mysite2.com, whereas my
> first domain doesn't ever seem to need the "www" subdomain.

That's a hint of the fact that Apache is not properly configured for
replying to requests for mysite2.com, and/or mysite2.com does not
resolve to an IP address.
Failing to connect to the web server at mysite2.com for the reasons
outlined above, Firefox tries to prepend "www." hoping for better luck,
and tries again.

Hope this helps a little,
 David
