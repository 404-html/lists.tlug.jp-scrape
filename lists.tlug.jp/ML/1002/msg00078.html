Sach Jobb さんは書きました:
>> BTW, how quickly is "fairly quickly" ? A minute, 5 s, 1 s, less?
> 
> Short enough so that it doesn't appear broken to anyone. They refresh
> it a few times, and "oh, yeah, there it is." Probably anything under
> 10 sec is OK.

Presumably the server which the user is checking for the new content is
also local (UK, NY)? If not there's no way around the fact that you have
to get all the data to Tokyo, and you might as well just upload straight
to a master server there in the first place. You could have the user
upload to a forwarding server locally so that the transfer won't seem slow.

If the data is going to be accessed from servers in the same place as
the user, maybe a periodic rsync wouldn't be too bad? (So long as you
can handle deletion yourself).
