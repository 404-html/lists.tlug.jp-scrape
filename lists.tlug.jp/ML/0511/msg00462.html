I'm thinking about adding native (as opposed to "comes with GTK" or
the like) support for the anthy/uim input methods to XEmacs.  Has
anybody written software and explicitly integrated anthy support?

Did the API look "SWIGgable"?  Of course I won't be using SWIG since
SWIG doesn't support Emacs Lisp, but basically the idea is to wrap the
library and let people write their own interfaces in Emacs Lisp,
rather than force a particular interface on them.
