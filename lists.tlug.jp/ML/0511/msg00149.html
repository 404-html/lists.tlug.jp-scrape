    Topics: 1) Subversion

I'd like to hear this talk but I'm not sure I'm going to be able to make it this weekend.  However, if anyone would care to wind Josh up:

1. merging worse in SVN compared to CVS.  In particular, you have to find the correct ancestor svn revision to merge from, a process that is automatically best-guessed in CVS and is correct 99% of the time.

2. When you want a diff to ignore the linebreak/formatting changes that some muppet committed: cvs diff -uBw just works, svn diff doesn't support the necessary options.

I actually quite like the tool, but there are places where it has yet to catch CVS up.
