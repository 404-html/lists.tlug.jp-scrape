This is not necessarily a bug. Though autoconf's AC_TRY_RUN macro has an
option for cross-compilation, there are other cases which cannot be
solved with this.
For example the one I encountered when creating an ipkg for anthy was
that a binary is compiled (from C) and executed to generate a data file.
In the case of cross compilation, this cross compiled generator will
obviously fail to run on the host system. The OpenEmbedded system has
solutions for this, namely native builds.
