On 27/01/06, Edward Middleton <edward@example.com> wrote:

> I was hoping to cross compile everything, but a lot of packages have tests in their
> configure scripts that try to execute the file even when cross compiling which obviously
> won't work.

Which certainly seems like a bug to me. Please submit bug entries for
all packages that you encounter that are b0rken in this way (attached
fixed ebuilds would be even better!), and I will make sure that all of
the fixes find their way into CVS.

To submit an ebuild-related bug:

http://bugs.gentoo.org/enter_bug.cgi?product=Gentoo%20Linux

(I'm sure you know this, Edward--it is just for the sake of
list-members and furture archive searchers who might not.)

Also, you can save a submission as a template--by clicking on the
"Remember values as a bookmarkable template" button instead of the
"Commit" one--which can really speed up batch submissions.

I have a new ebuild submission template here:

http://bugs.gentoo.org/enter_bug.cgi?product=Gentoo%20Linux&version=unspecified&component=Ebuilds&rep_platform=All&op_sys=All&priority=P2&bug_severity=enhancement&bug_status=NEW&assigned_to=&cc=&bug_file_loc=%24%7BHOMEPAGE%7D&short_desc=%24%7BCATEGORY%7D%2F%24%7BP%7D.ebuild%20%28New%20Package%29&comment=Find%20attached%20an%20ebuild%20for%20%24%7BP%7D%2C%20%24%7BDESCRIPTION%7D.%20It%20depends%20on%20%24%7BDEPEND%7D.&maketemplate=Remember%20values%20as%20bookmarkable%20template&form_name=enter_bug

if anyone wants it.

And remember, TLUGgers get priority in my queue of Gentoo work, so if
you have a pet ebuild or a piece of software that you would like to
see in the official Portage, just post to the list or email me
privately!

Cheers,
Josh
