Vincent Thomas wrote:
VT> I've made entries in my pap-secrets file, tried beefing up the options
VT> in my options file, but the debug output in the system files always
VT> shows
VT> that I cannot get beyond a CONNECT - I get an "LCP:timeout sending
VT> Config-Requests" every time. (Even if I try dialing in on minicom,
VT> exit without a reset and manually start pppd)
VT> 
VT> Eventually (30secs) I get kicked out of the system...

I had a very similar experience on MN128.  The reason was that the
factory-initialized default value of the communication protocol on
MN128 was "async only".  At that time I used AT command to set "use
sync-async PPP conversion when speed is over 38400bps".  I'm not
familiar with Sony TA but you may want to double-check the default.
