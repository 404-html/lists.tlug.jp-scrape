> > > Has anyone already figured out how to get a masqueraded machine
> > > to be able join in a multi-player FS98 flight? 
> > 
> > The new Linux kernels (2.0.33 at least) have a transparent proxy
> > capability that should be able to handle it - ie, it tries to masquerade
> > everything by a general mechanism... [...]
> 
> It doesn't seem to work with FS98.  I suspect it is a protocol where
> communications on one port controls/open the real conversation on another.
> (It does work with the Squawkbox ATC add-on however.)

A bit of snooping on the LAN last night with tcpdump got to a working
(but less than optimal) solution:

ipautofw -A -r udp 47624 47624 -h xxx.xxx.xxx.xxx
ipautofw -A -r tcp 1000 1200 -c udp 47624

(where xxx.xxx.xxx.xxx is the masqueraded client's bogus address)

FS98 seems to start by sending a UDP packet to port 47624, and the
responding system responds with a TCP connection on a port in the 1000~1200
range.
