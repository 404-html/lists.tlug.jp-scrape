The problem is that FTP clients only speak FTP. If your office doesn't have a
proxy also speaking the FTP protocol on the internal network (try ftp
your-proxy, login with ftp@example.com), you need a user agent which makes
HTTP requests with FTP URLs. Lynx and Netscape come to my mind.
