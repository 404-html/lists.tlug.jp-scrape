> Scott Stone <sstone@example.com> wrote,
> 
> > Since TurboLinux uses RPM, you can install any of Redhat's contrib/updates
> > directly into TL as well, at least beginning with TL 2.0 (versions < 2.0
> > don't do glibc).
> 
> When can we expect TL 2.0?

late June, early July for the USA version.  Mid-June for the Japanese
version.  If you're living in Japan, I can put you on the list for
1.9Beta3, if you want to help test (email me privately with your full
name, postal address, and phone number).

> 
> So, the file system structure, configuration structure etc
> is exactly the same in RH and TL?

Very similar.  TurboLinux J adds /usr/jp to keep track of jp-specific
apps, but other than that... both use the SysV style filesystem/init
scripts, etc, and both use /etc/sysconfig for most system configuration
stuff.  /etc/sysconfig, I think, is Red Hat's creation, which I copied for
TurboLinux.  /usr/jp is unique to TLJ, and SysV init is.. well, SysV :)

TurboLinux is compatible with RedHat RPM packages, as well, ... both use
RPM, so...
