OK, here's what you do:

1.  Load yellowfin.c in emacs.  Emacs will ask you if you want to set the
options as specified in the end of the source file.  Say yes.

2.  Execute emacs's make command - do a 'meta-X' (usually alt-X on PC
keyboards), and then type 'make'.  It should build just fine.  It even
checks for modversions in your kernel source..

Anyway, it builds fine on my TurboLinux system.  I can insmod it, although
it says 'init. failed', since I don't have one of these cards, but no
unresolved syms.

Anyway, this is cool - a GIGABIT ethernet adapter?  Who's making these
things now?  I'm definitely including this module in TL 1.9b3/2.0, seeing
as how Donald Becker has attached an 0.99A version number to it (heck, I
run stable servers on v0.36 of his eepro100 driver..)
