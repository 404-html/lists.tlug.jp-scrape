    Scott> On Tue, 26 May 1998, Stephen J. Turnbull wrote:

Both were into the 2.0.mid-teens, I think.  However, those boot disks
are long gone.

It wasn't a kernel problem.  It was a module issue.  Neither one had a
working DEC Tulip driver, although two were supplied in the kernel
source.

Thanks to similar problems with TurboLinux, I now know how to mount an
initrd via loopback from a boot image mounted via loopback, add
arbitrary modules to the root FS on the ramdisk, and create a new boot
disk.  (This is necessary because a single-floppy install means you
have a completely crippled system until the install works.)  The
Debian approach made life a lot simpler: just build a kernel with the
appropriate driver and sneakernet it over to the new machine, boot,
and run dselect.
