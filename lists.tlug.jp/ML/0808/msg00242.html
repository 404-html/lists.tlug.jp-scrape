To make it short:

What I will do and need:

1. Some HDD spaces( 1GB?)
2. I will install gcc 4.2 and Graphics Magick 1.3.
3. I will run GM 1.3 openMP test scripts.
4. Nothing else.

Subject: [GM-core] Need Linux volunteer for OpenMP testing
To: GraphicsMagick Core <graphicsmagick-core@example.com>, GraphicsMagick Help <graphicsmagick-help@example.com>


I need a volunteer with a recent Linux 2.6.X system with at least GCC
4.2 and multi-core hardware (e.g. Intel Core 2 Duo or a womping 16
core system) to assist with doing performance testing of
OpenMP-accellerated algorithms in development GraphicsMagick 1.3. The
task is not too challenging since all performance testing can be done
from the command line.  Programming skills are not required.

OpenMP (parallel processing) is starting to be rolled into
GraphicsMagick.  Performance is sensitive to OS, compiler, and CPU
type.  For some targets performance actually gets worse with more
threads whereas other targets see substantial speedup.  My objective
is to obtain as much speedup as possible on various systems without
experiencing loss of processing efficiency.

Can someone please volunteer for this?

Thanks,
