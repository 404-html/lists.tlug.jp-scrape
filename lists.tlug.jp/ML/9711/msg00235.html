send us your resolv.conf and the output from route -n after the PPP

    Michael> 3) My etc/resolv.conf has the name of my domain and
    Michael> nameserver of my ISP...  one question though... does
    Michael> anyone know if I should use "tky0.attnet.or.jp" or
    Michael> "attnet.or.jp"?

The nameserver should be numeric IP.  Otherwise you need the
nameserver to find the nameserver.  Oops.  (I think there's a way
around that but if you can't find it for yourself, better to do it
right inthe first place. ;-)  resolv.conf doesn't have a domain name
in it; I suppose you're referring to the "search" path?  These are
simply the default names to tack on the end and retry if the first try 
doesn't work.   Ie, if `foo' doesn't get resolved, the resolver tries
`foo.attnet.or.jp' next.  this is useful if you want to refer to
www.attnet.or.jp as simply "www"; however in principle you could put
"twics.com" in there and "www" should get resolved to "www.twics.com".

able to set up a private network using proxy ARP.
