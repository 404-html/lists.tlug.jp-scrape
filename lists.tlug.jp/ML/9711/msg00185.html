On Sat, 22 Nov 1997, Jason Molenda wrote:

> You can probably hack around this by looking at the declaration of
> sys_errlist in os.c.  Removing it would be easiest.

Yep, remove it.  IIRC, that snippet of code looks something like

#if !defined(FreeBSD) && !defined(NetBSD)
extern char *sys_errlist[];
#endif

... glibc wants that to be const char, of course.  The Right Thing To
Do(tm) is to add a !defined(Linux) to that #if block, but the pine
developers don't seem to be too receptive to non-UW coding suggestions.
