I think you're right, but to be honest, I don't follow the
FSF releases very closely, particularly gcc.  

In general, if you see a compiler abort message, there is a compiler
bug.  You should never see this message, and there should be no source
code, illegal or legal, which can cause it.

The FSF has been talking about releasing GCC 2.8 for quite a time now
(since August or September).  The EGCS project has finally started
doing their first release--that is going to be the best thing to use
once it's out.  (there are development snapshots out right now).  I
don't think the first EGCS release could possible get out any faster than
a month from now because of the necessary testing. http://www.cygnus.com/egcs/

Did people hear that the ANSI C++ committee has finally finalized the
technical parts of the standard?  There was a note from Strousup (sp?)
that went out earlier this week making the announcement.  After many
years of changes and dinking around, it is a great thing that this will
finally stop moving.
