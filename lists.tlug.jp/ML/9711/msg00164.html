Well, to cut right to the chase, I am writing this on a newly compiled
pine 3.96 with the Japanese patch.  Yeah.  

I had no problems.  I did the following:

/% tar xzvf  pine3.96.tar.gz
/% cd pine3.96
/pine3.96/% tar xzvf pine3.96lj1.1b7.tar.gz
/pine3.96/% patch -p1 <pine3.96lj1.1b7.diff

At this point I read the README file in /pine3.96 and it tells you
to make all kinds of changes to the make files pine/makefile.lnx
and pico/makefile.lnx.  I when I went to make these changes, I found
everything was already done as I was being instructed to do, so
for my case, I didn't need to make any changes, just compile with
the script that pine provides.  This script is located in the 
/pine3.96 directory.  It is call build and you just need to supply the
three letter code for your machine.  Linux is lnx.  So I typed:

/pine3.96/% build lnx

After about 5 minutes, I have the pine what I am now writing with
that can do 日本語 without invoking the alternate editor and 
so far (crossing my fingers) no 文字化け.
