I have ghostview 3.58 that I installed from PJE-0.1beta (the new Japanese packages).
This is the only program for which I can not see the Japanese correctly.  If I have
a tex file, I can see the Japanese correctly as I write it in mule or vi on kterm.
When I tex it, I can see the Japanese correctly in the .dvi file with xdvi.  When
I convert the .dvi file to a postscript file with dvi2ps, I can not see the Japanese
correctly with gv (ghostview) or gs (ghostscript).  When I print it out, it looks
fine, Japanese okay.  Now also, when one of my students rlogins in to my machine and
uses my gv or gs, but displaying on his machine, he can see the Japanese.  When he
logs in directly to my machine, he can not.  I must then assume the problem will
be somewhere in how I display Japanese, but again on any other program, I can see
Japanese with no problem(this includes xdvi, kterm, mule, netscape, etc).

Does anyone have an idea of where I should look to determine what is wrong here?
