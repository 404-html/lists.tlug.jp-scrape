Stephen J. Turnbull writes:
 > I noticed this in the Debian package list:
 > 
 > jadetex - LaTeX macros for SGML to DVI/PS conversion with Jade
 > 
 > jadetex provides a set of LaTeX macros to process the TeX files generated  
 > by jade. Using jade and jadetex you can print every SGML document for which
 > you have/make a DSSSL description.
 > 
 > Ie, in fact jade does not print anything, but uses TeX (possibly among 
 > other back ends) to do its printing.

Right. Although the jade distribution includes tools that run the
process from SGML to printable form (if there are no errors -- good
luck!), so that if you are content with the defaults you can do
something like 

$ jadesomething somedoc.sgml |lpr

and get a nicely formatted hardcopy.

Last I knew, the backends included TeX, RTF, HTML, PDF (via TeX),
ASCII, and SGML (i.e., transforming one doctype to another).

My info is out-of-date, but I'm planning to upgrade to the latest
version of Jade within the next few days, so if anyone's interested,
please e-mail me privately and I'll let you know what I come up with.
