On Sun, Aug 09, 1998 at 07:38:34PM +0900, Yuval wrote:
> and the MO is recognized, with a caveat, however, it will mount only
>  230MB MO's at /dev/sda (I am catching up now, Thanks Jim).
> I can do mke2fs on it, and it also reads vfat / msdos.

You should be able to use either 230MB or 540MB media, since they both
use 512 byte sectors.  (You will need to patch the 2.0.x kernel or move
to 2.1.x to use 640MB media.)

There are at least three media layouts used:

1. You can treat the whole disk like a floppy (the Fujitsu documentation
refers to this as SuperFloppy mode), in which case there is no partition
map/no partitions and you refer to the whole disk as /dev/sda/.

2. You can partition the disks with FDISK and mount the individual partitions
as you would with a hard disk.

3. The Fujitsu MO DOS-based format software appears to support another
variation on partitioning which does not seem to be compatible with the
standard DOS/Linux scheme.  It can be safely ignored.  :-)
