> Naru hodo, thanks a lot!!! BTW, when I just edited fstab for mounting the
> Windows partition under Linux I set the last number to 0, now I am wondering
> whether I better set it to 2?

The sixth ("last") field in each fstab entry controls the order the
filesystems are fscked during boot time.  For your Windows partition,
0 is appropriate, because it signals that partition does not need to
be checked.
