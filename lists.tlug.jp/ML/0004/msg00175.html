We're experiencing an odd thing with trying to build a 2.2.14-RAID kernel
on TurboLinux.  The same thing happens on both TL PRo 4.2 and 
TL Server 1.0 -based systems.

These examples are from a TL 4.2 Pro system:

If you do make menuconfig, you get:

[root@example.com linux-2.2.14-RAID]# make menuconfig
rm -f include/asm
( cd include ; ln -sf asm-i386 asm)
make -C scripts/lxdialog all
make[1]: Entering directory `/usr/src/linux-2.2.14-RAID/scripts/lxdialog'
make[1]: Leaving directory `/usr/src/linux-2.2.14-RAID/scripts/lxdialog'
/bin/sh scripts/Menuconfig arch/i386/config.in
Using defaults found in .config
Preparing scripts: functions, parsing......................................done.

And it just stops there and sits until you do a control-c.  Then you get:

make: *** [menuconfig] Error 1


If you do make depend, you get:

[root@example.com linux-2.2.14-RAID]# make depend
make[1]: Entering directory `/usr/src/linux-2.2.14-RAID/arch/i386/boot'
make[1]: Nothing to be done for `dep'.
make[1]: Leaving directory `/usr/src/linux-2.2.14-RAID/arch/i386/boot'
scripts/mkdep init/*.c > .depend
make: *** [dep-files] Error 139

But -- make xconfig works.

Does anybody know:

1) What causes this;
2) A workaround;
3) What error 139 is?

Thanks,

Jonathan
