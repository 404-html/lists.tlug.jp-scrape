There's a library here that would appear to do the business:
http://xmlresume.sourceforge.net/

I use this tool. It is even included in some distributions.
When I did research on this some years back, I found this one and the HR-XML which was mentioned previously in this thread.
xmlresume works pretty good, comes with tools to generate txt, html and pdf versions from the base xml.

I tried to use HR-XML but it was pretty large and complex, and no transformation tools were available at the time. This might have changed since then.

I'd have thought it would only take a few large employers of technical people - say Amazon, Google and Microsoft - to kick-start adoption of a standard like that by saying they like to receive resumes in that format.

That was my thought to begin with as well, so I decided I would just keep an XML file resume and update that, and use the tools to provide whatever format I was asked for. So far though I have not come across any job agency or employer that supports a direct import of these formats, even though I would love to have it. Typing ones details here and there, again and again is a real PIA.
