Well, a good point of CGIWrap is that users set-up their own 
personal cgi-bin directory so the script alias remains:

ScriptAlias /cgi-bin/ /usr/local/etc/httpd/cgi-bin/

and cgiwrap* stays in that directory. The cgi script doesn't get 
called by a server derived userid but by the user's userid. 

I kept playing around with it and still didn't get it to work so I 
put the links.pl in /usr/local/etc/httpd/cgi-bin/ and hacked it to 
it's new location and changed the links.html file to reflect the 
'normal' action and I'm right back to "The server made a boo boo"
