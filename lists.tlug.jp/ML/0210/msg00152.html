hatsuhiro wrote:

>     sox: Unable to sync dsp
> --------------------------------------------------------------------
> What do those SoX error messages mean? What can we conclude from the
> messages regarding my system? Since I believe the on-board VT82C686B (VIA
> PLE133T chipset) is working, I guess the VIA 82C686 Audio Codec driver
> doesn't have suitable sampling capability. How do you think?

I think this implies that sox is using mmap to map the cards buffer into
main memory and something is not working right, maybe the driver does
not implement this properly.
You might want to try the alsa drivers instead (from
http://www.alsa-project.org/). If you use Debian you can
apt-get install alsa-utils alsa-source
then go to /usr/src/modules/alsa-driver and
./configure
make && make install
