  • Organization: MMDC.net

I recently noticed that my DSL connection through Jens/Spinnet/Att/whatever started allowing mail servers, so just for fun, I set up a mail server in my apartment to handle mail for my domain.  (Don't worry, I *do* know how to prevent open relays...)
What had been doing was to configure my work mail server to forward anything (at) mmdc.net to my regular ATT account.  (I've since gone back to doing it that way.))
It worked reasonably well, except that before, I got no spam at all - after, I was getting literally dozens of messages a day.  
What I figure is that spamming software is not aware of MX records and just tries to connect to a mail server on the same IP address as a web server that it knows about.

Anyone else noticed this?  I thought it was kind of interesting...
