Try looking for the hwclock command in the startup scripts, which
AFAICG (*) is responsible for (A) setting the system clock initially
from the hardware clock and (B) setting the kernel timezone which is
responsible for the time offset for the fat filesystem (FAT stores
timestamps in local time and not UTC). Since (A) and (B) are both wrong
on your system, wrong environment/parameters to hwclock would seem to be
the likely culpit.

If you can't find where it is called, check the initial RAM disk as
well if you use one.

(*) as far as I can google
