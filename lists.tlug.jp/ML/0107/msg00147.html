Actually, my point was that while it tells you about dependencies it doesn't
actively download and install the needed packages automatically, which apt
does.  NetBSD pkgsrc does something very simillar to this. I believe Ports 
does also.  With Redhat if you don't already have a package you have to
download it by hand and install it manually. There is no automated conflict/
dependency resolution built in.
