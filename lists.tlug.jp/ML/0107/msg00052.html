OK, rather than your current rule for port 119 in 
/etc/sysconfig/ipchains, try this one:

-A input -s 0/0 -d 0/0 119 -p tcp -y -j ACCEPT

That says "Allow all connections on port 119."

Then restart ipchains and try to connect.  If you
still get a connection refused, then either:

1) As Tom suggested, you don't actually have a news server
   listening on port 119, or

2) If you do, it's configuration file is not set up to allow
   connections either from your localhost, or maybe just from
   anywhere at all.

If, on the other hand, this tweak to ipchains helps, then
you know the rule was the problem.
