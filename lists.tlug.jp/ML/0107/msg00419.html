On Mon, Jul 23, 2001 at 08:40:07PM +0900, A.Sajjad Zaidi wrote:
> I have similar problems with all of my machines. Its not that bad so I never
> really tried to fix it. It seems that whenever I reboot or unload the sound
> modules, the permissions on both /dev/dsp and /dev/mixer change (to 600).
> 
> The good news is that the owner 'usually' stays the same (me), but I have to
> manually set the permissions if another user needs to use it.
> 
> Strange.
> 
> 41 53 5A
> 
That`s not a problem, it`s a feature. In RedHat there is this thing called 'console access'. A pam module checks if you are logged on a physical console and if you are gives you rights you normally wouldn`t have- ownership of devices, ability to reboot the machine, etc. The user that logs in first gets to own the device files. Permissions are defined in /etc/security/console.perms. Look up pam_console(8) and console.perms(5) for more info.
