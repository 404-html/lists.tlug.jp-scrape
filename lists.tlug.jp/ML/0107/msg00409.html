scottro@example.com wrote:

> In /etc/sysconfig/i18n it has the lines
> SYSFONT="lat0-16"
> SYSFONTTACM="iso01"

There's nothing wrong with that.
It's not possible to display Japanese on a plain text console because of
standard PC Hardware limitations.
It would be possible in a framebuffer console but the linux kernel does 
not support that (yet).

> However, when I looked in /lib/kbd/consolefonts I found no Japanese fonts listed, though I did see what I assume to be russian, etc.

Because of the above situation a consolefont cannot contain more than
512 characters, which is obviously not enough for asian languages.

However there are svga/fb terminal emulators which you could use
(kon2, jfbterm). These should come with there own set of fonts,
use X11 fonts or both.

> I can work around it by starting X and running lynx or whatever in a kterm, but I'd like to be able to solve it without starting X.

You have to install and use kon and/or jfbterm.
