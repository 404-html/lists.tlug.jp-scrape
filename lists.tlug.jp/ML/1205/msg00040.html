On Sun, May 13, 2012 at 12:31:57PM +0200, Michal Hajek wrote:
> 
> we run an old Dell PowerEdge server with gentoo in our department as not so 
> important file server. It has two LSI MegaRAID controllers - 
> lspci -nn
> 
> 09:01.0 RAID bus controller [0104]: LSI Logic / Symbios Logic MegaRAID [1000:1960] (rev 01)
> 18:01.0 RAID bus controller [0104]: LSI Logic / Symbios Logic MegaRAID [1000:1960] (rev 01)
> 
> When I installed the system several years ago, just for fun and I managed 
> to put storage together in a raid (quite easy with this type of hw raid) 
> and manage disks with a "megamgr" utility. Thus all the time I have had two 
> raids, one on each adapter. One of the raids is smaller since the 
> machine has one disk slot free.  

I think for just a few hardware raids utilities exist to configure 
them.  For a slightly higher number one can atleast monitor the health
status from Linux with opensource software.

You should be able to access the raid configuration utility directly 
after restarting the system bevore Linux starts.
You seem to not reconfigure the raid very often, so having to reboot
to do that should be ok.
