> This may seem like a newbie answer but:
> ctrl-alt-<virtual console> doesn't work for you?

Weird, I thought I already wrote an answer to this one...
OK, so here I go again:

ctrl-alt-Fn works as long as the X Server is running.
But if the X-Server segfaults without switching back
to textmode not only is the keyboard usually stuck in
raw mode (which can be switched back using a ssh connection
from a different machine (My notebook) using kbd_mode -a).
After switching back to cooked mode I _can_ switch
virtual consoles but still see only the remains of my
X Desktop on the monitor.
I can however restart the server by just typing 
startx and hitting return...
