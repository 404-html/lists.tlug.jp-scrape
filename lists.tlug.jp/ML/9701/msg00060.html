On Thu, 23 Jan 1997, Stephen J. Turnbull wrote:

turnbull>uses XIM as kinput2 is very nice (Craig, have you noticed that you can
turnbull>select from the kanji window using the mouse as well?  If you convert
turnbull>"kan" or "shin" this can be worthwhile!) and would allow me to use the

Stephen,
I did play around with kinput2's ability to select kanji with the
mouse.  This is nice.  However, I've mutated to a keyboard existence
so much, that I don't even like to move my hards to press the arrow keys
and the escape key is pushing it.  A mouse is a bit tedious and I usually
use a trackpad which is located between the two halves of my keyboard.  I
will use a mouse when I need to.  However, one of the main reasons I like
to use mule is to avoid having to take my fingers off of the main key row. 
I have the esc key mapped to the left alt key.

I do like kinput2's ability to show a large number of kanji on the screen
at the same time.  Using Wnn or Canna, I just scroll through the
mini-buffer one line at a time.  With kinput2, I have several rows on the
screen at once.  This is why I would like to use kinput2 with mule.
However, I haven't been able to get this to work properly.  I think
Alberto mentioned that he was using it this way.  I also think that he was
using the standard emacs and not the patched version.  Though, he would
have to confirm this.

Regards,
Craig
