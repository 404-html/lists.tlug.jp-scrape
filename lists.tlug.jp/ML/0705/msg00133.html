I see what you mean, but I also think you're trying to go against the
Unix philosophy of one application doing only one thing, but doing it
well. mplayer is a media player. Period. It plays back media and it
does it well. If you want something else to *manage* the media into
playlists rather than just play it back, then you want another
application that can, however, feed the playlists it generates to
mplayer.
