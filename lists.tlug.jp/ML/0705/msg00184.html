Charles Muller wrote:
I had a power outage while downloading to an external usb drive which was formatted fat32. After this I had problems with file access on the drive. After reformatting it, it apparently works OK in Windows, but Linux (Debian testing 2.6.18-4) won't have anything to do with it.

During bootup:

buffer I/O error on device sdb logical block 29305205


I'd start by running "badblocks" on it. At least som mk*fs'es support badblocklisting so that you can make sure the new filesystem avoids any blocks that are bad.

I'd also be a bit cautious with that drive, it might be failing.
