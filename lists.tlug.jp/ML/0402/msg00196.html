> Not so fast.  The input methods matter, here.  The problem is that in
> Xt apps XIM works by filtering _all_ key events before the app (and
> that mean functions like XmbLookupString and the XKeysymTo* family)
> sees them; if the IM throws away key events, such as the compose key,
> you're hosed.  kinput2 is notoriously evil in this respect, and I had
> problems with xwnmo, too.
> 
> I don't know if GTK is any better, you'd hope so with the benefit of
> Xt's bad experience.

That's just another reason GTK is better, then. =D Try im-ja or uim. As 
a bonus you'll be able to switch input methods in gnome-terminal.
