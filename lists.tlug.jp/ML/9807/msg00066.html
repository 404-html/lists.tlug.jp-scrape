>From the RH List: 
=======================================================================

Folks, it looks like we made a pretty major mistake with the 5.1 CD
images that are currently shipping.  We accidentally used an SMP kernel
for the only kernel image on that disk.  It will work fine on sun4m
machines, but it will not work at all on sun4c machines (SS2, IPC, IPX,
etc).

It being the weekend (and a holiday one at that), I don't have any idea
what the official fix for this will be.  We will make an announcement
on Monday, however, of that fix.  For now the *only* alternative is to
boot via the network using the tftpboot.img and the instructions in
the Red Hat Installation Guide on www.redhat.com.  (There is also
the undocumented option of booting the tftpboot.img from a tape...I
posted instructions on this a day or two ago.)  Sun4m users should 
still be able to boot fine from the CD and will get a uniprocessor
kernel installed.

I apologize for this problem.  Believe me, it's a pretty major 
embarrassment.
