OK, I'm making a new netscape-communicator package which fixes java.
Basically it adds another script into the /etc/profile.d directory.  All
scripts in that dir are run at login time.  BAsically the netscape script
just sets the CLASSPATH env variable.


Side note - if guavac's classes.zip is specified in CLASSPATH *before*
Netscape's java stuff, Netscape will generate a bus error on ALL java
pages.... so my script makes sure that netscape's is first.  But any of
you who are having bus error problems with Java in Netscape, you might
want to check that out.
