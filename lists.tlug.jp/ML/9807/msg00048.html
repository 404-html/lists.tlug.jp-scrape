For those who have reason to use the FSF's Emacs rather than XEmacs,
but would like to use XIM (eg, kinput2), it apparently is possible
(according to Handa himself).

I don't understand very well, and XEmacs is not synched with this part
of the FSF's Emacs so I can't test it at the moment, but it sounds
like you just set up XIM like you would with kterm or the like, and
then select the `encoded-keyboard' (or something like that) input
method in Emacs in the usual way (C-x C-m C-\).

It is apparently buggy, possibly crashing Emacs and/or the IM in some
environments.

More later.
