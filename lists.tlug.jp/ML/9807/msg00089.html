All,

If you're thinking about dropping RH 5.1 onto a sun4c machine (IPC, SS2,
SS1, several others) -- check out which SCSI controller you have _before_
you install.

If you have a NCR53C90A, you're okay -- it'll give a couple of errors at
boot time, but it'll proceed.  

If you have a NCR53C90, you're screwed -- it'll die while it attempts to
determine the sector size of your first hard drive.

Rumor has it that kernels compiled with egcs trigger this behavior.  A
kernel compiled with gcc-2.7.2.[23] from RH 4.2 doesn't break.  Guess
which one RH 5.1 was compiled with?

It is unknown if the RedHat folks are working towards a solution.  I
believe that it's a bug in the kernel that's been exposed by egcs.  I'll
be able to verify whether using the older gcc fixes the problem by Friday
(heavy work load, unfortunately).
