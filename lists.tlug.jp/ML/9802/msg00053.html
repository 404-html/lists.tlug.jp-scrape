Yay!

It looks like my problems with XEmacs on Sparc/Linux have been
solved.  There is a serious memory corruption problem in the dump
stage (where the .el files get preloaded into the distribution binary) 
which potentially affects all ELF systems, and definitely manifests
itself on Linux.

Apparently it's horrible on Sparc/Linux because the Sparc is extremely
alignment sensitive (thanks for your wisdom, John Little!), but
(fortunately very rarely) manifests itself on Linux/i836 too.

Upgrade to XEmacs 20.4 when it becomes available is therefore _highly_ 
recommended.  This is the bug that has been holding up the 20.4 bugfix 
release, so 20.4 should be available within a week or two.

Also included will be fixes for a large number of small annoyances.
