TurboLinux includes Python.  Personally, I'd much rather use perl.  Python
is designed for GUI stuff and isn't as efficient as perl, IMHO, for cgi
stuff.  Although, apache will run *any* executable program as a cgi-script
under Linux.  It just has to send the proper content-type strings and
stuff as output so that the browser sends the output as html instead of
text.

Anyway, if we use TL 1.2 on the TLUG server (as we should! :) ), then I
can use perl, you can use python, someone else can use C, C++, whatever,
and we'll all be happy, I hope. :)
