In message <004401bd39d6$18142580$18d8ebca@example.com> you wrote:
> Can a FAT-16 file system on a Windows 95 machine be mounted as an NFS by
> Linux?  I'm also planning on migrating the Windows box to NT Workstation;
> can Linux mount an NTFS file system as NFS? What about FAT-32?

You can export a DOS partition using PC-NFS, and at the Linux side compile
the kernel with the option to support this filesystem. It worked for me
with Win 3.1, should work with Win95, but one drawback is that long filenames
will probably be lost (i.e., it's for plain FAT-16, not VFAT or FAT-32).

> Or is there a better way to move files back and forth between a Windows
> machine and a Linux machine on a TCP/IP network?

Samba? FTP?
