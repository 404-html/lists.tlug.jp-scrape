Just a recap of the issues on the server and domain.

Which Machine: John Little's generous donation of the 486
   Tower.

When: Probably first week of March

Transport Method: Takyuubin, TLUG will pay from collected funds

Place: Pacific HiTech, makers of TurboLinux,  is generously 
   donating bandwidth and space

Principle System Admin:   Scott Stone, developer of TurboLinux

Other Admin Policy: not decided, but I sure want shell access.   ;-)

           *******  Domain Issues *********

  -----  We need feedback on domain issues  ------------
  ------ write your comments to the list    ------------

 tlug.pht.co.jp
   benefits: Scott can set it up rapidly and maintain
   negative: TLUG becomes dependent on the pht DNS
             tlug.or.jp is more kakko ii.  "cool"

 tlug.ml.org
   benefits: we can use our own DNS and control the domain with
             TLUGers
   negative: not identified as being in Japan.  The ml is kind of
             distracting.

 tlug.org
   this is taken by the Toronto LUG

 tokyolug.org
   benefits: a bit cheaper than jpnic
   negative: again, that .or.jp status is nice

 tlug.or.jp
   benefits: identifiable as in Japan and as a non-profit
   negative: this is going to be the most time-consuming and expensive
             approach.  In addition to registration and routing charges,
             we have to prepare documentation for jpnic at time of
             registration


 --------------- Craig's personal comments below -----

My personal thought is to go with tlug.ml.org for the initial
run next week because it gives us the opportunity to control our
own DNS.  Also, the ml project is pretty cool.   I think we
should try to shoot for the tlug.or.jp over a period of time.
This will force us to write an organizational constitution and
stuff, but the name is pretty cool.  Right up there with 
jlug.or.jp.  From what I understand, the Japan Linux Users
Group took a while to register the jlug.or.jp domain. 


Regards,
