I was poking thru our 'extra hardware' in the back of the office and
stumbled upon a Quantum Fireball 4.3GB EIDE drive that's not being used.
I don't' know if it works or not, but I'll test it.  If it works, I'll try
and get Cliff to allow me to put it in the TLUG server.  I'd want to maybe
partition it into two, or something, and have it NFS mountable by my
machine so I can also use it for some devel space, but that shouldn't
affect the server at all.  But, hey, I think we might have a solution to
this hardware problem.

Just remember how nice I/PHT is being next time you all want to buy a new
Linux distribution :)

Speaking of which, I'd really like to install TurboLinux on the TLUG
server, for the publicity and for the 'real world testing' that it would
imply.  There'd still be plenty of config/setup to do.  You could either
use the TurboLinux config utilities or do it by hand.

Anyway, let me know.  The upshot is, though, that we'd need a motherboard
that's capable of addressing an IDE drive of that size - some of the older
486 boards can't do it.  Preferably the BIOS should also support LBA, so
we can get LILO working on it as well.
