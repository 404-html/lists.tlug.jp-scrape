Craig Oda asked:
> What does everyone think of tlug.ml.org as the domain?
> 
It has actually a nice ring to it - "tlug.ml" could even be
retconned to mean "Tokyo Linux User's Group Mailing
List". And since it doesn't actually cost anything, I'm all 
for it.

> We could slice off a chunk of the ml.org domain
> and control the tlug.ml.org section from our own DNS running
> on the TLUG server.  This way, we don't have to be dependent
> on the pht.co.jp DNS and can just rip in there ourselves.
> 
Like I said, I can host a secondary DNS on ns.inovis.de 
without a problem. All that would have to be done is to
declare that server a NS for tlug.ml.org in the zone file,
and I would have to know the IP of the primary server
so that our server can pull the zone from there.

> It could be fun to run the DNS.  I scanned through the FAQ and
> the first step is to set up the DNS on the server, then 
> apply with NIC.
> 
The depressing thing about BIND is that it works so well
once you've set it up :-) No work left for the hacker except
criticising the setup of others ;-)

Bye,
Thomas
