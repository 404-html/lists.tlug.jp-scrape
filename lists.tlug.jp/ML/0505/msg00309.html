I don't know how many people might be affected by this, but as FreeBSD
becomes more popular (trendy?) it might affect some people.

There has been a change in LC_CTYPE.  If you upgrade to CURRENT, and are
using, for example, kinput2 and canna or anything that requires setting
the locale to Japanese, you might run into problems. 

The suggested workaround is to copy your /usr/share/locale before
upgrading, then set the PATH_LOCALE environment variable to point to the
old locale directory. 

A riskier fix (though I didn't see it break anything) is to add to
/etc/libmap.conf (the file may have to be created)
libc.so.5	libc.so.6

That solution has the potential, however, to break other things, and I
just mention it because it's the quickest and easiest.    

The most tedious one is to reinstall all programs that use libc.so.5,
starting with perl.  The easiest way to do this is with portupgrade -f. 

On a fresh install, that is, where I put in 5.4 from the CD, and
immediately upgraded via cvsup to CURRENT, then did portupgrade -f perl,
and only then installed X and the various Japanese programs I would
want, I didn't run into this problem. So I assume it will be fixed by
itself in the near future, or at least when CURRENT becomes STABLE>

Again, I don't know how many FreeBSD users there are here, and out of
them, how many are planning an upgrade to CURRENT, but thought I'd toss
the info out on this list.
