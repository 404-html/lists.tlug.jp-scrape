On this level network-printing is "just an other ip-based service".
You could check from the lan if you can establish a tcp-connection to
port 515 on the networkprinter using 'telnet ip 515', making sure
the most common service is available. 
The same command should work from the XP home, and you should be able
to setup a network-printer on the XP home much as you would do on a
XP home on a standalone computer.
