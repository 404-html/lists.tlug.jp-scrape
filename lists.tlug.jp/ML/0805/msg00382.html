> Is that my provider blocks port 25 and I can't send outgiong emails?

It's possible.  You can get postfix to use another port.  (I'm sorry,
I'm too sleepy to doublecheck where you do that, but I'm sure it's a FAQ
somewhere) to try to check it.  Otherwise, the most likely culprit is
DNS.  

Another possibility is that if you don't have true reverse DNS working,
it won't go--in such cases, the best thing to do is use your ISP's smtp
server as relayhost.  (Some providers don't allow this, but in my case,
with Roadrunner, I am able to do it.  On the other hand, if I remember
correctly, you get a different sort of error in that case, so I'd put
that low on the list of possibilities.)

You can increase debugging as well, which is sometimes helpful.

(It's after midnight here, which is one reason I'm giving such
superficial answers--however, all of the possible problems I've
mentioned *should* be easily found by googling, along with possible
remedies. If nothing else, hopefully, I've given you some ideas of where
to start looking.)

These things can be aggravating--half the time, it's one line in main.cf
and it takes an hour of googling to find it.
