Japan Linuxers,

I've got an *old* (1999ish?) Corega PCC-T 10MB PCMCIA ethernet card that
is not correctly recognized by SUSE 10.1 or Edubuntu 6.10. In both cases
it is recognized as a wifi card, and of course cannot be properly
configured. I found a potential solution to this by googling, but it
turns out that the solution is relevant to the old way PCMCIA was
handled in Linux. Now, apparently, it's all in the kernel and very
different.

Has anyone seen this sort of problem, and know what the solution might
be?  I've already consulted some SUSE forums and wasn't able to be
helped there.  I was asked for the output of lspci -v, so since I had
already prepared it, I'll attach it here, along with ifconfig -a.

If anyone has any ideas, I'd be grateful for the help.

Dave
