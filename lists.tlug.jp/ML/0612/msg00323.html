I think you're missing the point - software RAID is tried and tested,
with widespread use, and suits the problem at hand. It is not something
you need to "cook up": someone else has already invented it for you.

The point is that from either reliability or cost points of view there
is little to recommend hardware RAID over software RAID. The performance
point of view sounds more complex; see the URL posted earlier in this
thread, but that is not going to really matter for the majority of
applications.
