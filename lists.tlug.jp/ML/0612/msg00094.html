Are you using Apache 1, Apache 2, tux or <insert name of any other web
server here>? You should really provide as much info as you can when
posting such questions....

I didn't get the Japanese contents some junk fonts coming after I done the
encoding only I am able to see the exact contents.

Please let me know how to add the charset=shift_jis font support in my Linux
server.


It may be that your html/xhtml or what ever is missing some important
information: http://www.w3.org/TR/html401/struct/dirlang.html#adef-lang

Or it could be that your web server has a default charset enabled
which *could* be messing things up. I generally do not use a default
charset on my web servers as I find it can muck things up.

For example if you are using Apache2 look for something like this
"AddDefaultCharset ISO-8859-1" and comment it out (then restart the
service).

Final point I find shift_jis to be cruddy and pretty much stick to
utf-8. You may want to look into this as an option.
