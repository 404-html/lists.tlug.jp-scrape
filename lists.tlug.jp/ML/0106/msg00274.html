Hello all. 

I am struggling with a couple of problems. 
Would appreciate your help very much. 

ppp for RH7.1 

I am installing RH7.1J(*) on the IBM Aptiva 520 (486DX) 
that is the heart of the ISSHO office (we only use the 
newest stuff). 
This machine needs to connect to GOL via PPP over a 
standard telephone line, and act as the server (192.168.1.1) 
with another Linux box and a Mac in tow. The latter two 
machines telnet in, dial the connection, download/upload 
mail and browse the web. 

The installation seems to be going pretty well, so I 
would like to stick with this distribution, rather than 
try something, if possible. 

I had been running Turbolinux 4.0 with a 2.2.11 kernel 
and ppp-2.3.7-1 . RH7.1 out of the box does not have 
ppp in the kernel but the module loads. 

My problem: 
I am having problems making the ppp connect. 
I do not have X installed on the server and do 
not intend to install it (can't find the right 
combination for the display, and don't have time 
to try right now - suffice it to say that the 
horiz/vert sync figures from the old X config 
file don't seem to fit this next generation X). 
So, I need to configure ppp without gnome/kde 
tools, etc. 

I used linuxconf to activate the ppp modules. 
That seemed to work ok. 

I copied the contents of the TL4.0 /etc/sysconfig/network-scripts/ 
to the new RH7.1 system, as well as, to see these would work. 
/etc/hosts
/etc/host.conf
/etc/resolv.conf
/etc/ppp/options


Then I did ln - s /dev/ttyS1 /dev/modem , as the 
old scripts use /dev/modem. 

The scripts succeed in dialing the number but 
I don't get a ppp connection (can't ping a remote 
IP, can't get a web page with lynx). After a few 
seconds the modem hangs up. 

This is what appears in /var/log/messages
