Jonathan Shore <jshore@example.com> writes in tlug@example.com:

> Ultimately think lisp deserves a look if you don't need performance down to
> the wire.

It is an urban legend that Lisp implementations cannot be efficient.

> Yeah, but some of those languages which let you set the start index are also
> pretty scary (ADA for instance).

Ada[1] was explicitly required to support embedded and real-time
systems and implementations were required to do GC.  The two are not
mutually exclusive.  This is all explained in The Rationale for the
Ada language by Jean Ichbiah.  Yes, it is a scary language in its
entirety but that doesn't stop one from writing efficient code in it.


Footnotes: 
[1]  Ada is not an acronym so it is a misspelling to write it `ADA'.
