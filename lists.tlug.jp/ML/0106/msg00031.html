It sounds like the init script that failed had brought up some of the 
parport_pc's dependencies. Thus the parport_pc module loaded 
fine. insmod has no knowledge of dependencies. Instead try 
'modprobe parport_pc'. modprobe is a wrapper program for insmod 
that examines the output of 'depmod -a'. you might want to run 
'lsmod' before and after modprobe to see which modules are being 
loaded.

James Gibson
