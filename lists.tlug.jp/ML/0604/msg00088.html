Scott,

One thing you can do is try turning up debugging in your cupsd.conf.
Look for LogLevel and see the comments.  Then you can check
/var/log/cups_error.log
I changed "info" to "debug" in the cupsd.conf file, as per the instructions in the comments.
Then I went to CUPS web interface and tried to print a test page. It showed the job immediately complete without printing anything, as described before. 

Then went back to the command prompt:

dave@example.com:~$ more /var/log/cups_error.log
/var/log/cups_error.log: No such file or directory
dave@example.com:~$ locate cups_error.log
dave@example.com:~$
