On 17/04/06, Dave M G <martin@example.com> wrote:

> dave@example.com:~$ sudo find /dev -name usblp0 | xargs sudo ls -l
> crw-rw---- 1 root lp 180, 0 2006-04-17 18:21 /dev/usblp0

Well, shit. This looks OK, as I am sure Ubuntu is smart enough to have
the user under which cupsd runs in the lp group. Just in case, you
could run:

sudo chmod 666 /dev/usblp0

and then try a print job, but I doubt that will change anything.

Please look your printer up on:

http://linuxprinting.org/printer_list.cgi

Which driver does it recommend? Is it the same driver as you are using?

Did you install your printer using some sort of GNOME printer wizard,
or did you use the CUPS web interface (http://127.0.0.1:631/)?
