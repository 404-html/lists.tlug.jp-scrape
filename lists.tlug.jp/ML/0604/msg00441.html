Godwin, Jim, Emory,

Thank you for responding.

Happily, I think I have the solution to this one.
It occurred to me that this problem might be related to me recent trouble with the permissions settings on my /home/dave directory. Without going over lengthy details, I accidentally set my /home/dave directory to a very restricted set of permissions, so that I was having trouble logging in.
To solve that problem, I set my /home/dave directory to 750, following advice here on this list.
It was suggested that 750 might be better just in case I was concerned about others using my computer.
But 755 was also suggested. I just now set my /home/dave directory to 755, and now I can see the web_sites directory at localhost from within FireFox.
My only question is... if the last number in the 3 number chmod set represents "others", then why was this a problem? If I'm accessing the /home/dave/web_sites directory from within the "dave" account, which owns the directory, why was I denied access? Why did I have to grant permissions to "others" in order to get access myself?
