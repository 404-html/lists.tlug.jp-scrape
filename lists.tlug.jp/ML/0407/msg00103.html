  • Organization: Nope, none here, it's a mess ;o)
Yes, I've googled already, and there won't be anything in the TLUG archives
I haven't already seen in the mailing list because it's an issue with recent
software...

I upgraded Qt and KDE and compiled versions 3.3.2 and 3.2.3 respectively
from source.

When running xine (various versions of xine-lib from 1.0rc3c upwards and
xine-ui 0.9.23 upwards), the KDE taskbar (kicker) and X start hogging 100%
CPU cycles between them and of course the task bar becomes unresponsive. X
too to a certain extent.

If I kill the GUI session, switch back to KDE 3.1 (still with the new
qt-3.3.2), fire up X again and repeat the experiment, all works fine.

Kill the session, switch back to KDE-3.2.3, fire up X, start xine, kicker
and X eventually grind to a halt. So it's definitely something related to
KDE-3.2.3.

kdebase-3.2.3 was compiled with the following options:

./configure --prefix=/opt/kde-3.2.3 --sysconfdir=/etc --disable-debug \
--with-qt-dir=/usr/lib/qt --with-qt-includes=/usr/lib/qt/include \
--with-qt-libraries=/usr/lib/qt/lib --with-ssl-dir=/usr/local/ssl --with-x \
--libdir=/opt/kde-3.2.3/lib --includedir=/opt/kde-3.2.3/include

This was immediately after kdelibs-3.2.3 was compiled thusly:

./configure --prefix=/opt/kde-3.2.3 --sysconfdir=/etc --disable-debug \
--with-extra-includes=/opt/kde-3.2.3/include \
--with-extra-libs=/opt/kde-3.2.3/lib --with-qt-dir=/usr/lib/qt \
--with-qt-includes=/usr/lib/qt/include \
--with-qt-libraries=/usr/lib/qt/lib --with-ipv6-lookup=no \
--with-ssl-dir=/usr/local/ssl

/usr/lib/qt is a symlink to /usr/lib/qt-3.3.2

I switch from one version of KDE to the other by making a symlink /opt/kde
either to /opt/kde-3.1 or /opt/kde-3.2.3.

xine-lib is compiled with these options:

./configure --prefix=/usr/local --sysconfdir=/etc \
--with-libiconv-prefix=/usr/local

xine-ui is compiled with:

./configure --prefix=/usr/local --sysconfdir=/etc --enable-xft \
--with-libiconv-prefix=/usr/local --with-x

So, I have a nice easy question. What have I screwed up in KDE-3.2.3? :)
