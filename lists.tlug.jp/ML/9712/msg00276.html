   Same subject, different method... I tried "online" a couple of
   weeks back and didn't much care for it (it also crashed my pilot
   with a memory error as couple of times, too). Because of that, I've
   been playing around with Top-Gun Telnet this evening and finally
   got it to work...

   I'm using 19200 baud with a network preferences service I created
   called "Linux" with the magic telephone number of "00", an empty
   script and a hard coded IP address.

   On the Linux box I do:-

	pppd 19200 /dev/pilot -detach local passive \
				172.17.172.70:172.17.172.69

   (where /dev/pilot is a symbolic link to the tty port which I use
   for pilot-link). Selecting the "connect" button brings up the
   link and I can then select "apps" to get into the TG_Telnet
   application. Typing an IP address into the very top line and
   selecting "open" gets me a telnet login to the linux machine. A
   termcap entry for the pilot is included in the TG_Telnet package.

   I still need to work out routing (probably proxy ARP) and make a
   domain-name service  available to the pilot, but the basic package
   seems quite solid. Definitely worth a try.
