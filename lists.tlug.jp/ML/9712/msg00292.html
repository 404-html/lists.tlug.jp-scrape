On Sat, 27 Dec 1997, joem wrote:

 
> After some fiddling around, I now have an apparently reliable 
> CDRom burning system.
> Yamaha CDR400tx (get bios version > 1.0d)
> Xcdroast 0.96d (beta version 1997.12.20)
>      ---> www.fh-muenchen.de/rz/xcdroast
> Xcdroast now uses cdrecord instead of cdwrite, and supports 
> a whole raft of writers.  The user interface is good, and 
> the software supports (with warnings) CD to CD direct copies.

  Yes, this makes a nice little system.  I'm using the Yamaha CDR200t.
CD to CD only works with data disks, not music CDs - heaven only knows
why.  And it seems to me that I found things didn't really work
properly unless I ran 'xcdroast' as 'root', but I've forgotten the
details now.   

  I have found that 'mkisofs', which XCDroast uses to build an iso9660
filesystem on hard-disk, seems to sometimes make a file (the original
has always been a short text file) unreadable - i.e. trying to read it
when the filesystem is mounted produces an I/O error.  I haven't had
time to pursue this, but you might want to check the readability of all
the files in a CD image (with 'file' or something) before burning the
CD.  

 mount /dev/sda1 -r -t iso9660 -o loop /cdrom

works fine for mounting the CDimage so that it can be checked out.
(This same idiom will work fine for mounting a file-image also).

  I've only tried copying a music CD twice, but I found both times
that the resulting CD won't play straight through, but tends to stop 
after every track - although each track can be played without any
problem once you manually start it playing.  If Joem or anyone knows
why, I'd like to know what I'm doing wrong.
