> When I attempted to write, the message was the drive was read-only. I set
> the permissions for writing, but this had no effect.

My guess is that RH (wisely) hasn't supplied UDF-write capabilities in the
kernel. Even in the latest 2.4.20 it is considered experimental and
dangerous.

Dangerous or not, it should be discouraged. When you "format" a CD in packet
mode you start by losing about 100 megs of capacity. Secondly, no Windows
user can read the CD without appropriate software. Finally, if you can burn
multisession CD-R's there really is no need for UDF-write support.
