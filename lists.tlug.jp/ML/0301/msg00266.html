On Wed, 22 Jan 2003 11:18:44 +0900, Jean-Christian Imbeault
<jc@example.com> wrote to tlug@example.com:

> If this is so, what is the proper way to compile a custom kernel *but* 
> have it create it's own set of modules without overwriting/deleting any 
> modules another kernel uses?

Edit /usr/src/linux/Makefile and change this:

VERSION = 2
PATCHLEVEL = 4
SUBLEVEL = 20
EXTRAVERSION =

...so that it reads:

VERSION = 2
PATCHLEVEL = 4
SUBLEVEL = 20
EXTRAVERSION = -alt

You'll now be compiling a 2.4.20-alt kernel and its modules will be sent to
/lib/modules/2.4.20-alt leaving the previous compile's modules intact.
