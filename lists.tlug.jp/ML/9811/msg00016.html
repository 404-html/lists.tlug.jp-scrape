>>>>> "Totoro" == Totoro  <riley@example.com> writes:

    Totoro> I'm puttering around with Debian lately, and can't get
    Totoro> j-pine to compile. Pine and Pico both want to link
    Totoro> termcap, however, Debian is supposed to be using
    Totoro> terminfo. With the old termcap-compat in, the errors
    Totoro> continue. I've already tried changing the makefiles on
    Totoro> Pine and Pico to not use termcap, but it won't run.

As far as I know, although terminfo is preferred to termcap for new
applications, legacy applications should compile with termcap support.

Unfortunately, Debian-JP does not measure up to the quality of Debian
in general IMO.  And in any case, you probably need to be extremely
careful about compiling Debian sources on the release they were
configured for.  You often won't be able to compile 1.3 .dscs (you are
building from .dscs, aren't you?) on a 2.0 system, or vice versa, and
"slink" (current experimental branch) sources are likely to be worse.

However, it's probably better to start from a Debian-JP .dsc for the
wrong system release (or an old version of Pine) than from UW sources
+ Japanization patches.  At least that will give you some idea of how
to work around the weirdness.
