On 16 Nov 98, at 17:30, Dave Gutteridge wrote:

---8<---

> What i'm left with is usually a
> directory, for instance "qpopper2.53" in my download. Is it okay to delete
> this directory?

Ninety-nine-point-ninety-nine percent okay.  ;-)  Seriously, any 
programmer worth his or her salt will have the install program create any 
needed but non-existent directories, and transfer all the essentials to 
them.  For the kind of programs you've mentioned, it's a virtual certainty 
that you can safely delete the directories created by untarring.

> some directories were created in my download directory that are
> only accessable by the root and won't let me delete them without super
> user status.

Because you created them as root, right?  (You unzipped the tarball as 
root.)

Build them in tmp directories.  Then, take a look around and see what 
the install program created.  More than likely, you'll see right away that 
the tmp directory tree is no longer needed.  If you want to play it real 
safe, however, mv the tmp tree first, then run the program.  If it squawks 
because it can't find something you moved, you'll know what the deal is.
