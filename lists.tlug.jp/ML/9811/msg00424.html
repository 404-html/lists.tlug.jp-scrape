I'll be doing some research work on sokoban (one person puzzle game, with
many similarities to real-life problems in warehouses and ports):
  http://xsokoban.lcs.mit.edu/xsokoban.html

I found a (Microsoft) windows version, but though it is freeware it does
not come with the source. So I thought I might try porting the unix X
windows version to win32 (and keeping my changes as 'open source' of course).

This will be useful for me to be using the same input/output files as other
researchers, as well as giving me a chance to learn a bit of X windows
programming.

My initial questions are:

1.Am I crazy? Is this a major task? There are 21 .h and .c files, totalling
141Kb.

2.Is there a utility (Linux or windows) to convert .xbm and .xpm files to
.bmp format?
