>From my earlier post <13896.4401.969032.997258@example.com>:

So you need to make sure that the IP (numerical) address -> domain
name -> IP address succeeds.  The CNAME record says which of the names 
will be returned by a PTR (numerical address) search.  The A record
says what IP address will be returned by a name search.

And for completeness:

NS:   a nameserver which knows about this domain (sort of redundant for 
      hosts, but necessary for domains which are networks)
SOA:  the primary source for information about this domain, along with 
      some parameters for expiring old cache information
PTR:  the inverse of A, a map from IP address to the CNAME
AXFR: all the information about this domain, including any subdomains

There are a couple of others, IIRC, but I don't recall what they are.
