>>>>> "me" == Stephen J Turnbull <turnbull@example.com> writes:

    me> It's a server, there should probably only be one running at

That is, I think you should probably start one from the command line,
once.  Once you've got that working, you can experiment, and if
starting it from init.d works, use that.

kinput2 does maintain multiple connections and input contexts as far
as I know, so I think running from init.d will work.

There is a potential security hole if it runs as root; I think that it 
does have the capability to manipulate dictionaries.
