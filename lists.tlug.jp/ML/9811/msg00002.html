On Sat, Oct 31, 1998 at 04:41:34PM +0900, Sean Bennett wrote:
> c) From a terminal window on my Win98 box, I can login as
> "sean" and then become "su", but I can't login as "root".

That is a security feature, since it adds an extra limit on who can become
root away from certain specified ports, as well as providing a bit more
logging.  login is generally configured so that anyone not at the console
must log in first and then su.  If you're on a network you trust (snicker
:-), you can remove this restriction by adding more entries (of the form
ttyp0, ttyp1, etc) to /etc/securetty ('man securetty').
