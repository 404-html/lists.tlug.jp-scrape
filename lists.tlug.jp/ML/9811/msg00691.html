My server machine got moved from one location to another, so i had to shut
down the machine and stop my Apache server for a bit. The machine still has
the same IP address, it's only in a different physical location in the
office. 
But now my problem is that the server won't come up.
Here beside me is my trusty "Apache for Dummies" book, and it says to do this:

	Start the new server using the new image:
	#./httpd -d 'pwd'

For starters, the book thinks the httpd file should be in
/usr/local/web/apache. But on my system i found it in /usr/sbin. So perhaps
i have the wrong file? But this is the only one that "find / -name httpd
-print" came up with (along with a few directories of that name).
So i did as they said, and i came up with these errors:

	# ./httpd
	Cannot resolve host name www.d-rave.com --- ignoring!
	Cannot resolve host name www.tokyostreethockey.com --- ignoring!
	# Cannot resolve host name www.d-rave.com --- ignoring!
	Cannot resolve host name www.tokyostreethockey.com --- ignoring!
	httpd: cannot determine the IP address of the specified ServerName
(www.tokyostreethockey.com).

I checked the Apache FAQ, and it is now geared towards version 1.3. I have
version 1.26. I read it anyways, and in the section on name based virtual
host support it says i should be adding lines to my conf file. But this
solution irks me a bit because the same conf files were working fine before
the machine moved. I've also lost my mail services which i had struggled so
long to get working (again they still have the same configuration files),
so i can't help but think the split is something more general.
But where?
Any suggestions would be greatly appreciated.
