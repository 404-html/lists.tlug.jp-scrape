In order to hibernate to disk you need to have a special partition set
aside for that.  If you nuked it when installing Linux, I don't think it
will hibernate to disk.

The APM_DEBUG might help, though.  TL's kernel source package is set up so
that you can go into /usr/src/linux and run 'make dep;make clean;make
zImage' and it will build a default TL kernel - so... you'll have to run
make menuconfig there and turn on APM, then turn on APM debugging in
whatever .h file it's defined in, and then 'make dep;make clean; make
zImage'.  You shouldn't have to remake the modules or modules_install,
however.
