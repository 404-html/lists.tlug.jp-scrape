On Sun, 15 Nov 1998, Tony Laszlo wrote:

> Quick newbie question: 
> 
> I ran 'dmesg | more' and noticed that there was 
> no mention of 'PPP' in the results. 
> I understand there should be something like 
> 'PPP Dynamic channel allocation code copyright 
> 1995 Caldera, Inc.' and 'PPP line discipline 
> registered.' in there. 
> 
> Before I download and install whichever file 
> is needed to adjust the kernel, I want to be 
> sure that those dmesg results necessarily 
> mean that my kernel was not configured to 
> run PPP when TL 2.0J Standard was installed.  

ppp is loaded as a module when you attempt to connect, please don't
recompile the kernel :)

I've noticed that some isps simply do not want to do PAP at all.  For
those, you'll want to dial up with minicom and see what the login/password
prompts look like, then run turbopppcfg and enter stuff in the
expect/sends input fields appropriately (ie, set up a series of 'look for
this and send this back' pairs).  Make sure you turn off the PAP checkbox
too.
