Your message is off topic for this mailing list. You have to
post this kind of question to comp.sys.sgi or fj.sys.sgi.

gcc-2.7.x for IRIX _only_ produce o32(old 32bit) object.
You have to use gcc-2.8.1. 2.8.1 can produce n32 object
because n32 object is much faster than o32 object.

The procedure to compile gcc/g++ is trivial. It is almost same
as Linux.

Some information is available in following URL:
http://reality.sgi.com/ariel_engr/freeware/gcc-2.8.1.html
