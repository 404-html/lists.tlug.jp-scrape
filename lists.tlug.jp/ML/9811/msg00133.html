Thanks for all the input so far on my LILO problem.  I do have some progress
to report and have apparently found the culprit.  When looking at the
Advanced Settings section of my BIOS, I found that the large disk access
mode had become DOS instead of Other.  Since I never touched this but know
for certain that it was previously set to Other, I'm mystified as to what
caused this, setting it back to Other fixed the problem with reporting the
number of heads on my 6.4 GB disk, and fdisk now reports 16 heads, 63
sectors, 12592 cyclinders.  So far, so good.

Running LILO from a shell prompt works.  So far, so good.

BUT - at bootup, it still gets as far as printing LI and then freezes, so
I'm still using a sysboot disk to boot the system.  The boot partition is
completely within the first 1024 cylinders, so this should work (indeed, it
used to), so I'm kind of stumped as to why it doesn't now.  What's the final
magic key I need to get it back to normal?

The partition table and lilo.conf are below.

Thanks,

Jonathan
