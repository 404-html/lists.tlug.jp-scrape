On Wed, 4 Nov 1998, Scott Stone wrote:

> > >         libtermcap.so.2 => /lib/libtermcap.so.2 (0x40004000)
> 
> What's wrong with the above?

ncurses replaces termcap/curses.  Across the board.  Termcap is obsolete.

If one has ncurses installed on the box, all applications should be linked
against it.  Termcap/curses should be around only for legacy binary
compatibility, _never_ linked against.  Once termcap/curses is eradicated,
then one can safely delete /etc/termcap since ncurses uses
/usr/share/terminfo.
