On Fri, 20 Nov 1998, Michael Casinghino wrote:

>I have moral objections to making distinctions between /usr/local/ and
>/usr/.  I mean isn't _everything_ local?  Really?  Is there a reason I
>shouldn't do this?

Eh, then you'd certainly have some pretty local things...
(/usr/local/local/local/local/local/local/local/local/local/local/bin...)

:-)

For me, it seems reasonable to install all user software compiled from
source code under /usr/local so some stupid "make install" won't over-
write the essential binaries and libs under /usr.

Other way to do this is to create for example a /opt directory for these.
