For openSUSE users there is also a special repository specific to input 
systems.

http://software.opensuse.org/download/M17N/openSUSE_10.2/

Import the authentication key by entering the following as root:

# rpm --import http://software.opensuse.org/openSUSE-Build-Service.asc

There's a good reason for doing so if you are using KDE, especially. There has 
long been a notorious bug caused by interaction between SCIM and qt, on which 
KDE is based. Some packages that fix the bug were uploaded by Mike Fabian of 
SUSE recently, and they are available only from the above site. So far, since 
updating the SCIM and other packages, I have not been able to reproduce the 
bug (which caused KDE programs such as KMail to crash whenever a shortcut key 
was pressed while in Japanese input mode).
