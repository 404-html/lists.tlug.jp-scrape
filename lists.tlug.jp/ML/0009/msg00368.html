On Mon, Sep 25, 2000 at 04:09:08PM +0900, A.Sajjad Zaidi wrote:
> Im trying to find a way to control access to the main samba server
> between different groups of people.

If you receive any suggestions other than this one, you should
probably heed them and feed this one to your nearest delete button.

I haven't checked out this spec in later versions of Samba.  When I
considered ways of working around this earlier this year, I planned to do a
script that could spin a copy of smb.conf, given an existing template file
and a configuration file that gives the members of individual groups. This
is not pretty, because you have to work the bugs out of the script (which I
have not yet written ...), but it should work, since Samba rereads its
config once per minute or so in between -- unless that lag-time is a problem
in your environment, you don't have to restart the server after spinning up
a new smb.conf file.

Cheers,
Frank Bennett
