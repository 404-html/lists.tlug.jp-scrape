Hi,

I'm using Pine to get my e-mail.  However, when I receive e-mail
from a mule mew mailer I get this message at the top of my
message.  

  [The following text is in the "ISO-2022-JP" character set]
  [Your display is set for the "US-ASCII" character set]
  [Some characters may be displayed incorrectly]


I can read the Japanese text okay.  However, my display is evidently
set incorrectly.  Does anyone know how I can change this display.

Also, my post to the Debian-JP mailing list was unreadable since my
outgoing message header contained this:

  Content-Type: TEXT/PLAIN; charset=US-ASCII


Evidently the mews reader checks the content type pretty closely.
Is there some way I can edit this within Pine?

As I mentioned before, the main reason I am using Pine is for the
IMAP capability.  Recently, I've tested Netscape Communicator 4.0b3
with IMAP capability and it works okay.  However, the editor on
Netscape sucks.  I would like to write my e-mail with mule and
get my e-mail with IMAP. My preference is to read my e-mail within
mule too, but this is not a top priority.    

The Pine-mule system I am using now would work okay if I could
change the Content-Type: mime header to indicate proper Japanese
encoding.

Regards,
Craig
