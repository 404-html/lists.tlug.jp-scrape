I just downloaded the JDK1.1.1, and am now downloading JDK1.1.1 docs.
For those of you on SINET, the HTML versions of the docs will be
available in http://turnbull.sk.tsukuba.ac.jp/Docs/Java/ (and Apache
HTTPd docs are in Docs/Apache).  If there is demand I'll make the tar
files available but this requires rearranging my file system (/WWW is
nearly full :-), so since most TLUGers seem to have great trouble
accessing SINET sites :-( I'll postpone that.

I noticed while hanging out at Javasoft that there is now a Japanese
version of Java.  Now, the java runtime itself is Unicode-based
(nyet?), so the localization issues should all be in the classes.
Thus, I imagine that we should be able to run 'generic' Java
applications.  Since `javac' et al are Java programs, we should be
able to use Solaris versions, right?  (Just like with HotJava - see
www.blackdown.org's information on using HotJava with Linux.)

But before I go nursing a delicate FTP connection for an hour or so
with multiple regets :-( I wonder if anybody else has tried this....
