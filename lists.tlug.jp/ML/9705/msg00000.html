> Right now, I have figured out how to use canna/kinput2
> with kterm. Even though the input seems to work on the
> command line, I can't get nvi to let me input japanese

I'm using jvim.  You need canna-nvi if you intend to use
canna with nvi.  Precompiled deb package available.  See
above URL.  

To get canna to work with mule, you need mule-canna and
put this in your .emacs

  (load-library "canna")
  (canna)

To get canna, mule, kinput2 to work together, start mule in
a kterm with mule -nw and add some stuff to your .emacs
This is documented on my web page.
