On Tue, 13 May 1997, Dennis McMurchy wrote:

>   I've only had a quick look, but unless I'm missing something, there's
> no 'Netscape Download' window when you're downloading.  Can this be? 
> You have no way to tell how the download is going or whether it is
> finished other than to look at the directory the file is going to.  That
> was certainly the case with the only download I've done with it.

I get the download box on my system. 

>   Although 4.0 tells you that you can alter the Cache settings in an
> Edit - Preferences  dialogue box, there's nothing there to do with the
> cache.

In Preferences, hit "Advanced."   This opens up a vertical menu.
Select Cache.

>   There's still no sound with Java, and although all the java demos seem
> to work (even without the library preloading kludge that is necessary with
> 3.0), 4.0 locked up on me when I looked at  

I don't have sound with Java either.  I also get some crashes on
certain sites.  

However, right now Hot Java is core dumping on me all the time and
XEmacs doesn't have Java support.  So, Netscape 4.0 is okay. 

>   From what I've seen so far, I'm not encouraged to explore the 'features'
> of Communicator 4.0 any further.  Anyone like to tell me why I should?

IMAP.

I'm putting together a bit of information for our membership
at http://www.twics.com/~craig/twics/imap/

It should be ready in a few days.  However, it covers the basic
differences between IMAP and POP.  Personally, I use Pine's IMAP
capability.  So, if you don't like to read your e-mail in Netscape,
you probably don't need Communicator 4.0 until it is out of 
Beta.  The messaging center is pretty cool because it integrates
e-mail and news.  However, XEmacs/mule will do this with better
integration.  If XEmacs supported IMAP, I would like in XEmacs
and just come outside for guest appearances.

Craig
