Is anyone else having a problem with:
  window overflow
  watchdog timer reset
occurring on their SparcLinux box?

I've been unable to narrow it down, but seem to get it frustratingly
often when trying to use either Grail or XEmacs's web browser.  At
first I thought it was network related, but I have been unable to
provoke the problem with Lynx... so now I am suspecting either X or
the GIF/JPEG graphics conversion routines (although admittedly it
doesn't have as many socket openings/closings in rapid succession).

The problem has happened with a number of vger snapshots, both the
RH 4.0 and RH 4.1 X distributions, Grail 0.2 and 0.3.
