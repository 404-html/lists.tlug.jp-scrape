I just upgraded to Pine 3.96 from Pine 3.95.  IMAP4 support is
better.  I don't get delays anymore every time I try to open my
inbox on the remote server.  I've also set Procmail on my
ISP's server to filter tlug mail into a seperate incoming 
box and then configured Pine to view multiple inboxes.

Here is the server side .procmail

beehive:~$ cat .procmailrc                                              
## Environment Variables
##

PATH=$HOME/bin:/usr/bin:/bin:/usr/local/bin:/opt/procmail/bin:.
MAILDIR=$HOME/mail      # You'd better make sure it exists
LOGFILE=$MAILDIR/from
LOCKFILE=$HOME/.lockmail
SHELL=/usr/bin/sh

##
## TLUG Mailing List
##
:0
* ^TO.*tlug@example.com
tlug.incoming

-----------------------

Here is the section I added to my local Linux .pinerc


# List of incoming msg folders besides INBOX, e.g. ={host2}inbox,
{host3}inbox
# Syntax: optnl-label {optnl-imap-host-name}folder-path
incoming-folders={beehive.twics.com}mail/tlug.incoming


------------

Re: Debian-JP 1.3 Bo

I've done an install of Debian-JP 1.3 Bo from scratch.  Bo is a
lot smoother than Rex.  It went very smoothly.  Bo contains
working binaries for Alladin GhostScript 4.0.3 with vector
font support and Japanese fonts.  I'm able to print Japanese
now without having to compile gs from source.  JLaTeX and jxdvi
installation also went smoothly.  Also, the connection to
ftp.debian.linux.or.jp now seems very fast from my system.
For the English Bo, the ftp.lab.kdd.co.jp FTP server with
its 256 megs of RAM on a dual processor 200mhz Pentium Pro
machine is really fast.

Pine is available in the non-free section of Debian.

Regards,
Craig
