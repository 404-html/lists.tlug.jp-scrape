Hello,
    Iam trying to get the display of KTERM in a Windows 95-Japanese Machine where I run my X-Server . I get a warning on startup of kterm saying.
"unable to open font "" , trying fixed" . This happens irrespective of the font I specify in the command line option .Iam running a font server (xfs) on the Linux Machine . On my Windows 95 machine I connect to this Font Server through a specified port. 
 I debugged the KTERM code , but I was not able to exactly figure out why this is happening. This "trying fixed" warning comes when I start Xterm also specifying the fl option for the desired font.

for example when I start KTERM with the following option
    kterm -display -fl -hp-gothic-medium-r-normal--14-*-c-*-jisx0201.1976-0
or any other font for that matter which is there ib my Xserver, I get the warning message . 

I tried Xfontsel , Xlsfonts and Xfd . Xlsfonts is able to display all the "hp" fonts 
when I specify the pattern "*hp*.

:~$ xfd -display  -fn -hp-gothic-medium-r-normal--14-*-c-*-jisx0201.1976-0
output:
Warning: Cannot convert string "-hp-gothic-medium-r-normal--14-*-c-*-jisx0201.1976-0" to type FontStruct
xfd:  no font to display

~$ xfontsel -display  -fn -hp-gothic-medium-r-normal--14-*-c-*-jisx0201.1976-0
output:
Warning: Cannot convert string "-hp-gothic-medium-r-normal--14-*-c-*-jisx0201.1976-0" to type FontStruct

$ xlsfonts -display  -fl "-hp-gothic-medium-r-normal--14-*-c-*-jisx0201.1976-0"

output:
-hp-gothic-medium-r-normal--14-101-100-100-c-70-jisx0201.1976-0

I have run mkfontdir on all the subdirectories . 

Can any one of you (KTERM gurus) figure out whether Iam running into a rough weather with this fonts .

PS: I also have all these fonts used by my X-Server under a separate subdirectory. These are (.FON) fies recognized by Windows . These are the same files as in the Linux subdirectory. So, I face the same problem even if I stop the font server .
