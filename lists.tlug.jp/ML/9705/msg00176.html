Hi,
I'm trying to print to a Kyocera L880S printer with Linux.  I
have the printer connected to my parallel port and can 
spit out a limited amount of ASCII text.  I want to use 
something like GhostScript to filter output to the printer.
The Kyocera printer is not listed in the supported DEVICE 
listing when I do a 

  # gs -h

I've tried the standard LaserJet4, NEC, and epson to see if
it was compatible with a common printer.  No luck so far.
Should I try to recompile gs and look for an optional file
with more printer devices?

The Kyocera printer does not appear to contain standard
PostScript fonts since a cat of a PostScript file prints
out the PostScript commands as text.

Any relevant printer battle stories would be appreciated.

Regards,
Craig
