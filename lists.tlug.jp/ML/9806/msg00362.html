>>>>> "Manuel" == Manuel Chakravarty <chak@example.com> writes:

    Manuel> Hi!  I am using Canna in Mule (2.3).

    Manuel> * Can I directly enter katakana?

Probably, but I don't know how.  fgrep key list/canna*.el.

    Manuel> I always enter hiragana
    Manuel> and then press space for conversion.  If the word is not
    Manuel> in the dictionary, I press two times up-arrow to get
    Manuel> katakana. That are definitely too many keystrokes.

You can save one by using SPC only if you're hoping for a spell-check.
You can save another by using C-N or down-arrow for the conversion,
instead of 2 ups.  (On XEmacs 21.0, and I'm pretty sure 20.[34].)

    Manuel> * Is there a quick way to produce ティ？ Entering `ti'
    Manuel> gives ち。

`x' is the make-small operator.

"te xi" C-N RET => ティ

    Manuel> * Entering kana, the <backspace> and <delete> key are
    Manuel> unbound, and I have to use C-h to erase a character.  This
    Manuel> is very un-Emacs-ish and I always mix up keys.  I could
    Manuel> start looking for the keymaps, but...

    Manuel> BTW, are there differences in key bindings for Canna
    Manuel> between the different Emacsen?

Probably.  For example, no keys are bound for Canna in the FSF's Emacs 
20.x since it isn't supported.

XEmacs binds <backspace> but not <delete> (which I never use anyway).

I don't have a copy of GNU's version.  In XEmacs this is done
automatically by copying all the global keymap stuff to the Canna map.
