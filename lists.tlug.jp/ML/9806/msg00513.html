Hi, folks--

This is to announce the first release of _mapster_, a Tcl/Tk-based
image-map generator. I wrote it because I wanted to make some
client-side image maps including complex polygons, and there didn't
seem to be any program running on Linux that would take care of the
tedious task of finding the coordinates. Now there is one. I'm calling
this an alpha release, but to the best of my knowledge it performs the
limited task it was designed for without too many serious problems.

Mapster will not boil your eggs or walk your dog for you ;-) all it
really is is a convenient way to map out areas on an image, find the
coordinates, and spit them out as HTML <AREA> tags.

To run the program, you'll probably need Tcl/Tk 8.0 (might work with
earlier versions, but not tested yet). Also, it uses only GIF images
internally, so if you want to import JPEG, PNG, or TIFF images, you
need the 'convert' program from ImageMagick (or something that works
the same way -- i.e., works with no option flags, only the old and new 
filenames as arguments).

So, anyone who has the need or desire, dozo! You can download it from (and
find more info at):

http://www.geocities.com/SiliconValley/Peaks/8143/ts-mapster.html

If anyone tries it, let me know how you like it.
