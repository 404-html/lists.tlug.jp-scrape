Ah ... yah.  About the package interface....

Anyway, packages aren't a released feature yet, although they have
worked since 20.3 (both the API and the UI were controversial and no
upgrade tools existed).  In beta/xemacs-21.0/packages/binary-packages,
do a `ls eudc*' and get the most recent version.  Create an ~/.xemacs
directory and unpack the eudc tarball there.  It should create some
subset of etc, lisp, and info subdirectories, with an eudc
subdirectory in etc (if it exists) and lisp and eudc.info in the info
subdirectory.

Then restart xemacs 20.4, and do C-H i eudc.  It should be that easy.
