I have not tried it in a long time, but I know a one point XFree86
trapped ctrl-alt-del, or some similar sequence to do a normal
shutdown. This way you don't have to worry about the file system
getting corrupted or wait for it to be checked when the system
reboots. 

Also, you might be able to do, oh what is it, ctrl-alt-F1 (F2 F3...) to
get another virtual session. This will be outside X, at which point
you can kill X and figure out what went wrong...

Hope this helps.
