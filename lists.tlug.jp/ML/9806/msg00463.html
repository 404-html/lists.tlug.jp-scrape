Hi all,

I want to automatically start an X session right after logging in,
however, I don't like using "xdm".  I think it's stupid to see the login
prompt come up, then wait 5-10 seconds for the xdm screen to come up. 
What I would like is once I login I want the 'xinit' command to
automatically initiate.  I thought I had a smart idea, but it doesn't
seem to work.  I knew that the .bashrc file was read immediately after
login, so I added the 'xinit' command to this file.  However, once the
session began I received the error message, "display X-0 is already
running, if not remove the /tmp/.X0-lock and restart."  Is it possible
to do what I want and not get this error message?

Would I be really messing up something if I added the following to the
.bashrc file?

exec xinit

if [-f /tmp/.X0-lock]; then
	rm /tmp/.X0-lock
fi

If anyone else has done something like this let me know.  I really hate
using 'xdm' and would rather just have the Xsession start right after
login.  Thanks in advance,

Blaine
