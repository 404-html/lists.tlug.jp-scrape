David Oftedal wrote:

> What would be REALLY cool, though, is if we finally got a tri-lingual
> CJK input method that didn't need X at all

The gtk2 api provides this. And afaik there are gtk2 input modules for
chinese also.


> Perhaps a modification to the VGA framebuffer module in the kernel
> would prove to be the final solution to this problem.

And then what? There is still lots to implement besides the low level
stuff that the framebuffer provides. GTK2 in itself does not satisfy all
the needs of a modern windowing system. For PDAs, handhelds and other
systems with limited capabilities it might be enough.

This rant about X beeing an outdated protocol often comes up at various
MLs and forums, which to some extent is valid, but until somebody comes
up which is powerful enough, there is not much of a choice.

ps: top posting is considered a bad netiquette on this list.
