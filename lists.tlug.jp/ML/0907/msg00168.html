Stephen J. Turnbull wrote:
> Edward Middleton writes:
>  > Do any of the mercurial hosting services have anything equivalent
>  > to the github network graph visualizer[2]?
> 
> Not that I know of.  I don't know why Mercurial users would want one,
> to be frank; Mercurial itself strongly encourages linear development
> and rapid reconvergence to a handful (at most) number of branches.

This graphs project divergence across all branches and forked
repositories of the base repository on github.  It also eliminates
modifications that have been merged so you only see divergence.

I guess the point is that not all users can or should have access to
the/a trunk, but seeing where their modifications fit in is useful.  The
graph viewer has some limitations when you look at something like rails
with over 600 repositories.
