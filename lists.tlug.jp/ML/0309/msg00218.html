Yet another font/locale problem on Linux...

This isn't Japanese-related but TLUG'ers might have an answer.

Fonts are set up as follows on my Slackware 9.0 box:

- M$ TrueType fonts (corefonts plus Tahoma) are set up in
  /usr/X11R6/lib/X11/fonts/ttf/ and there is a FontPath line
  referring to this path in my /etc/X11/XF86Config.

- /etc/X11/XF86 also contains Load "freetype".

- The font path mentioned above contains the required fonts.dir,
  fonts.scale and font.cache-1 files.

- TrueType fonts are rendered fine except for the case described
  later on.

- /etc/profile.d/lang.sh sets LANG to en_GB.ISO8859-1 and this is
  confirmed by running "locale".

Some helpful (not...) souls design web pages with extended ASCII characters
in them such as funky slanted apostrophes and quotes etc. One example page
like this is http://www.ljvideo.com/whazzit.html

In Mozilla/Firebird on Windows, this page renders like this (cropped on the
interesting bits): http://www.sgms-centre.com/fbwin.png

I also use Mozilla/Firebird on Linux (compiled with the --with-xft option in
order to get font anti-aliasing) and it looks like this:
http://www.sgms-centre.com/fblinux.png

If I download the source and load it into a GTK2 editor such as gedit-2.x,
using the same font (Tahoma), it looks like this:
http://www.sgms-centre.com/gedit2.png

A similar problem shows up in a GTK+1.2 application such as Sylpheed:
http://www.sgms-centre.com/sylph.png

And yet most extended characters, such as letters with accents, are rendered
fine.

There I was thinking this was an XFree86-specific problem until I loaded the
HTML page into OpenOffice-1.1.0rc3/Linux and got this:
http://www.sgms-centre.com/oolinux.png

So, in short, what is wrong with my X setup so that only OpenOffice can
render TrueType correctly?
