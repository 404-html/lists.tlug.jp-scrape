On 2014年05月08日 11:43, Raymond Wan wrote:
> It would be nice to be on Debian stable and use testing's Firefox (for
> example) without breaking anything else.  And when I finally upgrade
> the entire system to testing, it manages all those duplicate libraries
> auto-magically...

You can run the latest Firefox release (or Aurora if you want bleeding
edge) in Debian stable very easily, within the package manager:

http://mozilla.debian.net/

The problem in general (of running testing packages in stable) is solved
by backports:

http://backports.debian.org/

Backports are managed; not all of the software/versions that you want will
be available, most likely.  You can create your own packages as necessary,
however.

Note that Ubuntu's Personal Package Archives (PPAs) are used in a similar
way in Ubuntu:

https://launchpad.net/ubuntu/+ppas

Cheers,

Travis
