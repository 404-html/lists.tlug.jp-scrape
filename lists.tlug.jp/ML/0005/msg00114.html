Steve's problems are DNS related.  The SOA for the zone file on shako
has a serial number of `502', however there are some hosts that think
the serial number is `200003428'.  Presumably the cached data under
the higher serial number is erroneous.

> set type=soa
> sk.tsukuba.ac.jp
