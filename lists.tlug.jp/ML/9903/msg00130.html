>     I'm having some problems with my timed: I have a time server on an
> NT (hate that, but it's boss rules) and, for some reason, my timed ain't

> any master-server (at /var/timed.log) then shuts down, but it's looking

> and UDP conections on port 37... that's it!

Are you sure the NT time server is using the TSP protocol?  The man page
says timed selects a master based on *broadcast* not an explicit
connection to port 37.
