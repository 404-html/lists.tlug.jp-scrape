>>>>> "Richard" == Richard M Pavonarius <richard@example.com> writes:

    Richard> There are a few X apps I can't run because it says it
    Richard> can't find the libraries. But when I search for them,
    Richard> they're there, but obviously not in the "right"
    Richard> place. How do I figure out where the app expects the
    Richard> library to be?

My guess is that you have a mixed system, libc5 and libc6.  It's
pretty hard to get this to work with X, because the X libs don't have
a version difference between those compiled for libc5 and those
compiled for libc6.

/sbin/ldd <program>

will tell you what the program is looking for.
