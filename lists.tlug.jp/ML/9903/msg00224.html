xhost is not really a good thing to do. You should use xauth for this. 

I think most default X configurations use it instead of xhost. Also
xdm authentication is based only on xauth. And if both of them
(i.e. xhost and xauth) are present it's generally a *BAD THING* -
xhost overrides xauth.

The simplest solution I've found so far is:

su
cd /root
ln -s ~<your_id>/.Xauthority 

This will allow all connections from root to your X session on local
machine.

To allow connections from other machines its 'xauth add displayname
protocolname hexkey' - protocolname is usually a '.' for default.
