>>>>> "jb" == Jonathan Byrne <jq@example.com> writes:

    jb> The question I was researching when I hit the archive glitch
    jb> was: What do I need to do to be able to let other users
    jb> connect to my userid's X server and open displays?

Probably the best way to do this is to use `xhost +local:'.  This
allows all _non-network_ connections to your X server to succeed.

This would allow an outsider telnet'ing in to your box to mess with
your screen and even get information about your keystrokes and so on.
But if that's happening you've probably got more immediate worries
than that they'll be sufficiently anal to import the necessary hacking
tools to hack X windows.

You can also use more complicated per-user authorization schemes, but
I don't understand them, having never used them.
