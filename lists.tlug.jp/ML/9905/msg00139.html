One idea might be to see if it is possible to 
make "make -j" distribute processes between
computers in a cluster. This could (I say
"could" because network overhead may be
large) bring down the compile time for
large compiles.

I am lucky enough to use Linux at work and
for the current project we have a build time
of about 35 minutes (PII 450MHz/256MByte).
We would be very interested to see if it would
be possible to reduce compile times using
a cluster since we already have about 15 PII450
work stations running just X and emacs most
of the time.
