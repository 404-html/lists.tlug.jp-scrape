    jb> What's config.cache?

A record of all the information that the configure script learns about
your system so that it doesn't have to check directly the next time.
What gets cached depends on the specific configure script; you would
have to check that directly yourself.

It is, of course, a very bad practice to cache things specified by
explicit user option.  On the other hand, it saves a few keystrokes
for the lazy, and a lot of developers do that kind of thing.
