    John> I updated libc6. It compiled with no problems. After
    John> updating the following packages:

    John> libc6, libc6-dev, gcc, cpp, apt, g++,
    John> libstdc++2.9-glibc2.1-dev, libstdc++2.9-glibc2.1

Ok, I looked at your log file. In there was:

warning "This code is tested with gcc 2.7.2.x only. Using egcs/gcc 2.8.x needs"
warning "additional patches that have not been sufficiently tested to include by"
warning "default."
warning "See http://www.suse.de/~florian/kernel+egcs.html for more information"
error "Remove  this if you have applied the gcc 2.8/egcs patches and
wish to use them"

So, in upgrading the compiler (gcc) you went from gcc 2.7.2.3 to
either gcc 2.8.x or egcs.

Do gcc -v and see what you get. If your still have the old compiler
installed, you can use the -V (upper case) to select which version you 
want to use. Try "gcc -v -V 2.7.2.3" You should see something like:

gcc -v -V 2.7.2.3
Reading specs from /usr/lib/gcc-lib/i486-pc-linux-gnu/2.7.2.3/specs
gcc driver version egcs-2.91.57 19980901 (egcs-1.1 release) executing
gcc version 2.7.2.3

If you don't have it, then you will have to apply the patches
mentioned in the URL above.
