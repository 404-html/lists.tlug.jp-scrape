The standard header to the pages issued by the site's CGI reads:

***

<!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01 Transitional//EN"
           "http://www.w3.org/TR/html40/loose.dtd">
<html>
<head>
<META http-equiv="Content-Type" content="text/html; charset=EUC-JP">
<title>Distributed Translation Project (Distra)</title>
</head>

***

This turns up valid when the site is run through the HTML validator on
http://www.htmlhelp.com/tools/validator, and as far as I know this should
specify the EUC-JP character set for the benefit of the garden variety Web
browser.  The Lynx on my laptop shows:
