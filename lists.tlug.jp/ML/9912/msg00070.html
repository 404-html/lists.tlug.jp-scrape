Hello.

I've installed Mandrake 6.0 and am trying to restore what I had
working on my RedHat 5.2 installation. This time it's procmail.

A user mailmaster has .fetchmailrc, .procmailrc and .forward files in
his home directory. The latter contains

"|IFS=' '&&exec /usr/bin/procmail ||exit 75 #vik"

Any mail sent to mailmaster gets bounced with the following errors:

>>>>>>>>start

   ----- The following addresses had permanent fatal errors -----
"|IFS=' '&&exec /usr/bin/procmail ||exit 75 #vik"
    (expanded from: mailmaster)

   ----- Transcript of session follows -----
sh: cannot use & in command
554 "|IFS=' '&&exec /usr/bin/procmail ||exit 75 #vik"... Service
unavailable

>>>>>>>>end

If I change the .forward to contain

"|/usr/bin/procmail"

or remove it completely and add 

mda /usr/bin/procmail

to the .fetchmailrc, I get the following:

>>>>>>>>start

   ----- The following addresses had permanent fatal errors -----
"|/usr/bin/procmail"
    (expanded from: mailmaster)

   ----- Transcript of session follows -----
sh: procmail not available for sendmail programs
554 "|/usr/bin/procmail"... Service unavailable

>>>>>>>>end

I think it has something to do with sendmail restricted shell. I had a
similar problem when setting up majordomo at the office, and I solved
it by making a symblink to the majordomo executable in
/etc/smrsh/. This trick didn't help me with the current problem
however.

Still trying to live without learning sendmail...

Any advise will be greatly appreciated.
