laszlo@example.com wrote:

laszlo> Is there an easy way of converting a Japanese document 
laszlo> (text or html) to something (gif/jpeg?) that would be 
laszlo> readable by someone who doesn't have a Japanese-capable 
laszlo> environment (I suspect the recipient has a Mac that 
laszlo> doesn't do Japanese). 

If those documents are available through WWW, use Delegate.
Delegate (available from www.delegate.org) is a multi-purpose
network service gateway which has Japanese-character-to-XPM
conversion mode when run as HTTP proxy server.

This service was widely used in early days of the Web, when
NCSA Mosaic still wasn't localized to support Japanese.

So

  1. Set up Delegate as public HTTP proxy, with J-to-XPM conversion
     mode enabled.

  2. Set proxy host of recipient's browser to use Delegate.

should do. It'll be better if you can set up proxy at recipient's
internal network (or localhost at best), since then there'll be
less overhead for transferring extra bulk of images.

Hope this helps.
