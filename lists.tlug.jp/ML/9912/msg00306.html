Hi All,

I'm running Japanese RedHat 5.2 
with Netscape Communicator.
Communicator has had this really 
irritating problem ever since I
installed (and I have had the
same problem on multi-installs).

The problem is with form input.
Often, when I fill in a form and submit,
extra characters will be added to
the field data, resulting in incorrect
form input. This is especially irritating
when using web-based email for example,
because extra characters will be added onto
the end of the address I am trying to send to,
and so the address will be incorrect and
the mail will bounce. The number of characters
added is usually between 2-3, and the characters
themselves vary. Sometimes double-byte Japanese,
sometimes single byte characters.

Also, when filling out a form, then going to 
another page and returning to the form with
the back button, all of the form fields are
erased. Sometimes with the exception of the 
password field, which is then shown 'naked'
instead of masked by asterisks.

I have been tempted to blame this on poor 
handling of forms by linux Netscape in a 
double-byte environment, but I haven't seen
anyone else on here complaining about it,
so I wonder if it is just me.

Has anyone else had similar problems?

Thanks for any advice,
