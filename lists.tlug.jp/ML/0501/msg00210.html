I'm having trouble getting my printer using CUPS working under KDE in
Debian.

I first used the KDE Print manager/wizard. My printer (HP Deskjet
955c) was identified in the proper port, and CUPS had the driver. But
at the final stage of configuration, I get a not-authorized message
with my userID and password. Tried it again using root, but same
result.

I then re-installed the printer using the process described in the
Debian CUPS mini-howto at

  http://mumford1.dyndns.org/~bs7452/linuxhelp/cups.html

This time the printer setup seems to go through--now when I open up
the Printing Manager in KDE my printer is listed with an icon and identified:

   HP Deskjet 955c
   Type: Local printer
   State: Idle (accepting jobs)
   Location: local
   Description: HPDeskjet955c
   URI: ipp://xps8250:631/printers/deskjet
   Device: parallel:/dev/lp0 
   Model: HP Deskjet 955c footmatic/hpijs (recommended)

   Print system currently used: CUPS (Common UNIX Print System)
   Connected to localhost: 631

Applications such as Mozilla and Openoffice list the printer, but when
I try to print, nothing happens, no error messages, nothing.

Any guesses?
