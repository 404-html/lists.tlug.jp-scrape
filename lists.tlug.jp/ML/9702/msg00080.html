I've been told that Pine will ask you if you want to send to the
"Reply-To" address or to the actual author.  MH will also do this for
you, although it requires a little extra effort.

Simply copy the MH library file "replcomps" (usually in /usr/lib/mh/)
to a file, say "author" either in the MH library or in your MH Mail
directory.  Then apply the following diff

--- Mail/replcomps      Sat Dec 21 15:52:31 1996
+++ Mail/author         Wed Feb 19 04:50:24 1997
@@ -1 +1 @@
-%(lit)%(formataddr %<{reply-to}%?{from}%?{sender}%?{return-path}%>)\
+%(lit)%(formataddr %<{from}%?{sender}%?{return-path}%>)\

When you wish to reply to the author rather than the "reply-to"
address, use "repl -form author <args>".  This will accept the -cc and 
so on options as well so that you can send to any cc'd recipients.

I'm working on a set of patches for MH-E 5.0 (Emacs's interface to
MH), but I haven't decided on the interface yet.  I'm also going to
see if it's possible to do this in the same "replcomps" form without
having a separate file, but the language used in MH components files
is recondite to say the least.

I have no idea whether this technique can be easily applied to xmh or
exmh, but if you favor one of those flavors, maybe it can be done.
