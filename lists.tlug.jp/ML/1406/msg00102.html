Every time I install a new distro I butt heads with scanner
installation. (I have come here for help before.)

I got rid of Ubuntu. Can't stand Unity. At work I now use
Vector Linux with xfce. I like it a lot. No scanner there.

At home on one machine I have Linux Mint 16. Got
everything installed and working on that. On the other
one I put off dumping Ubuntu (12.04) until this weekend.
I installed Linux Mint 17. Using the "void" theme for both
LM 16 and LM 17. Like it! Got the printer (Brother MFC-
6490CW) working easily, as expected. The scanner
would only work to scan with xsane. Simple scan and
xsane preview would not run. "Invalid argument" to
start scan. So I had to scan the whole A3 size every
time and crop with GIMP. Not ideal.

So I gave up and re-installed the distro. This time I went
with the Brother printer/scanner installer that does all the
work for you. And this time no scanner at all.

And, yes, I did manually add the Brother device to the
"/lib/udev/rules.d/40-libsane.rules" file. I also added my
username to the scanner group.

I get "Failed to open device 'brother3:bus4;dev1':
Invalid argument."

Funny thing is, when I run "lsusb" I get "Bus 003
Device 002: ID 04f9:01f2 Brother Industries, Ltd."

and "Bus 004 Device 001: ID 1d6b:0003 Linux
Foundation 3.0 root hub"

standing? Full list here:
