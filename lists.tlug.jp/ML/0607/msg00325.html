>>>>> "Dave" == Dave M G <Dave> writes:

    Dave> Really? Some web sites indicate that ttyUSB* come and go
    Dave> with the pressing of the hotsync button. So much so that I
    Dave> assumed it was the norm.

It could be done that way, but I doubt that any devfs- or udev-based
system would.  USB was designed from the get-go to be hot-plugged, and
it has to work with "passive" devices like memory.  So part of the USB
standard is a way for the device to notify the controller that it's
there.  The controller tells the kernel busdriver, the bus driver
tells udev, and udev creates the device.  This approach makes it much
easier to write the higher-level programs like jpilot.

Somebody really oughtta write a GUI program that shows you a picture
of the bus architecture, and allows you to select devices and detect
traffic between them (possibly filtered).  Sortof a Karpski for the
motherboard.
