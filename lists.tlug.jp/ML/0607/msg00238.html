On Jul 16, 2006, at 20:10, Dave M G wrote:


Zend, however, has gone back to not being able to find the right MySQL
socket. But that's because of this next thing:
You should be able to set the socket name that Zend Studio expects in the php.ini file that Zend Studio is using (which is probably not the one that your standard PHP installation is using). If you installed Zend Studio from source, it could be in /usr/local for example. A page with phpinfo(); on it would tell you in a click by checking the Configuration File: entry of the resulting page. 

mysql.default_socket =/var/lib/mysql/mysql.sock
(Or wherever you are putting the socket with your MySQL configuration file.)
