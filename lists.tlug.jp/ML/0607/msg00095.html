I don't know if anyone else is still following this thread, but at any
rate, Jason, it seems that it isn't necessary to install Japanese
support during installation.  Actually, I think you're better off not
doing so, as it installs a bunch of other things that you probably don't
need.

You can install it, then install Japanese fonts (I chose truetype,
there's another one too that I've forgotten--doing 

yum search fonts-ja 

will give you the choices.

Then, you can go with what's readily available, as Jim does with kinput2
and canna or add that respository of Mr. Singh's that was mentioned in
the thread. (As this is a fresh post, and I didn't save the previous
ones, this email will appear as a new thread--sorry everyone.)

In my tests, I installed Mr. Singh's repository--the link is now on my
page, it's the one for Extras, not the one for Misc. and install 

yum install anthy scim scim-anthy.  

After doing so, putting the lines suggested in my page in your
.bash_profile will work, though you probably have to log out and log in
again.  Note that my page suggests using ja_JP.utf8.  I only got
en_US.utf8 to work in uxterm, while ja_JP.utf8 would work in
gnome-terminal, openoffice, etc.
