In addition to my previous post, I got curious enough to follow this up
a little further. I found the page 

http://soundlinuxtraining.com/portal/mod/book/print.php?id=2330

which explained how to add Mr. Singh's repository to CentOS.  I did so,
and then installed anthy, scim and scim-anthy from his extras repo. 

Upon doing so, I was then successfully able to input with scim in
OpenOffice, Evolution, gnome-terminal and uxterm.  I was also able to
use the LC_CTYPE of en_US.utf8.

So, for Jason, I'm guessing the problem is either the repo he used for
scim (or perhaps tried to install from source) or that he didn't install
Japanese support during intial installation.
