In the discussion about Japanese support in CentOS, Jim gave a list of
the programs in his CentOS install where he'd chosen Japanese support
and the install where he hadn't.

Today, in a fit of boredom, I downloaded and installed the latest
Blag--Blag is based on FC5, and seems to have access to its
repositories.  It comes on one CD, and leaves out many things that we
take for granted on 3 and 4 CD RH and RH based installs.

If one chooses English as the install language, there was no chance to
add other languages.  (It also doesn't give the opportunity to install
development tools, and various other things.  For Office type things, it
offers abiword and gnumeric, but everything seems readily available with
yum).

I added the following 

anthy, scim, scim-anthy (and it pulled in dependencies) and was able to
input Japanese as detailed in my Fedora section.  However, while doing
conversion, it would show dotted squares, which means scim can't find a
font.  (Also, in Abiword, there were no Japanese fonts listed.)

So, going back to Jim's list, I saw something like fonts-ja and did a
yum search for fonts-ja.  It came back with something like
fonts-japanese.  I forget the exact name, but apparently doing yum
search fonts-ja will get the result. 

I installed that, and afterwards, everything was fine--I could input
Japanese in abiword, gnome-terminal and uxterm.  (This was after adding
the GTK_IM_MODULE and the other things I suggest adding to .bash_profile
if one chooses a graphic login.)

So Jason, you might be able to get away with just those few programs.
As both Jim and I wrote last night, you'd have to enable Mr. Singh's 
repository for anthy, scim and scim-anthy.
