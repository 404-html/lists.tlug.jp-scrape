Ok, all kidding aside, my instructions for Fedora should work for
CentOS.

Step one.  During installation, did you install Japanese support?  If
not, go back and do it.  I'm not sure how, unfortunately, I don't use RH
these days, but you'll at least want fonts and locales.

Next, if you are using scim-anthy, have you disabled canna or is it
still running?

pgrep canna

If you get back a number, it means it's running, so stop it.  :)

Next, what terminal are you using.  Some of them don't support various
input methods.

Do you have the recommended lines in your .bash_profile?
export XMODIFIERS='@example.com=SCIM'
export GTK_IM_MODULE="scim"
export QT_IM_MODULE="scim"
export LC_CTYPE=ja_JP.utf8
scim -d

If the answer to all of that is yes, what terminal are you using?  What
programs are you using to try to input Japanese?  

Try with uxterm.  From any terminal type,
uxterm

When you get the terminal, make sure that things are correct

echo $LC_CTYPE
echo $XMODIFIERS

make sure that the results are ja_JP.utf8 for $LC_CTYPE and im=SCIM for
$XMODIFIERS.

Now, hit ctl+space.  In that little thingie on the right, is one of the
choices anthy?
