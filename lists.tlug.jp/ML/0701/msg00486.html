The average box has always had a CPU that was too fast for the supplied
RAM (CPU specs sell machine not RAM).  If you want a Celcceron III to
run an office application you need to put enough RAM in it so that your
office application isn't using swap.  One of the tricks used by
Microsoft applications to appear more efficient/responsive (i.e. fast
boot times, the appearance of low memory consumption) is to preload a
lot when the OS boots.  If you do this in a Linux system with adequate
RAM you will get similar results.
