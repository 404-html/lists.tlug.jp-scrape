Dave Gutteridge writes:

 > The kernel header file '/lib/modules/2.6.17.7/build/include/linux/version.h'
 > does not exist.

What does ls -l /lib/modules/2.6.17.7/build/include/linux/version.h
tell you?

 > The most likely reason for this is that the kernel source files in
 > '/lib/modules/2.6.17.7/build' have not been configured.

1.  Most distros have either a CVS repo or a semi- or non-public
archive server with a complete history of packages.  If they don't
have the binary package, they'll probably have a source package.  The
header packages are probably source-only.

2.  You can definitely get old headers at ftp.kernel.org (ie, the
whole source tarball).  You can read the distro's diff for the oldest
header package you can find to see if it touches any files you're
interested in (ie, named .h *and* related to modules), and see if it
applies to the old headers.

There must be other ways.
