However, I went back to jdk-1.1.5v1.2.  I couldn't get the
swingset 0.7 demo to work properly, I I downgraded.  What I 
do is in /usr/local/lib/  I have this:

cow:/usr/local/lib$   ls -ld jdk*
lrwxrwxrwx   1 root     root            8 Mar  9 22:17 jdk -> jdk1.1.5/
drwxr-xr-x   6 root     disk         1024 Aug 24  1997 jdk1.1.3/
drwxr-xr-x   5 root     root         1024 Dec 23 07:48 jdk1.1.5/
drwxr-xr-x   5 500      500          1024 Feb 26 03:27 jdk1.1.5-v3glibc/

I just move my symbolic links around.  My classpath can executable
path variables remain the same.

If the jdk1.1.5v1.2 works, you might want to stick with that.
I've heard that there is a memory leak in that version, but
as my main uses of Java are for testing and development, it
hasn't affected me.

Incidently, on the topic of Java, those of you that haven't
upgraded to Communicator 4.04 Java AWT 1.1 preview release 2
for Linux, should give it a try.  The JDK 1.1 support is pretty
good.  I was even able to view the python applets at python.org.
