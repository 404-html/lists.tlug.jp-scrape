Hi Dave,

dave> So I'll ask again: has anyone ever seen a case where a perfectly
dave> ordinary generic mouse (in fact 3 or 4 of them, different kinds)
dave> refused entirely to be recognized by XFree86?  If so, were you using
dave> a DIN-to-DB9 adapter at the time?

This happens to me sometimes when I'm working on a new machine.  It 
usually turns out to be an improper device being linked to /dev/mouse 
(ie psaux instead of ttyS0) or an IRQ conflict with a serial mouse.

You don't have to load X to test out mouse drivers, you can use gpm on
the console (it's faster).   See which types are available by typing gpm
-t help.  By playing with this and the driver links you should be able to
find a combination which works.  Then make sure this matches your
XF86Config file settings. 

If this doesn't work, try switching the IRQ settings for your serial
ports (assuming you are using a serial mouse).
