On Thu, 5 Mar 1998, Scott Stone wrote:

> Trying to access any Java applet gives me, "JavaLang Security Exception -
> system classes are not signed", or something to that affect.  How do I fix
> this (it's communicator 4.04, Linux)

Do 

  $ export CLASSPATH=/usr/local/netscape/java/classes/java40.jar
  

Your Netscape JVM is loading the classes from your Sun JDK
first.  You have to use the classes that come with communicator.
The java40.jar class set will load the rest of the classes, so that
is the only one you need in your path.
