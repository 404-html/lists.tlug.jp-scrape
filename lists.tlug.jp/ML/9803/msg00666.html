Sorry, I should have said that, "andy" is the user. Its the unix user
under which files will be checked in and out. The repository itself
can (should ?) be owned by some other user, say cvsadmin. CVS just
uses Unix file permissioning, so the users that will use CVS will need
to have read/write permission to the repository.

Although I have not used it, the latest versions ( from 1.9.1? ) no
longer use RCS commands. It implements function to manipulate RCS
files internally. See http://www.cyclic.com/cyclic-pages/cvsdev.html
for more info. 

I am currently playing with jCVS, which is a Java front end to
CVS. Seems a little shaky right now, but this could due to operator
error ( me ). See http://www.ice.com/cvs/
