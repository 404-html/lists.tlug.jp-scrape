On Wed, Mar 04, 1998 at 08:34:55PM +0900, Craig Oda wrote:
> Has anyone ever seen this kind of problem?  Anyone have a
> clue what I can do to get the 3-button mice to work?  (BTW,
> to those who might misunderstand the problem and tell me
> how to get a 2-button mouse to emulate the 3rd button, thanks,
> but I know that already :^)

I've just recently run into the same problem with the machine I am using
here in Singapore.  I discovered that GPM *could* recognize the third
button, while XF86 could not.  Reading through the GPM source, I learned
that there have been a couple of different extensions to the Microsoft
serial mouse protocol to report the third button.  The one my mouse uses
(according to the test program that is supplied with GPM) is one in which
the third button is reported as a mouse motion packet with Dx==Dy==0. It
looks like this format is not supported in th standard XF86 3.1 distribution
I've not tried hacking it in yet, but the GPM code seems easy enough to
steal.

You might check if GPM recognizes the middle button... which will eliminate
hardware questions from your tracking the problem down.
