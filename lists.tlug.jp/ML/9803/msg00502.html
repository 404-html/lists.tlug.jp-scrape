How about dividing the problem into two steps?  First, the install
program writes to a common log? history? file what new menu choices
are possible, with information as to the program name, functions, etc.
Second, add functionality to every window manager to customize menus.
It would start by showing the log? history? file entries.  Then the
user would choose where to put the new entry.  Several choices I can
think of are nowhere (command line only), in a menu, as part of the
automatic startup (and then as icon or full blown), or any other
condition that a windowed program can be in.

I don't even like installs to automatically mess with my info/dir
file; I can't imagine something automatically reconfiguring my window
manager properly :-)
