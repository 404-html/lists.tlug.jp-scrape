Josh Glover wrote:

> A nice tool for just this situation is tee(1).
> 
> strace xmms 2>&1 | tee xmms.log

If the problem is causing lockups hard enough to require reboots, 
then the output might not make to the hard drive. 
It's worth a try anyway. 

Other things to try would be to ssh into the troubled box, 
so that when the troubled box crashes, the stdout still 
appears on the other box (if it ever got sent before the crash). 

Some folks use RS-232 ports for debugging nasty lockups.
