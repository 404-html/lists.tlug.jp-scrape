Anybody got a lead on this one?

I just switched my RH 6 box at work to TL 4.0 (aside: doing that sure is easy
when you have /home, /var/spool/mail, and /usr/local all on separate file
systems  :-)   ) and after the reinstallation, I get this when I try to print
a file:

lpr: connect: Connection refused
jobs queued, but cannot start daemon.

This machine has two printers:  one is a remote printer on another host (my
machine is in /etc/hosts.lpd on that host) and the other is locally attached.
They are, respectively, set as lp and lp0, and both show the same problem.

It feels like permissions, but which ones?  Setting lpr to be world excutable
doesn't help, and lpr already is world-executable:

-r-sr-sr-x   1 root     root        48048 Jul 19 21:55 /usr/sbin/lpd*
-r-sr-sr-x   1 root     root        19760 Jul 19 21:56 /usr/bin/lpr*

Others:

drwxrwxr-x   4 root     daemon       1024 Jul 19 21:38 lpd/
   drwxr-xr-x   2 root     lp           1024 Jul 19 22:13 lp/
   drwxr-xr-x   2 root     lp           1024 Jul 19 22:02 lp0/


These seem OK, but . . . ?
