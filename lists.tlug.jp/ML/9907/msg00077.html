When I missed a /etc/printcap, I had the same problem.   Try this minimum 
printcap.
/etc/printcap-------------------------
#
# Please don't edit this file directly unless you know what you are doing!
# Be warned that the control-panel printtool requires a very strict format!
# Look at the printcap(5) man page for more info.
#
# This file can be edited with the printtool in the control-panel.


##PRINTTOOL3## LOCAL lips3 300x300 a4 {} CanonLIPS Default 0 
lp:\
        :sd=/var/spool/lpd/lp:\
        :mx#0:\
        :sh:\
        :lp=/dev/lp0:\
        :if=/var/spool/lpd/lp/filter:
-------------------------------------------------

> 
> I just switched my RH 6 box at work to TL 4.0 (aside: doing that sure is easy

> systems  :-)   ) and after the reinstallation, I get this when I try to print
> a file:
> 
> lpr: connect: Connection refused
> jobs queued, but cannot start daemon.
>

>
