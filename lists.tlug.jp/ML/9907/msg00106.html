** N505 - An Installation Odyssey, Episode II

Ok, I got Linux installed on that beast, but the next
problem already occured during the reboot directly after the 
install:

  VFS: Cannot open root device 00:30
  Kernel panic: VFS: Unable to mount root fs on 00:30

Not a pretty sight!  After some experimenting, I found that
when I enter 

  linux root=/dev/hda2

at the boot prompt, the system comes up.  The strange thing
is that I have `root=/dev/hda2' in lilo.conf; I also tried
re-running lilo - to no effect.

So far, I couldn't find anything on the net that helped me
figuring out what is wrong.  Any ideas, why the boot loader
is ignoring the setting in lilo.conf?

If I understand the device encoding correctly, 00:30 is
/dev/hda (not /dev/hda2).  Interestingly, when I run

  lilo -v -v -v

it says that it added `linux' with "ro root=302", which
seems correct to me.  How can the boot loader ignore this?
(I have lilo 0.21, which seems to be the latest version.)

Any ideas?

Manuel

PS: I also have trouble with PCMCIA...about everything in this
    machine seems to be a little different from how it is
    supposed to be.  (Apart from the trouble with Linux,
    however, it seems to be a nifty little machine.  But
    then, what is the best hardware worth if I cannot run
    Linux on it *sigh*)
