OK, now we know what we're dealing with.  If it's an NEC old enough to be
running DOS, it definitely isn't something standard.  Well, not outside of
Japan.  It's a PC98-series NEC, which means it's running an NEC-specific
version of DOS.  Standard DOS won't run on an NEC 98, and vice versa for NEC
DOS (won't run on standard PCs).

This thing is probably no more than a 386 (even a 286??) since you don't
mention that it even has any version of Windows on it.  In short, even if it
were a standard PC, it's probably too old to bother with at all. 

> I thought it was something standard, or sort of. Maybe that's the reason.
>I'll look for the port, even because, it could be for me the first
>oportunity to run Linux a on non Intel machine. 

It is an Intel machine, it's just not a fully PC-compatible Intel machine, due
to hardware differences in NEC's version.  NEC has now abandoned that
architecture in favor of building standard PCs.

Unless you just really, really want to get Linux running on an incredibly old
NEC machine, it might not be worth pursuing this.
