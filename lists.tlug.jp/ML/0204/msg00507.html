Hi everyone,

Firstly, I have to thank Baba-san for letting me test his wireless lan
card. It worked fine on my RH7.2 system after some changes to
/etc/pcmcia/config.

Secondly, I tried searching for a PCMCIA bridge adapter that is supported
by linux, but ended up having to buy an NEC access point. It was right
after that that I saw one at the corner shop behind "DOSV Paradise".

If anyone else is planning to get one, try here first. I cant remember the
name of the shop, but the card was on the 2nd floor.


Now for the ugly part. Everything works fine until I start to transfer
large files. eth0 times out and I get these error messages in the logs:

kernel: wvlan_cs: Initialization failed!
kernel: NETDEV WATCHDOG: eth0: transmit timed out
kernel: wvlan_cs: eth0 Tx timed out! Resetting card
kernel: wvlan_cs: MAC address on eth0 is 00 40 26 de b3 d1
kernel: wvlan_cs: Unrecognized card, card return vendor = 0x0000, please
report...
kernel: wvlan_cs: Found firmware 0x10000 (vendor 0) - Firmware
capabilities: 0-0-0-0-0


The network connection completely dies until I restart the pcmcia
service.

Does anyone know what this means? The system is RH7.2 with a 2.4.9
kernel. I added these lines to /etc/pcmcia/config:

card "BUFFALO WLI-PCM-S11 for AirStation R01"
	version "BUFFALO", "WLI-PCM-S11"
	bind "wvlan_cs"

and these to /etc/pcmcia/config.opts (same errors without these lines):

module "wvlan_cs" opts "channel=14"


Thank you,

Sajjad
