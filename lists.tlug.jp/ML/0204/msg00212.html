On Thu, Apr 11, 2002 at 09:41:14AM +0900, Michael Doughty wrote:

> Apparently NetBSD thinks that if a DHCP server isn't going to tell you which
> domain name to use, it has no business telling you where DNS servers are.

That's perfectly reasonable.  What it really means is that if a DHCP server
won't tell you which zones the DNS servers are authoritative for, the DHCP
server is misconfigured and therefore its DNS information is to be disregarded.

Unless one has foreknowlege of the domain searchlist, the supplied DNS
information is only going to be useful for reverse lookups.
