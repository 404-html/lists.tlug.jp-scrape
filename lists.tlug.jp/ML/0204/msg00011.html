Frank Bennett <bennett@example.com> wrote:

> 
> Is there a toggle in the python unicode object that will just
> drop non-conforming characters on the floor?  Failing that, is
> there __any__ filter that will strip out these blocking characters
> from a file, so that it can be run through these tools without
> blowing them up?
> 

There is a library and utility called jconv which is maintained by Kondara MNU/Linux and used by the Sylpheed mail client for Japanese character code conversion.  It is basically a wrapper for iconv for Japanese character code conversions.  They may have solved this problem to some extent.  Might be worth looking at, at least.
