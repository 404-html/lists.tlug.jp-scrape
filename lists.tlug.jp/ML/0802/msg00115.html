On Sun, February 24, 2008 00:37, Lyle H Saxon wrote:
> I often have to convert Visio image files (usually line drawings,
> sometimes photos with call-outs) into .png.  As it is, I have to open each
> and every bloody file to convert it by saving it as .png.  Is there a way
> to batch convert these?  I'm stuck with a bloody w-box at work, but there
> are test machines available at work that could be set up with Linux for
> the job....

Ah.  Visio.  A fine example of "who owns your data?" (Answer: it's not you).

How about a virtual printer that can "print" to a PNG file?  I thought I
was crazy when I first considered it but there is a product:

<http://www.componentsource.co.jp/enikk/products/gnostice-print2edoc-application/index.html>

Maybe you can do the same thing and create a PNG printer on Linux and
share it with Samba.  Then, in Windows, highlight all of the files you
want to print, right click and select print.  This uses DDE to load each
file with its associated app and print with the default settings to the
default printer.  If you select multiple files, multiple files will open
and print, then close themselves.

Have fun!

Andrew
