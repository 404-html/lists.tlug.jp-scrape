>>>>> "Andrew" == Andrew S Howell <andy@example.com> writes:

    Andrew> What about other Kanji fonts? Can I buy commercial fonts
    Andrew> and use them with Ghostscript?

Probably.  However, encodings are not very well standardized; you may
not be able to use either the JIS encoding or the EUC encoding
provided by Wadalab.  Check that before you buy.  Ghostscript does
know how to use standard encrypted Type 1 and Type 0 fonts, so you
shouldn't have any problem displaying them.  Getting the accessing
right (encoding) may be more difficult, or it may be trivial.  Depends 
on the vendor.

    Andrew> Are there other public domain kanji fonts?

Not that I know of.  There are tools for making (ugly) scalable type 1
fonts out of bitmaps, so if there's a X11 bitmap font you like, you
could probably make one that wouldn't look too bad at some
resolutions.  You could check and see if there are PDS kanji fonts for 
TeX's Metafont; there might be a tool for converting those to PS.

If you own Windowze, Ghostscript can handle TrueType[tm] fonts.  I
don't think that MS's license requires you to actually be *running*
Windowze, only *owning* it.  (I use Adobe's fonts under my DESQview/X
license; that license says nothing about running DV/X, just that DV/X
must be loaded on the machine ;-)  I know nothing about encoding issues,
however.  I believe that you also need to add a Type 42 font as a shim 
between the TT font and any Postscript program; I don't know how hard
that it.
