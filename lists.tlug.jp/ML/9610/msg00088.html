In message <199610161700.RAA16562@example.com> you wrote:
> How do I get it to start accepting romanji for conversion to Kanji?

I have to check the documentation I have at home, but I can tell you
how it is working in the machine I'm using here.

In the /usr/X11R6/lib/X11/app-defaults/Kinput2:

	!!
	!!      Wnn setup
	!!
	*CcWnn.Jserver:         wani.dohilab.dnj.ynu.ac.jp
	*CcWnn.jserver2nd:      kani.dohilab.dnj.ynu.ac.jp
	*CcWnn.ccdef:           ccdef.kinput2
	*CcWnn.WnnEnvrc:        /usr/local/lib/wnn/ja_JP/wnnenvrc
	*CcWnn.WnnOverrideEnv:  true

where "wani" is the primary jserver, and kani is the secondary.

Later, in the same file:

	!!
	!! conversion protocol dependent resources
	!!
	! KinputProtocol
	*KinputProtocol.BackwardCompatible: true
	*KinputProtocol.XlcConversionStartKey:  Shift<Key>space
	! XimpProtocol
	*XimpProtocol.ForceDefaultServer: true
	! X Input Method Protocol
	*IMProtocol.locales: ja_JP.SJIS, ja_JP.EUC, ja_JP, japanese, japan, ja
	! common to XimpProtocol and IMProtocol
	*defaultFontList: -misc-fixed-medium-r-normal--14-*-*-*-c-*
	*ConversionStartKeys: \
	        Shift<Key>space \n\
	        Ctrl<Key>Kanji

With this, kinput2 is set up to wake up when pressing shift+space.

These two were the only settings I had to do manually, after having
installed JE-0.9.7z.
