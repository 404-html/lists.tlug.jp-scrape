Someone sent me this lisp to get emacs to work with Japanese in shell
mode. It works for other stuff, but not xjdic. It looks like xjdic is
sending out control characters to the "kterm", which emacs does not
know how to interpret. Note that the following is for mule19.33gamma,
though it may work with other mules.

;;;
;;; set sh-mode to use EUC
;;;
(add-hook 'shell-mode-hook
	  '(lambda () (set-current-process-coding-system
		       'coding-system-euc-japan
		       'coding-system-euc-japan)))
