>>>>> "Dennis" == Dennis McMurchy <denismcm@example.com> writes:

    Dennis>   I am using "standard" Pine 3.91, too, right off the
    Dennis> Slackware CDROM (Infomagic 4/96).  To get the
    Dennis> "ISO-2022-JP" header to display when I have Japanese in
    Dennis> the text I changed the following line in .pinerc:

    Dennis> # Reflects capabilities of the display you have.
    Dennis> # Default: US-ASCII.  Typical alternatives include 
    Dennis> # ISO-8859-x, (x Dennis> is a number between 1 and 9).
    Dennis> character-set=iso-2022-jp

Thanks, I didn't notice it on the previous pass (and probably wouldn't 
have thought to try it, since I don't think of the character set being 
sent as being necessarily related to the capability of my display).

Anyway, it looks like Pine implements the MIME standards pretty well,
even for cases where it knows nothing about the encoding in question.
(I think that the extended character set mentioned in "character-set"
is going to be a good heuristic for unrecognized non-ASCII text in
most settings.)
