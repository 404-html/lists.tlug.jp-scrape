On Sun, 13 Oct 1996, Shige Abe wrote:

sabe>> 	Does anyone know how I can get vi on linux to work with X's
sabe>> cut and paste. Its driving me mad. If I try to cut or paste anything,
sabe>> all I get is a bell, and it pops me out of insert mode.

I have elvis, vim, and jvim working on a kterm.  Only
vim and jvim work on an xterm.  
I remember that I had this cut and paste  problem with an installation of 
slackware.  I'm not sure what the symbolic link in
/usr/bin was, but I think that it was elvis.  If you are
not desperate for RAM, do this:

niji:/usr/bin# ls -l vi
lrwxrwxrwx   1 root     root            5 Mar 24  1996 vi -> elvis*
niji:/usr/bin# 

to make sure it is a sym link.

  # rm vi
  # ln -s vim vi

Or, link it to jvim if you use Japanese a lot.

I just tried vim with an xterm and I was able to cut and paste
under X.


I'm wondering what everyone is using as their default terminal
shell.  I've used rxvt, pxvt, kterm, xterm, and color_xterm.
I'm using kterm now.  I found pxvt convenient on a laptop
due to its lower RAM usage.
