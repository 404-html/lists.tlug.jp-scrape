[tlug] [RFC] Proposed method for TAGGING subjects

I likewise have given considerable thought to this issue, and the points
raised by Keith, Scott, and CL (et al.) The main ones seem to be.

* subject line length too long
- limit the prefix tags to type and level tags and embed other tags in
the flow of the subject content.

* difficult to finding the appropriate tag
- solution would be to not use a category tag if you don't know or want
to check which ones are being used.


The proposed layout.

[type tag][level tag] the current subject line with embedded [category tags]

Some examples from current messages.

[RFC] Which [IME] does everyone use
[Q][C&C] How to use [FIREFOX] shortcuts
[Q] Mac OS 9.x and [CUPS] Network Laser [PRINTER] suggestions
[A] Mac OS 9.x and [CUPS] Network Laser [PRINTER] suggestions
[Q] using [MUTT] in a [BASH] script
[RFC] Adoption of Linux
[FYI] My first keitai in Japan
[FYI] Endeavor LX1000 [PRINTER] w/Linux Y60,690
[RFC] [LEGAL] issues now Apple owns CUPS
[ANN] Emergency nomikai August 17th
[Q] Fixing [EMACS] cperl-mode
[FYI][HW] giveaway
[Q] two [SSH] server on one IP


Type tags

Tags expressing the intent of the message.

[Q] asking a question
[SOLUTION] explaining how the submitter of the question was resolved the
issue
[SUMMARY] summarizing what has been discussed so fare
[RFC] requesting feedback on an issue
[FYI] informational post, possible without initial context
[ANN] an announcement
[C&C] contains a joke
[OT] material outside the scope of this mailing list.


Level tags

Tags indication the level the message is aimed at.

[NOOB] a new inexperienced user
[EXPERT] an advanced user


Category tags

Tags indicating topic categories

[HW] relating to hardware
-[PRINTER]

[IME] relating to input methods
-[SCIM]
-[UIM]
-[ATOK]

[ED] discussion relating to editors
-[EMACS] discussion relating to emacs, xemacs
-[VI] discussion relating to vi

[FIREFOX] discussion relating to firefox browser.
[MUTT] discussion relating to mutt
[SSH] discussion relating to ssh

[P-CODE] relating to programming in general
- [RUBY] relating to ruby programming language
- [PYTHON] relating to python programming language
- [PERL] relating to perl programming language
- [BASH] relating to bash programming language

[LEGAL] relating to general legal discussion
- [COPYRIGHT] discussion relating to copyright in general
* [CC] discussion relating to creative commons license
* [GPL] discussion relating to GPL license
-[PATENT] discussion relating to patents
