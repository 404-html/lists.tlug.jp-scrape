> >    I am using fetchmail as root, to collect the mail for all users on my
> >    system. The cronjob entry is as follows:
> >    
> >    fetchmail -a -K -s -l 50000 -w 1800 -t 90
> >    
> >    The limit and warning flags, do not notify the root or the individual users
> >    of an oversized mail still sitting on the ISP's POP server.

> 
> Why not run in the daemon mode instead of invoking as a cronjob? 
> My understanding is that fetchmail "mails" warnings only if invoked in
> the daemon mode. With the above commandline, the progress
> message on *stdout* may indicate the oversized messages and  crond (and 
> not fetcmail) should have mailed this output to the cronjob owner unless
> redirected elsewhere.
> 

Thanks for the suggestion. I will try that. It does make sense, because when I
ran fetchmail as a daemon for each user then I did get the email warning about
the oversized emails. The cron job does not report this problem.
