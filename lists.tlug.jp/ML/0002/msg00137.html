>>>>> "Dave" == Dave  <ddugay@example.com> writes:

    Dave> ifconfig eth0 192.168.1.10
    Dave> route add default gw 192.168.1.1

Looks OK.

    Dave> route -F tells me this:
    Dave> destination / gateway / genmask / flags / metric / ref / use / iface
    Dave> 192.168.1.1 /   *      / 255.255.255.0 / u /  0 /  0  /  0   / eth 0

This is a little weird.  Usually this automatic route ends up being
"localnet", ie 192.168.1.0.  Maybe this is due to the router?  The
router doesn't act as a hub, you actually have to use it as a gateway
to the other PCs on the LAN?  If it can act as a hub, you should try
`route add -net 192.168.1.0 dev eth0' and see if you can ping other
boxes on the local net.

    Dave> 127.0.0.1  /    *      / 255.0.0.0     / u /  0 /  0  /  0   / lo
    Dave> default / 192.168.1.1 / 0.0.0.0 /  ug    /  0   /   0  /  0  / eth0

    Dave> ifconfig looks like this:

ifconfig looks OK, except ...

           interrupt:18  base address:0xe800

You really have an IRQ 18?  Is that because of SMP?

    Dave> My LAN starts with an NTT ISDN router (ipmate 1300 rd).  The
    Dave> router is currently set to assign IP addresses automatically
    Dave> (dynamically) via DHCP.  The router IP address is
    Dave> 192.168.1.1

Are you sure nothing else has that 192.168.1.10 address?
