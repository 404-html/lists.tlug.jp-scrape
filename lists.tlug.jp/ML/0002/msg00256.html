On Wed, 23 Feb 2000, Dave wrote:

>  Im having major problems with my NIC and configuration.  Until I can
>  figure this out, my PC is much more valuable to me as a windows
>  machine.  The problem is that fdisk wont clear out the whole hard drive.
>  
>  I have a 24 MB linux partition and an extended dos partition which is 8
>  GB (at least thats what fdisk reports).  The odd thing is there are no
>  logical drives defined on the extended partition.  fdisk wont let me

To me this means it is not a dos partition. 

>  delete the extended partition until the logical drives are deleted, but
>  since no logical drives are defined, I cant delete them.
>  
>  Do I delete everything via Linux, or can it be done with DOS utilities?

Boot linux from floppy ( or a bootable installation CD) and run linux
fdisk to delete all partitions. Then reboot dos and create new 
partitions. 

If you want avoid booting linux, use partition magic. 

On Wed, 23 Feb 2000, Shimpei wrote:

>  Is this the DOS fdisk, or the Linux fdisk? The last time I messed
>  with disk partitioning, the smart thing to do was to use the Linux
>  fdisk to mess with Linux partitions, and use the DOS fdisk for DOS
>  partitions. 

Don't have to worry about that if you are only deleting all the
partitions ;) But, yes, while making partitions better use dos fdisk for
dos partitions.
