I build my kernels with CONFIG_PARPORT=y and CONFIG_PARPORT_PC=y (i.e.,
built into my kernel rather than as modules) and then compile the PPA code
as a module.  I note that if I don't apply power to my ZIP drive before
doing 'modprobe ppa' I will get the message:
/lib/modules/2.2.14/scsi/ppa.o: init_module: Device or resource busy

Jim

P.S. Do you have more than one parallel port in the machine?  Or is it the
port an unusual address or IRQ?  You may need to specify additional options
to specify the IO address and/or irq to use parport as a module.
