I'm assuming that the above message occurs because another 
machine on the network has mounted a nfs file export on 
this machine...and that it is nothing to worry about (?).

of another machine on it (the reverse of the above case), 
I get: 

Unmounting remote filesystems. cannot MOUNTPROG RPC:

and, a few second later: 

rpc.mountd forgot to set AF_INET in UDP sendmsg. Fix it! 

I am assuming that this is also a result of shutting down 
without unmounting the remote filesystems and is nothing 
to worry about (?).
