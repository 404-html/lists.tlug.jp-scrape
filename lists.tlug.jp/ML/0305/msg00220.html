Quoth Botond Botyanszki (Sun 2003-05-18 11:48:33PM +0900):

> I have been googling around and looked into various docs but haven't yet
> found a solution.
> I have some cron jobs installed into /etc/cron.d. Some of these run pretty
> frequently, and the following syslogs by pam are getting annoying. Is
> there a way to disable these? Of course I only want to disable the cron
> related ones. 
> 
> May 18 23:00:01 localhost cron(pam_unix)[1071]: session opened for user
> mail by (uid=0)
> May 18 23:00:01 localhost cron(pam_unix)[1072]: session opened for user
> root by (uid=0)

One solution would be to switch to syslog-ng and add a line in your config
file that would throw away the offending entries. 

You can grab syslog-ng from here:

http://www.balabit.com/downloads/syslog-ng/1.6/src/syslog-ng-1.6.0rc3.tar.gz

My own syslog-ng Quick-N-Dirty Guide(TM) might be of some use, as well:

http://www.jmglov.net/docs/qnd-syslog-ng
