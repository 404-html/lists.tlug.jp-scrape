>One other problem I have noticed is the slow startup of Apache at boot up
time.
>...
>The system waits for about 5 minutes before moving on to the next
instruction.

It could be getting caught on a DNS lookup. Do you have any virtual hosts
in httpd.conf? Make sure you've not mistyped any domain names.

Also Apache can be started late; make sure bind and the network interface
have been initialized beforehand.
