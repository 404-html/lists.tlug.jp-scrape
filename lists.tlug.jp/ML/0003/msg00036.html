>  Thank you very much for the help and comments, but I can't still solve the
>  problem.
>  I can't get through the installation program.When I'm asked by the installer
>  to define the partitions, I only create a swap partition and then a root (/)
>  partition, but the program always complains about "boot partition too big"
>  and I can't go further (I have not reached yet to the point in which I'm
>  asked if I want to use lilo or boot from a floppy).

What does the linux fdisk report? For example I have

bash# /sbin/fdisk /dev/hda

Command (m for help): p

Disk /dev/hda: 255 heads, 63 sectors, 523 cylinders
Units = cylinders of 16065 * 512 bytes

   Device Boot   Begin    Start      End   Blocks   Id  System
/dev/hda1            1        1       28   224878+  83  Linux native
/dev/hda2           29       29       43   120487+  82  Linux swap
/dev/hda3   *       44       44       63   160650   83  Linux native
/dev/hda4           64       64      523  3694950    5  Extended
/dev/hda5           64       64      191  1028128+  83  Linux native
/dev/hda6          192      192      383  1542208+  83  Linux native
/dev/hda7          384      384      523  1124518+  83  Linux native  


It is possible that at boot up linux has not sensed the correct values
for cylinders, heads and sectors of your disk. Comparing the linux
fdisk o/p with the specs in your hard disk manual or (printed on your
hard disk itself) may help in diagnosing this.
