Please find below the mail from RIKAI.
----- Original Message -----

Rikai is using too much CPU for the shared server it's on. Sadly, after
almost two years of helping people read Japanese, Rikai may need to be
turned off. Can anyone save Rikai?

What Rikai Needs:

Linux/Perl (other unixes should be ok)

> = 1GHZ processor (and it'll use a fair amount of CPU, though it runs
pretty
> well under nice so other non-CPU intensive sites can coexist)

Good enough bandwidth to support a decent size audience (currently approx.
4000 pageviews/day) during peak times (>= 300kb/s?). Keep in mind, if you
view a page through rikai, the server needs to fetch the page and serve a
modified version up, so the bandwidth requirements are effectively twice
that of a normal site serving up content.

I'd prefer an organisation or university over an individual if someone is
going to offer rikai a new home.

Or, if one or more donors could offer up the startup fee for a dedicated
hosting service (e.g., CIHOST costs $99/month + $400 startup) I could then
try to find some friends with static sites to share the $99 monthly. That'd
be most excellent, as it would also enable me to speed up Rikai quite a bit
in the future (e.g., by using modperl which generally isn't available on a
shared server). Also, if there is a similar service to be found for a decent
price here in Japan, of course that'd be better for most Rikai users.

What I can't offer is my time in exchange for the service. Unfortunately, I
have very little to spare. It's been suggested by some that I go to a
subscription model, but honestly I don't have the time to implement it and
work out the licensing, etc. etc. Besides, I'd much rather find a way to
keep Rikai free for the masses.

And please keep in mind that Rikai is purely a hobby for me.

If anyone can offer any other ideas/suggestions, please let me know at
trudick@example.com

cheers, Todd
