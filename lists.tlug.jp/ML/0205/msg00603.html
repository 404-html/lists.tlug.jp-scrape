On Wed, May 08, 2002 at 06:47:29PM +0900, Jean-Christian Imbeault wrote:
> Ok, I've run out of helpful URLs ... I've followed every bit of 
> installation information I can find for getting an XFS filesystem on my 
> machine but with no success ...
Havent looked at their site lately, but the way I installed it was by
first patching the kernel, building the programs, rebooting and making
the file system.

> I tought I had finally done everything right but when it actually came to 
> using 'mkfs' to get an xfs it just crapped out with:
> 
> mkfs.xfs: warning - cannot set blocksize on block device /dev/hda11: 
> Input/output error

The last I heard, the block size on Linux was fixed at 4KB. Try
'mkfs.xfs --help' and see if you can specify it on the command line.

But before that, what is the partition type of /dev/hda11? It should be
set to 'Linux Native (83)'.

> Any brave/helpful souls out there feeling generous enough to give me some 
> simple step-by-steps instructions?

Try a newer kernel (see below) and check the partition type. That should
work but in case it doesnt, Ill dig out my procedure.

> I don't know if XFS is evolving too quickly or what but even the 
> instructions on SGI's own XFS site's installation instructions don't seem 
> to work (URL to non-existant installer ISO images or commands that don't 
> apply anymore for example).

I dont think the installer caught on, but the patch method seems pretty
straightforward.

> PS I have a RH7.2 machine (kernel 2.4.7-10 is what it installed).

Im running it on 2.4.18, but you should at least upgrade to 2.4.9-31
which is the latest RH kernel for RH7.2. You can get the RPMS at any of
the mirrors, although most of them are currently busy dishing out 7.3.
