I'd like to recommend running rsync thru an encrypted channel using
openssl and stunnel. Make both server and client verify each other's x509
certificate and take special care to secure the certs.  (those can be
self-signed.) The tunnel works beautifully for all TCP traffic except FTP
(which requires two separate TCP sessions).

www.stunnel.org
www.openssl.org


If you don't have to automate the transfer, then you can just run it thru
ssh.
