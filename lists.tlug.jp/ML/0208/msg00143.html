Hatsuhiro-san,

You need to work on your posting technique. Your post is pretty much
of the flavour, "Help! I cannot foo!" This is unlikely to draw much
sympathy from this crowd, and is more likely to land you at the bottom
of people's .procmailrc (with a resounding *plonk*, might I add).

If you are looking for references, that is well and good, but you
should have at least tried to find them yourself. Posting "I need
references for foo, does anyone have a link?" is generally
unacceptable unless you are someone who is well known to the list,
someone that we trust has already tried to find the answer himself
before asking us. A better way would be, "I searched Google for 'foo
bar' in order to find some resources for setting up foo with bar, but
I could not find much. Does anyone know of a good document?"

And in this case, I *know* that this list has already referred you to
http://www.tlug.jp/m17n/. Have you made use of any of the documents
there? I know for a fact (because I have read it several times) that
Scott Robbins's doc on Japanese in Red Hat and Gentoo is easy to read
and will get you setup correctly.

Quoth hatsuhiro (Sat 2002-08-10 02:10:26AM +0900):
> I'm perplexed in configuration and compilation with Kinput2, Canna, and
> FreeWnn.

Again, did you read any of the above docs? Be more specific, what is
perplexing you?

> How can I get help?

By expending a little effort, doing a little Google searching, using
the references that have already been provided you, and *then* posting
to this list, stating your problems very specifically and verbosely.

> Are any referrences for default setting available?

Which default settings?

> Tell me how to redirect all messages on the display with error
> messages to show what's going.

I assume that you mean, "redirect the errors that are showing up on my
terminal to a file so that I can peruse them later"?

foo >my.log 2>&1

'man bash' for more information.

> What is XCMM in configuration files? I cannot find out "-lcanna16" in the
> directory.

Which config files?

'man foo' where 'foo' is whichever program you were running that gave
you an error when you ran it with th -lcanna16 switch.


Sorry to be so harsh, but:

1) I feel that you have earned my wrath.
2) If you heed my words, you will find this list much more helpful to
you.


--Josh
