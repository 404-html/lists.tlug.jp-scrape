Ryan Shaw wrote:

> is there a way to force an application to run singlethreaded on an SMP
> box?

No, but you can force the kernel to only use one processor with the
boot option "nosmp":

grep nosmp /usr/src/linux/Documentation/kernel-parameters.txt
	nosmp		[SMP] Tells an SMP kernel to act as a UP kernel
