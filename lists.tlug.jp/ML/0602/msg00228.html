On Fri, 10 Feb 2006 13:29:27 +0900, Edward Middleton

> It should be [1]65,517 files for a fat16 formated SD card.

Not necessarily.

The word at offset 0x11 in the filesystem bootsector contains the number of
entries allowed in the FAT16/FAT12 filesystem root directory. Typically,
it's 128 for a floppy and 256 or 512 for a HDD or removeable media. This is
handled differently by FAT32.

Note that this is only true for flat directory structures, contained entirely within the root directory - make a directory and you can store more files up to Edward's limit (for FAT16).  (You know, this limitation probably dates back to CP/M...)

From Dave's following message this may well be the problem.  I think the limitation is on file numbers and not file size, though.
