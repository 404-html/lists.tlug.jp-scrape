Dave Gutteridge wrote:
>     Thank you for the command. I have included the results below. It 
> doesn't say anything about aMule on port 80.
>   
That is because amule is connecting to a web site which is on port 80. 
(i.e. port 80 is the remote side).
> amule     11136     dave    9u  IPv4  17501       TCP *:4662 (LISTEN)
>   
You are listening on tcp port 4662 which means it needs to be open in
your firewall rules
> amule     11136     dave   11u  IPv4  21058       TCP 192.168.0.102:59362->83.149.123.189:www (ESTABLISHED)
>   
this says that amule is making an http connection to whatever is at ip
83.149.123.189. This is what is coming up as an "active connection" in
Firestarter but that is not your problem.
