Dear Everyone,

yesterday when I pulled out a 1U server from a rack, the power cable got
caught and consequently pulled out.  I stayed calm, knowing that I use a
journaling file system, namely ReiserFS.  Alas, after powering on again,
Linux got a kernel panic when attempting to mount the root partition.

It was easily fixed running fsck from a gentoo boot disk (which played back
the journal), but I had assumed such a thing to happen automatically. 

/dev/hda1 mounted as boot, ext2, 100MB
/dev/hda2 swapdisk
/dev/hda3 mounted as root, resiserfs, 30G

Any way to ensure a clean restart without manual intervention?
