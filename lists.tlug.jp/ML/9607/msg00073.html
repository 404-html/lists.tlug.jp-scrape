I installed QMAIL on one of the machines in the office last week.
It is a smallish replacement for Sendmail that has some interesting
features.  The author claims that it can be "an order of magnitude"
faster than Sendmail when doing mailing list processing.  More
interesting I think is the way it handles local addresses.  Any
thing addressed to user-{anything} gets routed to "user".  The
user then has the option of handling different addressess differently...
like having an auto-responder when people send mail to jwt-info for
example.

It is very easy to configure, based on a huge number of single function
configuration files.

I found a copy in the ftp.redhat.com:/pub/contrib collection.
