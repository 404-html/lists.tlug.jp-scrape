On 2017-12-25 00:23 +0900 (Mon), kts wrote:

> So you are suggesting continuing debian 8, install Docker containers
> and get a debian 9 VM running....

Note that these are completely orthogonal. VMs have nothing to do with
Docker (they are separate kernels running on virtual hardware) and
Docker containers are not VMs but are simply processes, with lots of
restrictions and namespace remappings, running on an existing kernel.
The Docker daemon uses `chroot` and similar stuff to give an isolated
environment to a process.
