I never had problems with this myself on various Ubuntus from 8.xx.

Check that the kernel detects your mouse correctly, and you don't have some compatibility setting in your BIOS setup:

$ grep  psmouse  /var/log/kern.log
Jan 20 07:51:47 sonic kernel: [    2.802159] psmouse serio1: synaptics: Touchpad model: 1, fw: 8.0, id: 0x1e2b1, caps: 0xd001a3/0x940300/0x120c00, board id: 1611, fw id: 774180
Jan 20 07:51:47 sonic kernel: [    2.802169] psmouse serio1: synaptics: serio: Synaptics pass-through port at isa0060/serio1/input0
Jan 20 07:51:47 sonic kernel: [    6.365181] psmouse serio2: alps: Unknown ALPS touchpad: E7=10 00 64, EC=10 00 64
Jan 20 07:51:47 sonic kernel: [    7.911992] psmouse serio2: trackpoint: IBM TrackPoint firmware: 0x0e, buttons: 3/3

Apart from that:

1. check you don't have any xorg.conf hanging around with "Middle mouse button emulation"  (unlikely I think these days)

2. start a plain X server from the console with 'xinit' or similar and use 'xev' to see if X sees your mouse movements.
Middle button press/release looks like this

ButtonPress event, serial 37, synthetic NO, window 0x4c00001,
    root 0x94, subw 0x0, time 7534470, (174,87), root:(899,428),
    state 0x0, button 2, same_screen YES

ButtonRelease event, serial 37, synthetic NO, window 0x4c00001,
    root 0x94, subw 0x0, time 7534470, (174,87), root:(899,428),
    state 0x200, button 2, same_screen YES

3. Try the 'xev' again from your normal desktop environment.
If the result is different then it must be something in your desktop manager settings  (look in the control panel etc)
