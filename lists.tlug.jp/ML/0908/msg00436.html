Sotaro, Shawn, Curt, Alexi, Martin,

Thank you all for responding.

Seems Amarok only uses utf-8:
https://answers.launchpad.net/ubuntu/+source/amarok/+question/20308

Which should be fine, because Android seems to be exclusively utf-8 as
well. Looking up encoding issues on the web, most people have problems
with tags that are *not* utf-8, like this guy trying to get Chinese
characters to display:
http://www.yuan.se/?p=60

Easytag reads the Japanese text just fine...

I used eyeD3, and piped the results into a text file, which I then
opened in OpenOffice and used utf8 to read it... The Japanese comes out
fine. Although opening it in Gedit doesn't read it so well.

So I *think* everything is in UTF-8.

Any suggestions on where things might not be as they seem?
