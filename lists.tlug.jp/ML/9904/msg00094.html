The problem:
        Every time I send mail from my laptop (Using Eudora Pro v4), relaying through my server d-rave.com, i get the following message:
The recipeint [whoever I'm sending to] is not acceptable to your SMTP server. The message can not be sent until the recipient has been changed"
        I can recieve mail from my dave@example.com account, but i can not send.

What I've done:
        I checked the page http://www.sendmail.org/tips/relaying.html, and then, based on that page's advise, and then on what Chris added to that, I went into my sendmail.cf file and added the following line:
                FR-o /etc/mail/relay-domains
        Then I created a file called /etc/mail/relay-domains, and in it i put one line:
                avant.co.jp RELAY
        Then I asked the computer what the process number of sendmail was with:
                ps ax | grep sendmail
        And then i killed and restarted (?) sendmail with the following command:
                kill -USR1 79
        I tried sending a test e-mail to myself, and I encountered the same error. So I went back to /etc/mail/relay-domains and changed
                avant.co.jp RELAY
        to
                210.152.248 RELAY
        thinking that might be more specific. I then killed and restarted sendmail again, but it seems i still can not relay from where I am.

        Any suggestions?

        Dave
