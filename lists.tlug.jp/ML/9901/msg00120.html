On Wed, 13 Jan 1999, Jonathan Byrne wrote:

> I'm trying to build an RPM from source, and it requires GTK 1.1.2 or
> higher.  I currently have 1.1.9 and 1.0.5 installed (both from RPM).

Playing with gnome, I see ;)

I assume that 1.0.5 is installed because you want to run gimp, yes?[1]  If
the package dependencies are set such that a newer version of gtk breaks
things, then something is seriously messed up.

If so, then all you need is the 1.0.5 shared libraries.  You want to keep
them but eradicate all of its executables and headers.  I'd uninstall both
gtk RPMs, extract the shared libraries from that rpm with rpm2cpio, put
them where ldconfig can find them, then install the 1.1.9 RPM.

-- Chris

[1] gimp doesn't work with gtk-1.1.x (odd, that, as gtk means Gimp Tool
    Kit)
