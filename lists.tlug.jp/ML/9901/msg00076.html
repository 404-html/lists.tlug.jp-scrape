Seems quite doable. The one thing that won't work, I think, is IP to
host name lookup. I don't think this should be too big of a problem
though.

dyndns.com just has a script that you can run to register your IP. I
don't know that it has to be that limited. As I mentioned, my friend
and I just set it up to exchange a zone map by mail. This way I can
put whatever I want in it. To be more secure, we would probably want
to use, say, PGP signed messages. Instead of mail we would probably
want to setup a daemon that sits on some port waiting for updates.

Also, to be nice, we would probably want some type of refresh
mechanism. Without it, if my machine goes away and someone looks at
www.zoomer.org, they may find some other user. The timeout could be
the same as the DHCP refresh interval, which you can get from dchpcd.
