Here are some parallel-processing links I've got in my bookmarks. They
might be of use, as would searching for "pthreads".

http://www.linux-howto.com/LDP/HOWTO/Parallel-Processing-HOWTO-1.html
http://www.mcs.anl.gov/dbpp/text/node4.html
http://www.orl.co.uk/omniORB/doc/omnithread/node1.html

Threads give you the chance to introduce new types of bugs into your
program, like deadlock, so only use threads when you actually have a need
for them.

BTW, the thread implementation in python is easy to use, so useful for
learning.
