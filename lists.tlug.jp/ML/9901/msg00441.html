It _does_ need to be setgid <GID of /var/spool/mail> to write
lockfiles there.  Usual approach is to 

su
chown root.mail /var/spool/mail
chmod 0660 /var/spool/mail
for u in $USERLIST; do
 # if you're not group-per-user, the group here MUST NOT be mail
 chown $u.$u /var/spool/mail/$u
 # YMMV on the group permissions
 chmod 0660 /var/spool/mail/$u
done
exit

so my guess is that either ownership of /var/spool/mail is .root, or
the program doesn't check that and simply blindly says "Gimme root!"
