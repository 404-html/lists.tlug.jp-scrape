Janos,

>> Not that I
>> want to fuel your fire but I wonder what pissed you off so much. It
>> might be good for many of us to know.

I doubt it.

I could go on about how MySQL has no systematic syntax and is just a
phone book of differently structured queries.

But really the issue is me. I'm a visual designer, and I want to be
doing visual design work, not pure coding projects.

Somehow I've ended up here on this one project doing something that has
no visual elements whatsoever. It's all about solving problems like
sorting Japanese characters, and connecting to email servers, and all
sorts of stuff that both bores me to tears and I'm not that good at anyway.

I think to be really good at something, you have to have a passion for
it. I have a passion for design, and I'm good at that.

I have no passion for MySQL. Or PHP. I only got into them so that I
could make more robust web sites, but I never intended to deviate so far
from design as to be doing pure coding. A little bit here and there,
fine. But as the entirety of the project? Blah.

This particular project I'm working on now is really grinding me down.
It's driest coding project I've ever dealt with. When it's done, if I
never had to write another MySQL query again, I would be a happy man.
