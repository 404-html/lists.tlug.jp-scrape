I have been able to print from my NT machine to a printer connected to my
Linux machine for the past year.  I would get periodic errors that said
there was a problem with the printer.  To solve this problem I just
deleted the printer and re-installed it again.  Recently I tried to do
this again, but now when I try to open up the linux machine from NT I get
the following error:

Linux Machine Nameにアクセスできません。
そのアカウントは、このワークステーションからのログインを許可されていません。

Where do I let Samba know that
I want to allow my NT machine to access it?  I didn't do this before and
I had no problem, no all of a sudden there is a problem.  

I have the following entries in smb.conf.  Pretty much everything is a
default install because I was able to use it as is before.

[homes]
   comment = Home Directories
   browseable = no
   writable = yes
   read only = no
   preserve case = yes
   short preserve case = yes
   create mode = 0644

[printers]
   comment = All Printers
   path = /var/spool/samba
   browseable = no
   printable = yes
; Set public = yes to allow user 'guest account' to print
   public = no
   writable = no
   create mode = 0700

Any help would be appreciated.
