Actually I was talking about majordomo. Not individual user.
in my /etc/mail/aliases, I had

vivaldi-list:         :include:/var/test/majordomo/lists/vivaldi
owner-vivaldi:        bud@example.com
vivaldi-request:      "|/usr/test/majordomo/wrapper request-answer vivaldi"
vivaldi-approval:     bud@example.com

so, it was set up more like distribusion list rather than useful 
majordomo.

what I had to do to reflect the change on test.config was to change
the aliases file so it reads

vivaldi:              "|/usr/test/majordomo/wrapper resend -l vivaldi vivaldi-list"
vivaldi-list:         :include:/var/test/majordomo/lists/vivaldi
owner-vivaldi:        bud@example.com
vivaldi-request:      "|/usr/test/majordomo/wrapper request-answer vivaldi"
vivaldi-approval:     bud@example.com

now i can do all kinds of stuff with the .config file.
