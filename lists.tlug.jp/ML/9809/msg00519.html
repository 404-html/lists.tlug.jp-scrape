craigoda@example.com wrote:

> There are probably several ways to do this.  Under xdm, I think there is
> the option to have a console that has the error messages.  If you start
> X with something like
> 
>   $ startx
> 
> you can do
> 
> 1)  $ startx > X.log  2>&1
> 
> Then within an xterm do
> 
> 2)  $ tail -f X.log
> 
> Explanation:
> 1) run the standard startx script, but direct both standard output
> and standard error to the file X.log
> 
> 2) print a continuous listing of the contents of the file X.log.  Since
> the file X.log can grow with additional error messages, the -f flag is used
> to specifiy that tail not exit when it reaches the end of the file.
