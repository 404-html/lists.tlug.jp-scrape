On Tue, 8 Sep 1998, Eric S. Standlee wrote:

> on 'make clean all' in the lxdialog dir:
> 
> [root@example.com lxdialog]# make clean all
> rm -f core *.o *~ lxdialog
> gcc -O2 -Wall -fomit-frame-pointer -DLOCALE  -DCURSES_LOC="<curses.h>"
> -c lxdialog.c -o lxdialog.o
> In file included from lxdialog.c:22:
> dialog.h:29: curses.h: No such file or directory
> make: *** [lxdialog.o] Error 1
> 
> Is that alright?
> 
> If I go to no modules, will that make my kernel horribly big?
> 
> As always, thanks for the help.

You probably should just do 'ln -s ncurses/curses.h /usr/include/curses.h'
and try again.

On TL, at least on the 2.1.0 beta, the ncurses-devel package sets this
symbolic link.  I believe it's similar for RH.
