Takeshi Yokokawa "=?ISO-2022-JP?B?GyRCMiNAbiEhN3IbKEI=?= "  <yokokawa@example.com> writes:

> Now platex can find the graphics.sty but complains like:
> 
> >>! Package graphics Error: No driver specified.
> >>
> >>See the graphics package documentation for explanation.

That's graphics.sty from LaTeX2e asking for the graphics.cfg
file.  It should contain (at least one) command that looks like:

  % this file is called graphics.cfg
  \ExecuteOptions{dvips}

If you use a different driver (unlikely...), see the docs for
what's supported.  This one *.cfg file should be drawn upon by
the rest of the styles in the graphics bundle; so this should
pretty much be the last thing you need to configure to get the
package up and running (he said).

Hope this helps.
