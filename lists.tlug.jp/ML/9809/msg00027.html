>Not to start a flame war, but what is you definition of big? I would
>have thought that 27k was reasonable. Granted, that 27k to all the
>...
>Just curious as what you and others feel is the correct etiquette?

On a mailing list no attachments is a good rule of thumb. I'd only make an
exception if it was small and important for the current discussion. Putting
it up on a web or ftp server is better.

For normal email it's good etiquette to ask before sending a large
attachment. I'd define large as above 15-20K but then people are very
sensitive about bandwidth in Japan.

As a system administrator the real fun starts when people send a 2Mb Word
file to everyone else in their department/company. Mail server disk space
vanishes in an instant!
