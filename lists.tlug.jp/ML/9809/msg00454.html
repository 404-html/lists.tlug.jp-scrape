>Incidentally, when I was looking at the apache.org site today, they had an
>announcement that apache 1.3 has been released, and there is a a win32 build.
>How's that for an alternative to MS? :-)  They did mention that the win32
>version is not yet as stable as the UNIX version, but I'm sure that will get
>better.  They have the disclaimer that at this time, Windows support is
>entirely only experimental and they recommend Apache for win32 only for
>experienced users.  Doesn't sound quite like something I'd want to turn loose
>in a critical environment yet.

I've been running NT Apache for a few months now, and they don't really need
those disclaimers (IMHO). I had some problems with cgi and beta 3, but once
it was out of beta I've not had any problems at all. I'm using it on
real-world sites, though nothing really big yet.

Configuration files are identical to those in unix. This is great for me,
developing cross-platform cgi. Now I can develop Apache modules that run on
both NT and unix (and it works - I've been working on a project for IBM
Japan that is using this). The modules are DLL's on NT, so no need to
recompile Apache itself.

One warning: the default directory suggested by the installation program has
a space in it, which confuses some cgi programs. So change the default to
"c:\apache" or similar.

Darren
