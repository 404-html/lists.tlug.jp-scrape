Majordomo config file for tlug-ml on tlug server limits the size of
mail to 40000 bytes, and any mail item exceeding this size will not
be distributed but forarded to the mail list owner.  This 40000 bytes
is Majordomo default and so I assume the author of Majordomo does
not think 40k "too big".

At office I often receive non-compressed large mails of several 
hundred K from colleagues (many are getting through dial up RAS) ... 
this is really troublesome.
