I'm using RH 5.1, and have patched up to kernel 2.0.35. Recompiled and
run lilo and everything's fine so far.

When I boot, I have the following problem. After (or in the middle of?)
SCSI card checking, I get the following:-

"Unable to open initial console"

and it stops. I can load the old kernel no problems. My lilo.conf reads
as follows:-

boot=/dev/sda
map=/boot/map
install=/boot/boot.b
prompt
timeout=50
image=/boot/vmlinuz-2.0.34-0.6
	label=linux
	root=/dev/sda1
	initrd=/boot/initrd-2.0.34-0.6.img
	read-only
image=/boot/vmlinuz-2.0.35
	label=new-kernel
	root=/dev/sda1
	read-only
append="mem=127M"

I have no idea what the "initrd" line is for the original kernel, so I
omitted it for the new kernel. Do I need it? If so, how do I make the
new .img file?
