> For RHEL 4, bash 3.2 doesn't check $HOME/.profile

In Slackware, bash first reads /etc/profile and then any *.sh files
in /etc/profile.d.

It then reads ~/.bash_profile if invoked with the --login option, or
~/.bashrc otherwise.

At no point does it read ~/.profile unless instructed to by any of the
other scripts processed when it starts.
