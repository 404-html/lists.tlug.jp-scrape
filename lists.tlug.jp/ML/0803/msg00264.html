Josh Glover writes:

 > I guess reading the man page of a programme is the only way to
 > determine the resource class?

In general, yes.  The convention is that you capitalize the first
letter of the name, unless it's x, in which case you capitalize the
first two.  So the class for emacs is Emacs, and for xterm XTerm.

xemacs and lemacs of course were really emacs historically, so we
originally just used Emacs.  Then as the resources diverged
dramatically (rms is so selfcentered he doesn't believe in backwards
compatibility with previous versions of Emacs, let alone compatibility
with XEmacs), we used the rule that if "XEmacs" is used as a class
anywhere in the XRDB, it's XEmacs (coincidentally conventional),
otherwise it's Emacs.

The main exception AFAIK is that for Athena-widget-based programs,
there is the EDITRES protocol.  For participating programs, viewres
will show you the resources relevant to various widgets and editres
will allow you to actually change them.

Of course, XEmacs is a participant.
