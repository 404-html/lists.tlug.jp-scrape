Hi there

I got a VPS server with some limitations but it's cheap and powerful :)
I can't setup SElinux, I can't create encrypted file systems and I can't create nested virtual instances on it.
The same server will be used as production and pre-prod, so developers will be able to login by ssh/sftp/svn to update webfiles only. They are free to do whatever they want on pre-prod sites but final changes on production will be done by me.
My concern is about PHP. It's easy to restrict users access to certain folders by a good group/permission policy and also ssh-jail them but Apache will be the final user of those PHP scripts and Apache user doesn't have the same restrictions as a limited user. It can browse whatever is browsable for a common generic user. 

Any suggestion to prevent this scenario?

Thanks!
