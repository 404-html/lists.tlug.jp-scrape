On 4 September 2013 15:21, Stephen J. Turnbull <stephen@example.com> wrote:
> When everything is on the same LAN, I don't really see why.

Neither do I. This is exactly the kind of use case for which mDNS was
devised. The only real drawback with mDNS is that there is no real
countermeasures against poisoning/spoofing; then, again, if you are
not running some sort of key authentication on any non-trivial
communication, or, come to think of it, are letting untrusted people
into your private network to begin with, you have got other security
issues to worry about.
