On Tue, 13 Jan 1998, Jim Schweizer wrote:

> Well, I guess the .xsession WAS the ticket after all. Thanks
> Rob. Now, if I could just figure out where that console xterm is
> coming from:-)

  That console xterm is specified in the /var/X11R6/lib/xdm/Xsetup_0
file that Craig Oda mentioned.  This is the place, among other things,
to specify a background for the login window, if you want one.

  A quick summary of the main ideas contributed to this thread
regarding booting to the xdm login window:

  1. Edit /etc/inittab default runlevel to read:
           id:4:initdefault:

  2. Copy the appropriate /var/X11R6/lib/xinit/xinitrc.* file to
     /var/X11R6/lib/xinit/xinitrc , in order to start your window manager
     of choice.

  3. cp ~/.xinitrc to ~/.xsession for all users on your system to preserve
     personal tailorings of the X startup.

  [ 4. Optionally put a line like this:
           xv  -root -quit ~denismcm/wallpaper/newfoundation.jpg
     in the /var/X11R6/lib/xdm/Xsetup_0 file to colour up the login
     window background.]

  All this works just right on my Slackware 3.1 (yeah, I know...)
system with kernel 2.0.33.
