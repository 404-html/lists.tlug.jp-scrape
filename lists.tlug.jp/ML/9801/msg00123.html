>   Copying any one of the xinitrc.* to xinitrc will bring up the corresponding
> w-manager, but I don't see how we get the personal ~/.xinitrc files read.
> Any further ideas?

If you boot to X directly, the file that your system will look for
is called .xsession, not .xinitrc.  Probably there is a setting somewhere
where you could change it to look for .xinitrc, but as a default
for all systems as far as I know, your system will look for
/home/username/.xsession and if it doesn't exist, will use a default
file somewhere else.  It will in fact ignore .xinitrc.
