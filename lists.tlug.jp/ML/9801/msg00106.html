Hi,
it appears that the Xlib that comes with both Debian Hamm and
Red Hat 5.0 is not compiled with locale support.

Linux JDK1.1.5 will give the following error without locale
support. 

  /usr/local/jdk1.1/lib/Linux/green_threads/libawt.so: undefined symbol:
  _Xsetlocale (libawt.so)
  java.lang.UnsatisfiedLinkError: no awt in shared library path


There are two paths toward the solution.

  Path One - lazy one - my path: get the binaries from 
     ftp.xfree86.org.  Make sure you get the libraries 
     compiled with glibc6

  Path Two - probably better 
      get the source for X, get the locale add-on for glibc2 if you
      don't already have it, compile locale support into the X libs
      directly.  


I've only got this to work on Debian at the moment.  I haven't
tried Red Hat yet, but I read on the java-linux-devel, that 
RH5 had the same problem.
