schweiz># Default runlevel.
schweiz># id:5:initdefault:

schweiz>
schweiz>and yep, it boots right to an X login. Funny thing is, though, it
schweiz>doesn't read ~USER/.xinitrc anymore. 

The login is probably xdm, or X Display manager.

In /usr/X11R6/lib/X11, you may have a sub-directory called
  xdm/

You may have a file called xdm-config.

Now, cat the xdm-config file:

cow:/etc/X11/xdm$ more xdm-config 
! $XConsortium: xdm-conf.cpp,v 1.2 93/09/28 14:30:32 gildea Exp $
DisplayManager.authDir:         /var/lib/xdm
DisplayManager.errorLogFile:    /var/log/xdm-errors
DisplayManager.pidFile:         /var/run/xdm-pid
DisplayManager.keyFile:         /etc/X11/xdm/xdm-keys
DisplayManager.servers:         /etc/X11/xdm/Xservers
DisplayManager.accessFile:      /etc/X11/xdm/Xaccess
DisplayManager._0.authorize:    true
DisplayManager._0.resources:    /etc/X11/xdm/Xresources_0
DisplayManager._0.setup:        /etc/X11/xdm/Xsetup_0
DisplayManager._0.startup:      /etc/X11/xdm/Xstartup_0
DisplayManager._0.reset:        /etc/X11/xdm/Xreset_0
DisplayManager*resources:       /etc/X11/xdm/Xresources
DisplayManager*setup:           /etc/X11/xdm/Xsetup
DisplayManager*startup:         /etc/X11/xdm/Xstartup
DisplayManager*reset:           /etc/X11/xdm/Xreset
DisplayManager*session:         /etc/X11/Xsession
DisplayManager*userPath: 
/usr/local/bin:/usr/bin:/bin:/usr/bin/X11:/usr/games



Okay, now grep through those files for fvwm or read the 
scripts and see if it is calling another config file.

I had this same problem with xdm when I first played around
with it.  The startup files can be different which confused
me.  Hey, after all this talk, I think I'll set my run-level
to use xdm again, just for the excitement of it all.
