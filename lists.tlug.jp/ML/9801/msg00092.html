Subject: Linux Japanese RPM Project

Hi, 

I just looked at your Linux-Nihongo page where you said:

> The Linux Japanese RPM Project takes the JE distribution and converts it
> into RPM format using these tools: 
> 
>        tgz2rpm 
>        jeinst2rpm 
>        je2rpm 
>        spec2rpm 
>        rpmrm 
> 
> Since Project Japanese Extensions is making the JE distribution obsolete 
> and PJE has a RPM distribution, it is likely that the Linux Japanese RPM
> Project will be either absorbed by PJE or be made obsolete by it. 

Well, this is not true.  The RPM packages of the Linux Japanese RPM
Project are prepared from the source, which you can see from the
src.rpm files.  I have contributed one package to it.  It's completely
independent from JE, and it doesn't have any obsolete software of the
old JE distribution.

I'm under the impression that the JRPM project may be more helpful to
people reading the Linux-Nihongo page than PJE, since I think it's
more up-to-date with Red Hat Linux and you can install any packages
individually as you please.  With PJE, if I understand it correctly,
you have to first install ezinst, etc., even when you use rpm
packages.  

Makoto Kanazawa
