Two questions about kterm. I am using the kterm from TLJ 4.0 
on TLC 4.0 (Chinese). It works quite well with jvim, kinput2+canna, 
kinput2x+atok, etc. But, it gives the following error when invoked 
usually within xterm or cxterm. 

Warning: translation table syntax error: Unknown keysym name:  Space
Warning: ... found while parsing 'Shift<Key>Space:      begin-conversion(_JAPANESE_CONVERSION)'
Couldn't set locale: ja_JP.eucJP,ja_JP.ujis,ja_JP.EUC,japanese.euc,Japanese-EUC,ja,japan

What am I missing? It looks like the locale files exist within
/usr/share. 

I'm wondering if two errors that I get are not related to the 
above problem.
The first is that the command to startup atok, (kinput2x -atok -server
localhost) , results in a "can't connect" error (sometimes it 
works, sometimes it doesn't).
The second is that xterm, kterm and other terminals complain 
that the display is not set. this can be fixed by setting the 
display with export DISPLAY=<machinename>:0 . Are these 
problems occurring because I am defying some law of nature 
by running kterm on a Chinese-oriented system?
