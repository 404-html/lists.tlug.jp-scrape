I've been thinking about Ghost, and it might work, but there's the 
annoying bits that I have to have an NT/2k/XP system floating around to 
"manage" stuff...

The second bit, doing it with RH and copying the NT partition files, 
could work -- it requires fiddling, though, as you point out...  And 
"dd" could take forever.  As an option, however, if I go for FAT, I can 
use "rsync" to clean things up, instead of "dd":  this would result in 
a more "vulnerable" filesystem (no permissions), but more easily 
cleaned -- this might result in more frequent maintenance, but likely 
more easy.  (If ntfs writes were "safe", things would be much easier -- 
permissions locks to reduce abuse, and "rsync" to clean up anyway.)  As 
an additional bit moving me to "rsync", XP and its ilk should 
(hopefully) fit in a 2GB partition, reducing the space we have to 
scour...
