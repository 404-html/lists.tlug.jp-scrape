  • Subject: [tlug] Postfix + IMAP
At the risk of being told to shut up and sod off (not necessarily in that 
order), I have a question for all the mail admins out there.

I have run Postfix for a goodly while now, so when the time came to setup the
mailserver for my domain, it was a no-brainer. The choice of which IMAP server
to use was less trivial, but in the end, it looks like Cyrus 2.1 is not a bad
choice.

The question is this: I want to use SpamAssassin as a Postfix SMTP filter, then
use Procmail as the SMTP delivery agent. Fine, I know how to do both of these
things. The real question is, will Postfix deliver mail to Cyrus mailboxes? My
dream is this: I will have multiple mail aliases set up (e.g. tlug@example.com,
colug@example.com, wm@example.com), and mail sent to these aliases will be delivered
in separate sub-folders of my inbox. Feasible? Anybody done something like this?

If there is a better way of doing this that still allows me to put SpamAssassin
and Procmail in the loop, I am open to that, as well.

Cheers,
Josh
