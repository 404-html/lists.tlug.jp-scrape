[tlug] Japanese input failure

  • Date: Fri, 6 Dec 2002 11:23:34 +0900
  • From: "hatsuhiro" <hatuhiro@example.com>
  • Subject: [tlug] Japanese input failure
There is a failure report of Japanese input posted on a bulletin in
OpenOffice.org. I used StarSuite6 and found the same symptoms. I am
depressed. Is there any solution? Please give me related information. My
environment is Canna36, Kinput2-v3.1-beta4, Slackware8, using KDE and
gcanna.

Referrence： OpenOffice.org 障害報告掲示板 日本語入力が停止します (Japanese)


Symptom：

When inputting Japanese:

Shift+Space    [あ]
a　i                [あ]       あい
Space            [漢字]    愛
Space, →, ←
Enter             [漢字]    藍
Enter             [あ]        藍    （a single kanji character）

It ends with no problem in this case. However:

Shift+Space    [あ]
a　i                [あ]       あい
Space            [漢字]    愛
Space, →, ←   （more than twice）
Enter             [漢字]    合い
Enter             [漢字]    合い
・・・・・・

After that, an input failure always happens. StarSuite frequently hangs up,
leaving such a file as /tmp/OSL_PIPE_0_SingleOfficeIPC_1234567890.

It looks to have no problems with functions with Insert and Home keys and
character conversion with arrow keys, but seems to have a few more troubles.

Sorry for some Japanese characters.

Hoshino
