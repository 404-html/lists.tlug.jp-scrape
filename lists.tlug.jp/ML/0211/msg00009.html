On Saturday 02 November 2002 08:33 pm, Eric O. Flores wrote:
> Distribution = Mandrake 8.2
>

I tried to get kinput2 working in Mandrake 8.2 with KDE 2.2.2 and found I had 
prohibitive problems. KDE applications would segmentation fault if I gave 
them the environment that would set their locale.

For example:

export LC_ALL=ja_JP
export LANG=ja_JP.eucJP
export XMODIFIERS='@example.com=kinput2'
kmail -inputstyle overthespot

And then it would SEGV.

> I've done locale changes several times thinking I got it fix but then
> it locks out my normal user account then I have to fall back to the
> above settings.  The locks means when trying to log in as normal user
> into KDE it tries but returns to the log in screen.  Any other GUI
> environment will work.  As you can see I am trying to use Japanese
> within the English environment.

I was also trying to get apps to allow Japanese with English as my default 
locale. I didn't try any other GUI, however.

I ended up fixing it by updating my KDE to 3.0.3 packages that were released 
for Mandrake 8.2 prior to Mandrake 9 being released.

I now run Mandrake 9 and can happily set the environment I mention above for 
at least kmail, mozilla, konqueror, knode, etc.
