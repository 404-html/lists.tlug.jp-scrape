> linux server.  The two win2k's are seen in the network and the w2k's can
> see the samba server but no access to the server yet is granted.  I wish

These are different problems.  You'll want to get the IP
address assignment sorted out first.

> Because I want dynamic ip addressing, I am using a logic for the above
> card (eth1) to match my my dhcpd.conf1
> 
> /etc/dhcpd.conf

Have you confirmed that that dhcpd is actually running and
listening on port 67?  (It only needs to be listening on eth1
in your case.  With RedHat, you can supply the command line
arguments to dhcpd in /etc/sysconfig/dhcpd to add this limitation.)
Do you have a packet filter rule that is keeping dhcp requests
from reaching the server?
