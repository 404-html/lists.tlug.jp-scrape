Charles Muller wrote:

> I have /mnt/win_h created with the right permissions, but when I try
> to do this, I get
>
>    [root@example.com chuck]# mount -o uid=root,gid=root /mnt/win_h
>    mount: /dev/sda5 is not a valid block device

You might wan't to look at the partitions of the disk. `fdisk -l
/dev/sda` should list them, if /dev/sda is there and there is a driver
loaded for it.  If it has worked once and the partition type didn't
change, then most probably you need to load a module.
You might wan't to check out the usb docs in the linux kernel source
under Documentation, or this:
http://www.linux-usb.org/USB-guide/book1.html and the "Mass Storage
Devices" section.


> It doesn't show up under
> "Hard Drives" but it does show up under "system devices."
