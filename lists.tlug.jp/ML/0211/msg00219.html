Assuming MD works like most distributions, you should be able to check
that with 
iptables -L -n  (I'm in FreeBSD right now, so someone correct me if that
-n is wrong--it should make the results numeric, that is port 22 rather
than ssh and usually makes the result pop up more quickly. I'm also
making the assumption that MD runs iptables.  If you get something like
no such command, or your kernel doesn't have support or something like
that, then try ipchains -L -n.

I'm also assuming that you meant to type that you're not sure rather
than now sure, if that is working. If the answer you get back is
something like INPUT  ACCEPT  OUTPUT ACCEPT FORWARD ACCEPT (again, can't
check the exact syntax at this instant as I'm not in Linux and FreeBSD
uses a different type of firewalling) then at least you know it's not a
firewall issue.
