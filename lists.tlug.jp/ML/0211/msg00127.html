Sure, of course. That's what this place is for - as long as you've done your
homework first.

> 1..Would it be possible to run apache as standalone
> server (without DNS server)?

Of course. Apache doesn't need DNS at all AFAIK. It either binds to a
virtual interface, in which case it knows its own IP#, or it differentiates
virtual domains based on their hostname. Either way round, it's the client's
DNS service which does the resolving.

> 2..Where should I replace IP address of my network, if
> I want someone to access my server from outside?

Do you have a fixed public IP#? If so, get your own domain name and point
the A record to your IP#.

If not, go and take a look at www.dyndns.org.

> 3..Where can I get simple "httpd.conf" file ? The
> default http.conf file have a lot of option which I
> don't understand.

The basic httpd.conf file requires little tweaking in fact. I suggest you
look at the manual installed with apache in order to get better acquainted
with all the options.

http://localhost/manual/
