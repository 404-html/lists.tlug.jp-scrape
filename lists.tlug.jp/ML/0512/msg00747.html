Josh Glover wrote:
>>I still find Emacs frustrating... too many keys to press... vi just seems so much more
>>direct
> 
> I am surprised at this statement. In vi, you have to press keys to
> enter edit mode. How is this more direct than Emacs, where you are
> always in edit mode, unless you start leaning on the Ctrl and Meta
> keys?

Of course, but complex ed/sed commands of vi are already known by every
system admin, as they <oops-i-guess>use it every day for file
manipulation/filtering</oops-i-guess>. Emacs commands must be learnt
from scratch, and are not obvious (they are used only in Emacs).

But I use both of them in fact: vi for short/quick changes to a file,
emacs when I think there is quite a difficult work to do, multiple files
edition, or to use special features such as remote files (files that
emacs will transparently open/save with ftp). And obviously for programming.
