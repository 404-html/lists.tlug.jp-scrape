Thanks for the decoding.  I think I would have been less mystified by it

Yeah, now I'm *re*-trying. Lemme know how it goes. 

I tried just Thunderbird first, checking the file at each stage.  First
stage, all the lines with "print" in them got marked with "//" on the
front, and the print was ghosted.

Second stage, all the lines from Mozilla prefs that had "print" in them
were added in at the bottom.  I had left the file open in KWrite, so was
informed that another program had changed it and given a choice of what
to do; I "Replace"d.

Third stage, I reopened Thunderbird, found my test file and printed it.
 A row of little squares.

I closed Thunderbird, after which I was informed that another program
had changed the file, and I let it be replaced.  All of the
commented-out lines had vanished.  I confirmed this by comparing this
file side by side with the Mozilla prefs file.  All the lines containing
"print" in the two files were identical.

Next I repeated the same process with Firefox, and ended up with only
duplicates of Mozilla prefs for any line containing "print," but with
print paper showing little squares where kanji and kana should be.

Just to be sure, I ran Mozilla Browser again, and printed Japanese just fine.

This is of course quite impossible, except in certain parallel universes.

I find that I get an odd line in Shell after trying to print Japanese in Firefox (but not when I try it with Thunderbird):

bart@example.com:~$ LANG=ja_JP XMODIFIERS=@example.com=kinpu?? firefox &
[1] 1237
bart@example.com:~$  NO FONT WAS FOUND Name["Sans"]

I can't guess what *that* is all about, but maybe it's a clue...
