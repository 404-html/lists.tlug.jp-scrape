  • Subject: [tlug] A Japanese/Unicode-savvy zip utility?
  • Organization: TNI
A customer sent me a large number of files for translation, as a zip
archive (from Windows). I was able to unzip the files in the default
utility that comes with Kubuntu, but the folders and files that have
Japanese names could not be opened after extracting. The file names
were all gibberish, and OpenOffice.org complained that the files did
not exist.

I rebooted in Windows, extracted the archive with WinRAR, and then
copied the result into Linux. The resulting files are all perfectly
readable (PDF, doc, and ppt) in Linux.

Is there a zip utility for Linux that does not have this problem with
files and directories that have Japanese names?
