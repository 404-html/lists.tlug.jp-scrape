bruno raoult writes:

 > Agreed. A relational DB does good locking.
 > And good CGI do too with plain text files.
 > 
 > When lock is well managed, it works, whatever the underlying data
 > management is.

Sure.  I don't want to manage locks, I want my DBMS to do that.  I
suspect the same will be true of the TLUG webmaster.

BTW, SQL is not a hard language to learn.  It's hard to use
efficiently, and the data description language has a lot of difficult
corner cases in most implementations.  But what difference does it
make if you have time to eat lunch vs. only have time to refill your
coffee mug while your recovery script runs?

It's also a generally useful tool.  I expect that TLUG webmasters will
in general already know enough SQL or be happy to pick it up.
