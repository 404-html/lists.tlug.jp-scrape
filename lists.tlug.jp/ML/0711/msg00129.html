Were you using en_US.utf8 before?  

I've found that in OpenOffice, in both Linux and FreeBSD, that I have to
set LC_CTYPE as ja_JP.UTF-8 (FreeBSD syntax).  Although all the other
programs I use work without problems if I have en_US.UTF-8, OpenOffice
requires ja_JP for scim to work.  There's probably some reason for this,
but I never even googled for it, just found it out by trial and error
and found the workaround simple enough so that I didn't bother trying 
to find a solution.
