The reason that I take issue with the comment (in the Linux kernel,
which is where I first saw it) is not because I am insulted, rather
because I feel that if you really *must* write code so clever /
elegant that it is resistant to understanding just by reading it,
you'd bloody well better spend half an hour figuring out how to
explain the code, and either comment extensively, or use a comment to
refer the reader to external documentation. As in the Bell Labs
example, it is often the case that if you do not take the time to
explain something, you do not truly understand it.

You know the old saying: "the best way to learn is to teach".
