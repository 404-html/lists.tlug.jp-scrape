On 10/27/05, Marc Christensen <marc@example.com> wrote:

> Most likely, if they are opening port 80, they're also opening port 443
> (https).
>
> If so, and you are not running a secure web server at home, you could
> run an instance of sshd on port 443 on your home machine:
>
>     # sshd -p 443

This would work provided that his work environment does not force
HTTP[S] connexions through a proxy. In that case, a Corkscrew-like
solution (i.e. tunnelling SSH over HTTP) is the only way forward.
