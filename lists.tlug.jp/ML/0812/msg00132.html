Lyle H Saxon wrote:

After using automatic update arrangements with RedHat, Suse, & Kubuntu, I suddenly don't know how it's done with Debian (it's not the same, is it?).

I recently saw a package in the Debian repositories that seems to run apt-update automatically like the automatic updates in Red Hat, Suse, etc. The problem is, though, that once you begin to do that, if you are running Testing or Unstable (I know of very few individual users who run the Stable version), eventually there is going to be a broken dependency somewhere, and if you are running your update automatically, you won't be aware of it, and will thus create the same problems you experienced with those other distros.

I would suggest that you just run apt-update manually once every couple of weeks or so, and watch it to make sure it finishes correctly. That should keep you sufficiently updated with all the latest software.

I also don't know how to switch over to KDE and use that instead of the default Gnome, or how to install various packages.

The way I install Debian is to just install the basic system (without any other recommended Desktop/Laptop/X setup). Then reboot and do

   apt-get install x-window-system kdm kde

This will give you a basic system with X and KDE (without Gnome), and you can then add whatever packages you need after that, as you need them.

I have my own personal file of the packages (continually being updated) that I like to keep on my system. I just install these, so I don't need to install all the other junk that gets included in "recommended" installations. I'll attach it below FYI.
