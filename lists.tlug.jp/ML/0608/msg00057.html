> So by default, you should have netfilter/iptables support enabled

He won't.

As already mentioned, there were significant changes in how netfilter
is managed at the kernel config layer between 2.6.15 and 2.6.16 (I
originally thought it was 2.6.16 and 2.6.17 but was wrong).

If you cp /boot/config-2.6.15 /usr/src/linux-2.6.17/.config and then
from within the 2.6.17 tree run make menuconfig, you get a metric
buttload of error messages about undefined constants, all of them
relating to the netfilter subsection, and the relevant modules are
*not* built.
