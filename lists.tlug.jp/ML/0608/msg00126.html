"Hector Akamine" <akamine@example.com> wrote:

> (not sure about if it is secure enough). However I have found the
> following line does not allow mails to arrive to the root:
> 
> never_users = root
> ...
> setting, but I wondered why it is set this way
This is for security purposes. AFAIK one reason is that to deliver to
root, exim would need to run as root. The other reason is that the root
account is for administration and definitely not for reading mail. You
should set up /etc/aliases to have root's mail delivered to a normal user.
