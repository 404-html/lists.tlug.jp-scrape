  • Subject: [tlug] Change Apache's LogFormat. Webalizer fails to run
Hello all,

The default of LogFormat in apache is

LogFormat "%h %t %l %u \"%r\" %>s %b \"%{Referer}i\"
\"%{User-Agent}i\"" combined

but I wanted to see the return code easier, I changed the order of  %>s

LogFormat "%h %>s %t %l %u \"%r\"  %b \"%{Referer}i\"
\"%{User-Agent}i\"" combined

Of course, Apache was working fine, it logged request to access_log
but in different format in one file.

I observered the problem when I run webalizer with the log, it said:

1075456 records (490 ignored, 16480 bad) in 84.22 seconds, 12769/sec

so I have 16480 request in "bad" ( different ) format .

The simplest solution is to split access_log file into 02 different
files with different format: before the change and after the change.

But because the file contains more than 1 million lines, I want to do
something smarter.

Any idea?

Apache 2.0.48
Webalizer 2.01
