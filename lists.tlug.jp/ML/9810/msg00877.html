>>>>> "mc" == Michael Casinghino <michael@example.com> writes:

    mc> My mule uses shift-space to start kinput2.  How can I stop it
    mc> from doing this?

Under a Bourne shell (including bash), you can start Mule with

XMODIFIERS="" mule

in which case mule shouldn't use kinput2 at all.  I don't know how to
get that effect under csh or ksh.

Or, you can stop kinput2 from doing that ... if you don't want to use
it elsewhere.

Add

Kinput2*ConversionStartKeys: \
	Ctrl<Key>Kanji

to .Xresources or .Xdefaults (whichever you have), then do `xrdb <
.Xresources'.  But this means no other application can use shift-space
to wake up kinput2, either.  Unfortunately this is not something that
you can change easily on an app by app basis, in this respect XIM is
broken by design.  (I think I know how to do it by hacking the mule
sources, but I've not tried it.)
