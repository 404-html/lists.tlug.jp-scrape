On Tue, 13 Oct 1998, Scott Stone wrote:

>I have a weird thing happening here.  We have two machines that share one
>monitor via a switchbox.  If the switch is on A and the 'B' machine is
>powered up, we get beeps: 1 long, three short, just like the one we were
>discussing here before, that Jonathan's book says is a memory problem.

That's a 1-3-3 pattern for the memory problem.  There are a bunch of others,
too, depending on what kind of memory problem, and those were Phoenix
BIOS-specific, but it also has codes for other ones.

>However, it starts fine (no beeps) when the monitor switch is on B at
>powerup time.  Why would this be?

Does it have an energy-conserving monitor?  Maybe there is enough smarts in
the system that the BIOS can look for it and complain if it doesn't find it.
Tell me what kind of BIOS it is, and I'll see if there's a code like that when
I get home.
