ZD Net has an article this morning with the captivating title, "Is Linux
ready for publishing?"

Here's a short excerpt:

"Linux is a great desktop system, but it's not quite ready for print and
prepress," said Michael Hammell, a Dallas-based software engineer and
author of a forthcoming book on Linux graphics. The problem, Hammell
said, is that the underpinnings of current graphics systems --
PostScript, PDF and color management systems -- are proprietary and not
available as open-source code.... 

The article goes on to praise the GIMP, claiming it is in some ways
(e.g., scripting) superior to PhotoShop, but that it is more useful for
Web publishing than for print due to lack of color separation functions.
It also notes the lack of a Quark XPress. Even the Linux Journal uses a PC
running Windows 95 for XPress.

Try this link; if it doesn't work, go to the headline scan section and
find the above title. (ZD Net and others like to put commas in their
URLs, which don't work on some software as clickable links.)
