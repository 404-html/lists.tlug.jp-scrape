Thanks to everyone for all the info on my disk yesterday.  As usual, Linux
support is the best around.  I think (I hope!) I've made some progress.  I
zapped the old partition structure and re-fdisked it with separate
partitions as follows:

The first primary partition is a 128 meg swap.
The second primary partition is 128 meg for the root file system. 

After that is an extended partition containing 5 logical partitions, as
follows:

/var is 300 meg
/tmp is 300 meg
/home is 1000 meg
/usr/local is a bit over 1000 meg
/usr has everything else (around 3400 meg)

On the BIOS side, the reported info for the disk is:

Type                      [user]  6498 MB
Cyclinders                [12592]
Heads                     [16]
Sectors/track             [63]
Write precom              [none]
Multi-secotr transfers    [16 sectors]
Prefetch                  [enabled]
LBA mode control          [enabled]
32-bit I/O                [enabled]
Transfer mode             [Fast PIO 4]

When auto-detecting the disk, it always comes up with LBA active.

Fdisk is now reporting 16 heads, 63 sectors, 12592 cyclinders, so the zero
heads problem appears to be gone.  It still gives me the warning about LILO
and 1024, but the root partition is completely within the first 1024
cyclinders, so I'm hoping I won't have a problem there now.

I didn't have time to test it, but I  did a cp -a copy of the existing disk to
the corresponding partitions on the new disk.  Tonight I'll do the switch and
see how it goes.

Cheers,
