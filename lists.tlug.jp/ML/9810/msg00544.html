> kterm: error in loading shared libraries
> : undefined symbol: _Xsetlocale

What X are you using?  XFree86?
I use Accelerated-X, so it might be a little different,
but check where the Xsetlocale is defined, by

$ for i in /usr/X11R6/lib/lib*.a; do
>   echo $i
>   nm $i | egrep Xsetlocale
> done

In my Accelerated-X, _Xsetlocale is defined in libX11.a,
and __Xsetlocale is defined in libXintl.a
If not defined anywhere, but you used the same kterm binary
before, maybe some of the libraries been replaced.
