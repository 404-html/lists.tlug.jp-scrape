> With an xterm I can accomplish this with the following in my .Xdefaults
> file:
> 
>     XTerm*VT100.Translations:  #override <Key>F1: set-vt-font(6) \n\
>             <Key>F2: set-vt-font(d)
> 
> Unfortunately, this trick doesn't seem to work with a kterm.

Hmm, that's strange.  My kterm can work like this.
Maybe you're not setting the font lists correctly.
Or, make sure the class name is KTerm, not XTerm.
KTerm is an extension of xterm, so basically every feature for
xterm should work for kterm as well.

Here's my fontlist setting;

----8<----8<----8<----8<----8<----8<----8<----8<----8<----8<----8<----
*fontMenu.Label:	VT Fonts
*fontMenu*fontdefault*Label:	6x12
*VT100*fontList:		-local-tty-medium-r-normal--12-*
*VT100*boldFontList:		-local-tty-bold-r-normal--12-*
*fontMenu*font1*Label:		5x9
*VT100*fontList1:		-local-tty-medium-r-normal--9-*
*VT100*boldFontList1:		-local-tty-bold-r-normal--9-*
*fontMenu*font2*Label:		7x14
*VT100*fontList2:		-local-tty-medium-r-normal--14-*
*VT100*boldFontList2:		-local-tty-bold-r-normal--14-*
*fontMenu*font3*Label:		8x16
*VT100*fontList3:		-local-tty-medium-r-normal--16-*
*VT100*boldFontList3:		-local-tty-bold-r-normal--16-*
*fontMenu*font4*Label:		9x18
*VT100*fontList4:		-local-tty-medium-r-normal--18-*
*fontMenu*font5*Label:		12x24
*VT100*fontList5:		-local-tty-medium-r-normal--24-*
*fontMenu*font6*Label:		13x26
*VT100*fontList6:		-local-tty-medium-r-normal--26-*
*fontMenu*fontescape*Label:	Escape Sequence
*fontMenu*fontsel*Label:	Selection
----8<----8<----8<----8<----8<----8<----8<----8<----8<----8<----8<----

I create a /usr/X11R6/lib/X11/fonts/local/ directory,
and put local fonts there.  The fonts.alias in this directory
sets up various fonts for terminal to be named -local-tty-*...
