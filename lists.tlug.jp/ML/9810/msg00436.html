On Tue, 13 Oct 1998, Michael Chiu wrote:

> Hi,
> 
> If anyone can help... We are currently trying to use TL2.0J to install
> via SMB... We have placed the CD into a CD-ROM drive located on a WinNT
> 4.0 Server, shared the drive, etc. and now are having problems
> connecting to the server.
> 
> The name of the server is Altiserv, and the share name is E.  We have
> tried using combinations such as \\Altiserv with \E and without the
> \s... but fail to connect...

Use forward slashes, not backslashes.  That should be documented better
(note to self).  We tested this extensively, though, so after you keel
those slashes over, it should work great, albeit slowly (SMB is a member
of the 'garbage protocol' family, along with AppleTalk).  NFS is faster,
FTP is fastest. (of the TL install protocols, at least).  Well, local hard
disk is probably the fastest, but then again, you have to copy everything
to the HD first.. sigh.
