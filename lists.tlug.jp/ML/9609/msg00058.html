I started playing around with mule 19.28.1 with the w3 mode
to test out Japanese input on WWW pages.  I'm running
it under X-windows, but mule does not support inline 
graphics yet as xemacs does.  However, as far as I know,
the newest version of xemacs does not support Japanese.  Does it?

I'm starting to like the w3 mode of mule (emacs) because it is
pretty fast with just a few keystrokes.  Netscape always annoys
me when I have to use the mouse, the page up - page down buttons,
the arrows, etc.  Most of the commands for the w3 mode of mule are
on the main keyboard rows.  

I'm have a problem with links when the colors are specified in HTML.

We have this on our home page:

<Body BACKGROUND="twics/images/twicsbg2.gif"
TEXT="#045381" LINK="#1B14CD" VLINK="#E5002A " ALINK="# 800000">

On my mule, the links are unreadable unless I highlight it with
my cursor.  I've tried modifying my .Xdefaults file and have
not been successful.  If anyone is using mule w3, can you take
a look at http://twics.com  and see if you can read the links.

I suppose there is a newer version of mule...
