I don't know if this is a Debian issue, or if it is Linux-wide, but the most recent installations I've done with straight Linux and (K)Ubuntu all seem to disable the function of the right alt key--a royal PIA for anyone who uses (X)Emacs.
I found out that I can correct this by doing mapping it to the left alt key by doing this: 

xmodmap -e "keycode 108 = Alt_L"
(after determining that 108 is the proper number for the right alt key on my keyboard). But since this is only effective for the session, I also had to add it into a startup script.
This is all fine--for practical purposes the problem is solved. But it seems that there must be a way to do this at a more fundamental level. I'm just curious as to whether anyone has come across this and fixed it permanently and properly.
