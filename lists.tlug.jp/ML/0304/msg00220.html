Is it possible that you providers firewall is doing some sort of "fixup" on 
port 80 traffic?
Some firewalls do rewriting of protocols passing through them, for various 
reasons.
Perhaps the connection timeout is because the firewall is unable to recognise 
the ssh traffic passing by on port 80 and is dropping the packets as a 
result?
