It's not as simple as I thought.  Evidently TWICS is applying
different rules to TWICS lines than it does to everybody else.
(sendmail can do this.)  Or maybe not.

Given the formats of the error and the Received: header in David's
mail, I'm wondering if the problem isn't a failure of an IDENT check
rather than the HELO line.  (This is also weak as a spam check; every
Windows IRC client known to man spoofs identd; I'm sure the spammers
do so as well.)

Kei, if you're _not_ running an identd daemon, you could try running
one and see if that allows you to get through to TWICS.
