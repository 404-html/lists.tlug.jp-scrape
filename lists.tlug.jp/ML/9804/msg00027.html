On Tue, 31 Mar 1998, Christopher Sekiya wrote:

> On Tue, 31 Mar 1998, Gliebe Harald wrote:
> 
> > In the new version (3.3.2) the server is no longer SUID, but needs
> > Xwrapper to get started.
> 
> Yep.
> 
> If you're a heavy X user, I'd recommend using xdm and starting it from
> init.  Sidesteps a fair amount of nasty issues.

There's even an unpublished shortcut for this in TurboLinux (you can also
use the 'published' shortcut by running Xconfigurator/TurboXconfigurator
and telling it to start with a graphical login).  here's the quicker way:

/usr/bin/X11/change_initrl.pl -8bpp

(I know, the commandline is stupid - it's adapted from a different script
in 1.0 that would change the default color depth, before we started using
the DefaultColorDepth flag in XF86Config).  The only relevant commandline
options to change_initrl.pl are -8bpp and -text.
