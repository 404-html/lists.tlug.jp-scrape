I have an old 486DX2 running RH Linux 4.2 here at the office.
Admittedly I don't use it alot (hence the delay in getting around to fix it)
but it has an annoying problem of giving a time-out problem on ethernet:

eth0: transmit timed out with status 8080, IRQ conflict?
eth0: timeout registers: 8080 8182 0306 e85a e720 0000 8000 0001
eth0: transmit timed out with status 0000, network cable problem?
eth0: timeout registers: 0000 8282 0606 e8e8 aaaa 8181 4141 6666.
eth0: transmit timed out with status 0000, network cable problem?
eth0: timeout registers: 0000 8282 0606 e8e8 0000 8181 4141 5555.
eth0: transmit timed out with status 0000, network cable problem?
eth0: timeout registers: 0000 8282 0606 e8e8 0000 8181 4141 8484.

[Note, I changed the cable and connected it to an independent branch 
of the network just to eliminate that variable]

I now seem to have narrowed the the problem to the point were I think that I
have only observed the problem after requesting a web page from it (it's 
running Apache).   In particular, it is most likely to occur if requesting a 
large (say 1MB file) from it.

Note however, that it doesn't normally have this problem oftherwise - 
transfering large file to/from it with SAMBA, or NETATALK.
It can run happily for a whole day, so long as I don't access any of the web
pages it serves.

It seems strange to me that I would get an IRQ error msg on eth0 only after
using http when it normally doesn't miss a beat.    Anyone have any 
suggestions on what I could try?  [i'm sure that all the IRQ settings are 
correct and free of conflicts]
