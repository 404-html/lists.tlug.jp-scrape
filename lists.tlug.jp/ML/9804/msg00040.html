Well, I built mine from the TL pine-3.96 SRPM (patched the .spec file to
apply the J patch), which was in turn taken from the RH 5.0 pine-3.96
SRPM.  Redhat did some path manipulation in their patches (and I had to
remove -lcrypt to make it build on libc5) - looks like you had build
failures.  if an executable fails, pine's makefile doesn't quit, it just
keeps going (kind of stupid).  Anyway, you can always  get my .src.rpm and
rebuild it from that if you *really* want to compile it yourself :)
