I had this problem once upon a time.  I just hacked out the offending
#include.

I think the problem will go away if you install the pppd-2.2+ (which
you will need to do with 1.3.45+ anyway).  Get the latest from
/linux/system/Network/Serial.  Follow the steps in README.linux
(one of which you can skip... the one about making sure the Makefile
matches the given pattern... it does nowadays).  One of the steps
will cause the makefile to install/move include files.  Then:
  a) rebuild kernel (and modules if using those)
  b) build/install the new pppd
Life will be good.
