    Lyle> Is there some kind of limit on the number of .jpg files that
    Lyle> can be in one folder?

You could (theoretically) run out of inodes or space, but that error
would occur at write time, not read time.  Apache doesn't care, for
sure.

If you have a traditional Unix file system, the directory search is
basically linear which can be somewhat slow.  However, this shouldn't
kick in until you've got literally thousands of files, and probably
would not be noticable over the Internet (network latency should be
much higher than file access time).  Modern file systems (Reiser, even
recent versions of ext2) use hash tables which makes the lookup
essentially constant time.
