Lyle (Hiroshi) Saxon wrote:

> composer.  I'm sure there are much better tools available than Mozilla 
> Composer, but I've gotten used to using that.  Any suggestions for a 
> replacement?

Why, Vim, of course!

No, that was only half serious. If you like Composer, you are not going 
to like Vim, at least not at first. I'll just say that if you've never 
seriously tried to use a programmer's editor, you will be surprised at 
how productive you can be once you learn to use one. That goes for Vim, 
(X)Emacs ... those are the main two, but there's a few others that are 
good, maybe Nedit, Cooledit, etc. Give one or more of them a try if 
you're ever feeling adventurous.

But for something a bit more comfortable, you might try Bluefish. It 
doesn't attempt the pseudo-WYSIWYG thing like Composer--you would be 
dealing directly with HTML code--but it's got lots of nice graphical 
tools to help you manipulate HTML.
