> F5, in Openoffice, is assigned for headline/body style
> templates, and Shift+Space doesn't work, either. How do you get the
> input for Openoffice going?
im-ja-xim-server uses the same settings as the gtk module does, so
whatever you have in im-ja-conf. (Preedit colors don't work with xim
though). You need to restart the server if you change any of your
settings, im-ja-xim-server doesn't read in the new settings yet, this will
be implemented soon though.

You should start im-ja-xim-server in a terminal and then start oowriter
(or some gtk1 app, sylpheed for example) with the above command in another
one. It's possible that the value of LC_ALL is different in your .xinitrc.
Also you could try recompiling with --enable-debug; when the the
xim-server gets a new connection from a client im-ja-xim-server should
start dumping a bunch of debug messages indicating that the client can
talk to the xim server.
