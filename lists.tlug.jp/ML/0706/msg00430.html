Charles Muller <acmuller@example.com> wrote:
> After installing the VLAN package in Debian, when I try to connect to 
> the Internet, I am presented with the same authentication/login 
> interface as in XP. But after plugging in my userID/password, I receive 
> an error message:
> 
>       Hostname Mismatch: The hostname in the server
>       security certificate does not match the name of
>       the server.
> 
>       Hostname of URL: webview.nw.tyg.jp
> 
>       Hostname of certificate: WebView

As stated in the error message, seems it doesn't like the (SSL?) certificate.  This is the same as if you go to https://aaa.com/ but the certificate returned by the site says it is for bbb.com.

In this case did you input the name "webview.nw.tyg.jp" some time during the client configuration?  Try using "WebView" instead so the URL agrees with the certificate.  If the client can't find the IP address of "WebView", you can try putting it in /etc/hosts.

> The VLAN package seems not to have a man page, and my googling for such 
> strings as "vlan howto" doesn't seem to yield any pertinent information.
> 
> Does anyone have any advice on possible strategies for solving this?

Seems like your University is using authenticated VLANs.  Not sure if this is the exact package you use but the Network Configuration Guides here shows how one is set up.

http://www.ind.alcatel.co.jp/library/manuals.html
