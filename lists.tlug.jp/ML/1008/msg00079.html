My $.02:
I have worked as a software engineer for 20+ years. Mostly C++ but also
a fair share of bash, sed,
awk etc. a lot of perl, SQL, php, javascript and recently even some
C#/.NET (and the some I only touched briefly like Java and Python).
I guess there are different approaches to learning a language and
refreshing it but mine has always
been to familiarize myself with the documentation and I keep a list of
links on my web site that I
update whenever I find a new useful site.
With all this I would say that I would never bother to remember stuff
that can a) be easily found in the documentation (like function calls)
or b) syntax, for those compiled languages where the compiler
produces intelligible error messages (gcc, for exampl did not always do
that).

I am honestly not sure your quiz would make such a great difference to
me, even though it is fun. that's not the way I operate. I almost tries
not burden my weak memory with too much details.
