Christian, Scott, Darren,

Thank you all for responding.


One idea is to try booting from a linux CD
Pretty good idea. I booted from my Ubuntu 10.04 live CD, and it got to the purple Ubuntu splash screen, way, way faster than when I boot off the hard drive. There was one point when I saw a screen with a command prompt, but it lasted maybe two seconds, not thirty.
If Darren's theory is right, and I think it is, this indicates strongly that the blame is with my Ubuntu install, not a BIOS issue. 

Other points:
I do have nomodeset in my GRUB config. These are the uncommented lines from /etc/default/grub: 

GRUB_DEFAULT=0
GRUB_HIDDEN_TIMEOUT=0
GRUB_HIDDEN_TIMEOUT_QUIET=true
GRUB_TIMEOUT=0
GRUB_DISTRIBUTOR=`lsb_release -i -s 2> /dev/null || echo Debian`
GRUB_CMDLINE_LINUX_DEFAULT="splash nomodeset video=uvesafb:mode_option=1280x1024-24,mtrr=3,scroll=ywrap" 
GRUB_CMDLINE_LINUX="splash vga=795"
I will try to post up a video somewhere, hopefully late tonight, as showing is always better than telling. 

Still... ideas are always welcome in the meantime.
Thanks to everyone for offering up so many great suggestions on how to diagnose this problem.
