I've read on the syslinux site that there used to be something like a 
pxe server.
With the current version, though, all you need is a DHCP server (win2k in my
case) and a tftp server (SuSE 9 pro). You'll need only two options supplied
by dhcpd, the tftpserver name or ip and the filename to fetch at boot 
(pxelinux.0
copied to /tftpboot from syslinux-2.08, version 2.06 that came with SuSE 
didn't
work).

Then you create a config file as outlined in:
http://portal.suse.de/sdb/en/2003/06/tsieden_pxe-install.html
and put it in |/tftpboot/pxelinux.cfg/default

I've successfully booted intel nics (intel boot agent v3.0.05) and
broadcom gigabit adapters (NetXtreme Boot agent v6.2.6)
