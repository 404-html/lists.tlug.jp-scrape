I've found, on two separate installs of the new FreeBSD-11, that fcitx-mozc
which always works pretty well for me, doesn't work on firefox  
without extra effort.

Usually, I put in .xinitrc 
export XMODIFIERS=@im=fcitx
export GTK_IM_MODULE=fcitx
export LC_CTYPE=en_US.UTF-8

This has worked through FreeBSD-10,x as well as several flavors of Linux.

However, on FreeBSD-11 it doesn't work with Firefox, though it works with
urxvt, Chromium, Libreoffice and even Thunderbird, which I don't use on
these machines, but tried to see if it was just a Mozilla thing.

If I start it with LC_CTYPE=ja_JP.UTF-8 firefox, it works.  I don't know
enough about this to even know where to look. Starting from terminals isn't
giving me any useful error messages.

I'm not planning to put very much time or effort into solving this, as I
don't use Japanese in a browser too often, and if I need to, I can either
use chrome or start it with the LC_CTYPE set.  However, I'm curious if
there are any FreeBSD-11 users who ran into something similar. I realize
it's only been out a couple of days--well, it was out, then it wasn't,
those who use it probably know the story, but regardless, has anyone else
run into this, or does anyone know of an easy fix?
