  • Subject: [tlug] Kweird
As mentioned recently, when Kword opened an RTF document, it displayed
mojibake and could not be corrected regardess of font settings.

Monopolysoft Turd can open the same RTF document and save it in Word 2000
format.  Kweird can open the resulting DOC file.  The result has no
mojibake.  The initial display has its formatting only moderately screwed
up.  Things can be improved somewhat by doing a "select all" and changing
the font to bitstream charter.  But then move the cursor to a line of mixed
Kanji and Romaji.  Move the cursor from left to right.  In positions near
the left, the cursor is displayed between characters as normal.  Near the
middle of the line, the cursor is displayed on top of characters, cutting a
character in half.  Near the right, the cursor might be displayed between
characters again but deletions and insertions will be in locations off by
one from where the cursor sits.

Kweird can save the file in several formats.  One of the possibilities is
RTF.  Try saving the file as RTF.  It warns you that some formatting might
be lost.  Accept it.  Close Kweird.  Start up Kweird again and open its own
saved RTF file.  Half of the Kanji and 10% of the Romaji are gone.  Kweird
only warned about the formatting, it didn't warn that half the contents
would be gone.
