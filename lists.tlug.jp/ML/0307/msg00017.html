Not if the network admin has more than half a neuron between hir ears.

Since 99.999% of the time the Return-Path:, From: and Reply-To: headers in a
spam e-mail are forged, the C/R request is going to be fired off to an
innocent bystander (if the spammer is hoe-jobbing someone) or to a
non-existent address (thus causing a bounce message which will trigger off
yet another C/R request).

Challenge-Response is totally brainless IMO.
