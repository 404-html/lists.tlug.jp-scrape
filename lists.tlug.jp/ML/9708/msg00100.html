hijimamashite.  Welcome to the group and thank you for your comment.
I hope to meet you at the next meeting. 

I would like to use Java because:
   1) I like it    :-)
   2) I'm developing on multiple platforms - Linux Intel, Solaris Sparc,
	possibly NT (not my decision) at some point in the future.
	If I use a native method, my program becomes non-portable.
	Also, in this particular case, I do not have access to the
	shell or the compiler on the Sparc.  I have to only use
	FTP.

I think nkf would work.  However, there must be some Java class
to do the encoding change.  I read fj.lang.java last night and
could not find anything.  Is there a FAQ specific to Japanese
questions, not just a translation of the English FAQ?

I remember that there were some objects to do the conversion of
ShiftJIS into unicode.  I think I read this in Internet User.
I'm trying to reuse someone else's code instead of porting
something like nkf from C into Java.   I'm lazy.  :-)
