I don't know how many people use aterm (or how many still use EUC). 

At any rate, someone who saw my page on Japanese in Unix like systems
was kind enough to send me an email, telling me that he's created a
patch for aterm 1.x that enables Japanese (in EUC.  Aterm itself doesn't
yet support UTF-8.)

The gentleman is a geexbox developer and can be reached at

calkina [at] geexbox.org.  He mentioned that he hopes, after September
when he hopes to have more Copious Free Time to work on some other
aspects of aterm, including UTF-8 support.

If anyone is interested (personally, I very much like aterm, though
these days, when I use EUC, I tend to use mrxvt) the patch is in CVS.

cvs -d :pserver:anonymous@example.com:/home/cvsroot co aterm1

Configure it with --enable-kanji --enable-xim and you will be able to
input Japanese in aterm with EUC encoding. 

The developer did say that if enough people email aterm's developer (it
seems to only be one person at present) the patch will be included in
aterm's next release. 

(Aterm's developer can be reached at sasha [at] aftercode.net)

So, thought I'd throw the information out here, as I doubt I'm the only
one who might like aterm, but wound up not using it very much since
there wasn't any Japanese support.  (FreeBSD's older version, 0.4.2, has
a patch for Japanese, which seems to be harder to find these days, but
there hasn't been a patch for 1.x.  I ~think~ that Debian, and
therefore, by extension, probably Ubuntu and friends, has an aterm-ml
but it doesn't do on the spot conversion.)

Note that if you do use the patch, the xim option isn't mentioned in
README.configure, but if look at the Makefile, you can find it.  

Thanks.
