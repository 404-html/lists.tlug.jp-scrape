Klaus Kudielka <klaus.kudielka@example.com> wrote,

> I still get lots of oopses and tend to believe it's 
> pcmcia-cs. If anybody successfully runs
> 2.2.x/pcmcia-cs-3.0.x on a VAIO, please send me your
> /usr/src/linux/.config

I run 2.2.1/pcmcia-cs-3.0.8.  However, I didn't compile this 
kernel myself (shame on me ;-) but use the lattest kernel of 
the Japanese TurboLinux distribution

  ftp://ftp.pht.co.jp/pub/turbolinuxj/updates/3.0/kernel/2.2.1/APM/kernel-2.2.1.apm-1.i386.rpm

(There is also the matching kernel source in the same directory.)

I have a PCG-505EX, which has a Ricoh RL5C465 pcmcia
controller.  As mentioned ealier, I have trouble with
CardBus cards, but 8-bit cards work without any problems.
(The whole system is perfectly stable.)

BTW, why did you by a 505V?  As far as I understand, the
only difference to the 505SX is that it has some additional
video capture stuff.  Is that right?  Is there a price
difference?

Manuel

PS: I assume that you did install a distribution, which by
    default uses a 2.2 kernel.  If not, there are a couple
    of other things that you have to update when installing
    a 2.2 kernel on an older distribution (eg, modutils).
