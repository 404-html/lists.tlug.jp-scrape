I  have mailed the  following message before  to tlug@example.com but I
looks like it has  not reached the  TLUG-mailinglist.  (I couldn't see
it, although I am subscribed). Therefore I try  again and apologize if
it reaches you several times.

After upgrading to  Redhat 6.0 with Kernel 2.2.10  I have the problem,
that the NFS directories of the other computer in my local net are not
mounted  at boot time.  NFS  file system support  is compiled directly
into   the Kernel, NOT as  a   module.  /var/log/messages contains the
error messages:

Jun 21 02:36:10 laputa portmap: portmap startup succeeded 
Jun 21 02:36:10 laputa mount: mount: RPC: Port mapper failure - RPC: Unable to send 
Jun 21 02:36:10 laputa netfs: Mounting NFS filesystems failed 
Jun 21 02:36:10 laputa mount: mount: RPC: Port mapper failure - RPC: Unable to send 
Jun 21 02:36:10 laputa netfs: Mounting other filesystems failed

/etc/fstab contains:

micky:/        /mnt/micky     nfs   rsize=4096,wsize=4096,hard,intr,nosuid 0 0

and after the system  has started, I  can mount the  root-directory of
micky by using `mount -a'. Before I upgraded from Redhat 5.2 to Redhat
6.0, the mounting worked correctly during booting.

Does anybody have an idea what I am doing wrong?
